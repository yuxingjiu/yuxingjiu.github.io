会话与进程

  1. tmux ls
  2. tmux a -t 0
  3. tmux new -s 新窗口
  4. tmux detach
  5. tmux new -s 名字

命令行的典型使用方式是，打开一个终端窗口，在这里输入命令，用户与计算机的这种临时交互称为一次”会话”

会话的一个重要特点是，窗口与其中启动的进程是连在一起的，打开窗口，会话开始，关闭窗口，会话结束，会话内部的进程也随之终止，不管有没有运行完

为了解决这个可以使会话与窗口”解绑”，窗口关闭时，会话并不终止，而是继续运行，等到以后需要的时候，在让会话绑定其他窗口

  1. 查看会话
tmux ls
  1. 切换会话
tmux a -t 0

回到原来断开的状态

  1. 离开会话
tmux detach
  1. 重命名一个会话
tmux new -s kali

find

基于某些条件查找一些文件
采用递归方式，根据目标的名称、类型、大小等不同属性进行精细查找

查找类型关键字描述
按名称查找-name根据目标文件的名称进行查找，允许使用”*“及”?”通配符
按文件大小查找-size根据目标文件的大小进行查找一般使用”+”、”-“号设置超过或小于指定
的大小作为查找条件常用的容量单位包括kB[注意k是小写]、MB、GB
按文件属主查找-user根据文件是否属于目标用户进行查找
按文件属组查找-group根据文件是否属于目标组进行查找
按文件类型查找-type根据文件的类型进行查找文件类型
包括普通文件[f]、目录[d]、链接文件[l]、块设备文件[b]、字符设备文件[c]等
块设备是指成块读取数据的设备[硬盘、内存等]
字符设备是指按单个字符读取数据的设备[键盘、鼠标等]
按照权限查找-perm指定搜索文件的权限
按照文件索引节点查找-inum可以使用 ls -li 文件 或 stat 文件 查看文件的inode节点
执行命令-exec对执行成功的文件执行指定命令 -exec command {} \;
逻辑运算符描述
-a表示且and
-o表示或or
  1. 根据名字查找
find / -name "*.conf" 2>/dev/null
  1. 按照文件大小查找
find / -size +1G 2>/dev/null
  1. 查找当前目录下24小时内文件内容被修改过的所有文件
find -mtime -1 -type f 2?/dev/null
  1. 根据权限查找
// 查找设置了suid位的文件
find / -perm -u=s -type f 2>/dev/null  // -u=s表示只要包含s权限即可
find / -perm u=s -type f 2>/dev/null   // u=s表示权限只有一个s才会匹配
find / -perm -4000 -type f 2>/dev/null  // 和第一个查找结果一样

// 查找设置了sgid的文件


// 查找设置了stick的文件
  1. 根据文件类型
find / -type f -name sqlmap 2>/dev/null
  1. 根据文件索引节点
find -inum 节点号 -exec ls -li {} \; // 根据节点号对文件进行查看
find -inum 节点号 -exec rm -i {} \;  // 根据节点号对文件进行删除
find -inum 节点号 -delete // 根据节点号对文件进行查看
  1. 根据属主和属组查看
find / -user 用户 -type f 2>/dev/null
find / -group 组 -type f 2>/dev/null
  1. find提权（-exec）
sudo find / -name 1.txt -exec /bin/bash -p \;

sudo

sudo授权文件位于 /etc/sudoers

这条配置命令意思是 : 授权sudo组在在我当前主机所有角色以所有用户身份去执行所有的指令

第一段：用户名或者用户组，表示谁有权限来使用后面的配置。%sudo代表sudo组下的所有用户
第二段：表示来源地，即从哪执行这条命令。ALL表示所有计算机
第三段：表示sudo可以切换到什么用户。ALL表示所有用户
第四段：表示sudo可以切换到哪些组下的用户。ALL表示所有组
第五段：表示sudo之后能够执行的命令。NOPASSWD:ALL表示执行任意命令都不需要密码

sudo -l // 查看授权了哪些命令

新用户解锁

新用户创建时不设置密码，会在 /etc/shadow 的密码位置显示 **!**，表示锁定状态，而可以通过 passwd 和 usermod 来对其进行解锁，但是 passwd -u 新用户 不会产生解锁效果，因为没有设置密码，可以直接 passwd 新用户 设置完成后就解锁了，或者通过 usermod -p 密码 用户 来解锁

┌──(root㉿kali)-[/home/mpy]
└─# useradd -d /home/test -s /bin/bash test

┌──(root㉿kali)-[/home/mpy]
└─# tail -1 /etc/shadow                    
test:!:19418:0:99999:7:::

┌──(root㉿kali)-[/home/mpy]
└─# passwd -u test     
passwd：解锁密码将产生一个没有密码的账户。
您应该使用 usermod -p 来为此账户设置密码并解锁。

passwd

passwd 用户名，直接设置密码

┌──(root㉿kali)-[/home/mpy]
└─# passwd test   
新的 密码：
重新输入新的 密码：
passwd：已成功更新密码

┌──(root㉿kali)-[/home/mpy]
└─# tail -1 /etc/shadow
test:$y$j9T$LR1t9xMWp4vl7P3SE3Opn0$L4JQURpxVUrF0ixYB6qTo82HhpTo0EHUqslEcvlkiCD:19418:0:99999:7:::

usermod

usermod -p 密码 用户，但是这样设置后就变成了明文密码，可以通过openssl passwd -1 -salt 用户 密码 生成加盐密文，在使用 usermod -p 密文 用户配置即可（这里的密文一定要加上单引号）

┌──(root㉿kali)-[/home/mpy]
└─# usermod -p 123456 test

┌──(root㉿kali)-[/home/mpy]
└─# tail -1 /etc/shadow   
test:123456:19418:0:99999:7:::

┌──(root㉿kali)-[/home/mpy]
└─# openssl passwd -1 -salt test 123456             
$1$test$at615QShYKduQlx5z9Zm7/

┌──(root㉿kali)-[/home/mpy]
└─# usermod -p '$1$test$at615QShYKduQlx5z9Zm7/' test
  
┌──(root㉿kali)-[/home/mpy]
└─# tail -1 /etc/shadow                             
test:$1$test$at615QShYKduQlx5z9Zm7/:19418:0:99999:7:::
