部署YUM仓库

指定软件仓

  1. 使用本地【基于光盘】
/etc/yum.repos.d/*
  1. 重新挂载光盘
umount /dev/sr0
mount /dev/sr0 /media
  1. 移动源文件
mkdir /root/yum.bak
mv /etc/yum.repos.d/* /root/yum.bak
  1. 指定源
vim /etc/yum.repos.d/test.repo

[local-yum]
name=local-yum
baseurl=file:///media/
enabled=1
gpgcheck=0

// 两个都一样

[test]
name=test
baseurl=file:///media
enable=1
gpgcheck=0
  1. 清除yum缓存，重新生成
yum clean all && yum list && yum repolist
  1. 安装软件
yum install -y 软件
yum install -y vsftpd

服务器安装源

  1. 服务器端设置
    1. 修改挂载点，将目录设置为ftp共享目录
    2. 直接将挂载点设置为默认的/var/ftp/
  2. 重新挂载光盘
umount /dev/sr0
mount /dev/sr0 /media/
  1. 设置好ftp服务[允许匿名访问，修改他的默认共享路径为/media]
vim /etc/vsftpd/vsftpd.conf

anonymous_enable=YES // 开启匿名访问
local_enable=NO // 关闭本地用户
write_enable=YES // 开放服务器写的权限
anon_umask=022 
anon_upload_enable=YES // 允许匿名上传
anon_mkdir_write_enable=YES // 允许匿名用户创建目录
anon_other_write_enable=YES // 开放其他写入权
anon_root=/media // 设置共享路径
dirmessage_enable=YES

ftp://IP地址可以访问后即可

  1. 客户端设置
    1. 保存原来的源
mkdir /root/yum.bak
mv /etc/yum.repos.d/* /root/yum.bak
  1. 新建源
vim yum-ftp.repo

[yum-ftp]
name=yum-ftp
baseurl=ftp://IP地址
enable=1
gpgcheck=0
  1. 清除原来的源缓存
yum clean all
  1. 更新源
yum list

网络安装源

  1. 下载互联网YUM源（以阿里为例）
  2. 在浏览器里搜索阿里巴巴开源镜像 https://developer.aliyun.com/mirror/
  1. 找到对应的版本进行下载
wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo

vi使用

三种工作模式

命令模式 –输入a、i、o–> 进入输入模式 –按esc–> 命令模式 –输入:–> 进入末行模式

输入模式

a : 当前光标的下一行位置插入
i : 当前光标位置插入
o : 下一行的首位置插入

命令模式

  1. 光标操作
操作描述
:set nu显示行号
:set nonu不显示行号
#G跳转到第几行[# 代表行号]
G跳转到文件的行尾
gg跳转到文件的行首
  1. 删除操作
操作描述
x或Del删除光标处的单个字符
dd删除当前光标所在行
#dd删除从光标处开始的#行内容
d^删除当前光标之前到行首的所有字符
d$删除当前光标之后到行尾的所有字符
  1. 复制操作
操作描述
yy复当前整行的内容到剪贴板
#yy复制从光标处开始复制#行内容[从光标处开始往下数#行，一起复制]
  1. 粘贴操作
操作描述
P将缓冲区中的内容粘贴到光标位置处之后
p粘贴到光标位置处之前
  1. 文件内容查找
操作描述
/word从上而下在文件中查找字符串word
?word从下而上在文件中查找字符串word
n定位下一个匹配到的字符串
N定位上一个匹配到的字符串
  1. 撤销编辑
操作描述
u按一次取消最近的操作，支持多次重复按u恢复到原来的状态
U取消对当前行所做的所有编辑
ZZ保存当前文件内容并退出

末行模式

输入 “:” 进入末行模式

  1. 保存退出
操作描述
:w保存修改的内容
:w /root/newfile另存为其他内容
:q未修改退出
:q!放弃对文件的修改直接退出
:wq保存并退出
  1. 编辑其他文件
操作描述
:e 其他文件的绝对路径从编辑当前文件切换到编辑其他文件
:r 其他文件的绝对路径在当前文件中读取其他文件的内容
:sp 其他文件的绝对路径分屏编辑多个文件
ctl+w后按上或下分屏编辑后切换编辑对象

查找替换

操作描述
:s/old/new将当前行找到的第一个old字符串替换为new字符串
:s/old/new/g将当前行找到的所有old字符串全部替换new字符串
:#,#s/old/new/g在行号”#到#”范围内查找old字符串，全部替换为new字符串
:%s/old/new/g将整个文件中的old字符串全部替换为new字符串
:s/old/new/c对每个替换动作进行提示确认

SAMBA服务

  1. SMB协议 ：服务消息块

  2. CIFS协议 ：通用互联网文件系统

  3. Samba协议

    1. smbd : 提供对服务器中文件、打印资源的共享访问[TCP139、445]
    2. nmdb : 提供基于NetBIOS主机名的解析[UDP137、138]

  4. 配置目录

/etc/samba
/etc/samba/smb.conf
  1. 配置文件检查工具
testparm

配置信息

配置目录

/etc/samba/smb.conf

[global] : 全局设置

[homes] : 用户目录共享设置

[printers] : 打印机共享设置

[myshare] : 自定义名称的共享目录设置

全局配置项

workgroup : 所在工作组名称

server string : 服务器描述信息

security : 安全级别，可用值[shell、user、server、domain]

log file : 日志文件位置，"%m"变量表示客户机地址

passwd backend : 设置共享账户文件的类型

共享目录配置项

comment : 对共享目录的注释、说明信息

path : 共享目录在服务器中对应的实际路径

browseable : 该共享目录在网上邻居中是否可见

guestok : 是否允许所有人访问，等效于"public"

writable : 是否可写，与read only的作用相反

read only : 是否可读

write list : 是否可写

valid users : 允许哪些用户或组可读【valid users=admin,@manager // admin表示用户，@manager表示manager组】

directory mask : 默认上传目录权限[0666]

create mask : 默认上传文件权限[0777]

辅助配置内容

注释行 : 以#开头的行

配置样例行 : 以;开头的行

结合grep命令可以提取有效配置行

cat /etc/samba/smb.conf.example | grep -v "^#" smb.conf | grep -v "^;" | grep -v ^$

搭建过程

建立数据库文件

  1. 安装rpm包
rpm -ivh samba-xxx-xxx.rpm
  1. 建立Samba用户
useradd testuser // 添加系统用户

pdbedit -a -u testuser // 添加用户testuser

pdbedit -vL testuser // 查看testuser用户信息
  1. \\192.168.123.123访问（默认情况下家目录共享）

设置用户访问权限

  1. 设置上传目录及权限
mkdir -m 777 /upload
  1. 配置文件新增
vim /etc/samba/smb.conf

[upload]
    comment=upload
    path=/upload
  public=yes
  read only=yes // 所有人可以读
  write list=admin // admin用户可写
  directory mask = 0755 // 默认上传目录权限
  create mask = 0644 // 默认上传文件权限
  1. 共享账号映射[别名]
vim /etc/samba/smbusers

添加以下信息

zhangsan = zs
root = administrator
  1. 启用别名
vim /etc/samba/smb.conf

在[global]中添加或修改
username map = /etc/samba/smbusers
  1. 重启服务
systemctl restart smb nmb

  1. 设置用户访问授权

    1. 新建共享目录与共享用户
    2. 修改smb.conf配置文件，添加名为tools的共享目录
    3. 重新加载smb.conf文件、或重启smb服务

  2. 确定用户访问权限

    1. 设置目录权限
    2. 设置上传文件和目录的默认权限

  3. 访问地址限制

    1. 一般用在全局配置[global]部分
    2. hosts allow配置项 : 仅允许特定的客户机
    3. hosts deny 配置项 : 仅拒绝特定的客户机
      1. 客户机地址表示形式
      2. 以空格分隔多个地址
      3. 主机名或IP地址，例如 : 192.168.12.101或者prtsvr
      4. 网络地址，例如 : 173.17.或者173.17.0.0/255/255/0/0
vim /etc/samba/smb.conf

[global]
hosts allow = 192.168.4. 173.17.

linux访问共享文件夹

  1. smbclient访问共享目录
smbclient -L 192.168.1.111 // 查看共享，只能在开启匿名访问才可以查看

smbclient -U 用户 //192.168.1.111/共享目录 // 登录共享
  1. 使用mount挂在共享目录到本地
mkdir /media/smbdir

mount -o username=用户名 //192.168.1.111/共享目录 /media/smbdir

FTP服务

FTP概述

控制连接 : TCP21，用于发送FTP命令信息

数据连接 : TCP20，用于上传、下载数据

  1. 数据连接的建立类型

    1. 主动模式 : 服务端从20端口主动向客户端发起连接
    2. 被动模式 : 服务端在指定范围内某个端口被动等待客户连接

  2. FTP传输模式

    1. 文本模式 : ASCII模式，以文本序列传输数据
    2. 二进制模式 : Binary模式，以二进制序列传输数据

  3. FTP类型

    1. 匿名用户、本地用户、虚拟用户

  4. 常见的FTP服务器程序

    1. IIS、Serv-U
    2. wu-ftpd、Proftpd
    3. vsfftp[Very Secure FTP Daemon]

  5. 常见的FTP客户端程序

    1. ftp命令
    2. CuteFTP、FlashFXP、LeapFTP、Filezila
    3. gftp、kuftp

vsftp概述

  1. 主程序 : /usr/sbin/vsftpd
  2. 服务名 : vsftpd
  3. 用户控制列表文件
    1. /etc/vsftpd/ftpusers : 黑名单，出现的用户禁止登陆
    2. /etc/vsftpd/user_list : 用户列表
  4. 主配置文件
/etc/vsftpd/vsftpd.conf

全局配置项

listen=YES : 是否独立运行的方式监听服务
listen_address=192.168.12.1 : 设置监听的IP地址
listen_port : 设置监听FTP服务的端口
write_enable=YES : 是否启用写入权限
download_enable=YES : 是否允许下载文件
userlist_enable=YES : 是否启用user_list列表文件
userlist_deny=YES : 是否禁用user_list中的用户
max_clients=0 : 限制并发客户端连接属
max_per_ip=0 : 限制同一IP地址的并发连接数

匿名FTP配置项

anonymous_enable=YES : 启用匿名访问
anon_umask=022 : 匿名用户上传文件的权限掩码 
文件默认是666，这里的022表示的是反码[666-022=644的权限]
目录默认是777[777-022=755的权限]
anon_root=/var/ftp : 匿名用户的FTP根目录
anon_upload_enable=YES : 允许上传文件
anon_mkdir_write_enable=YES : 允许创建目录
anon_other_write_enable=YES : 开放其他写入权
anon_max_rate=0 : 限制最大传输速率[字节/秒]

本地用户FTP配置项

local_enable=YES : 是否启用本地系统用户
local_umask=022 : 本地用户上传文件的权限掩码
chroot_local_user=YES : 是否将用户禁锢在主目录
local_max_rate=0 : 限制最大传输速率[字节/秒]

vsftp支持tcp_wrappers的访问控制

tcp_wrappers=YES

搭建服务

正常用户认证

  1. 准备匿名FTP访问的目录（使用匿名用户FTP对该目录有写入权限）
mkdir /var/ftp/pub

chown ftp /var/ftp/pub
  1. 开放匿名用户配置
vim /etc/vsftpd/vsftpd.conf

anonymous_enable=YES // 允许匿名访问
local_enable=NO
write_enable=YES // 开放服务器的写权限
anon_umask=022
anon_upload_enable=YES // 允许匿名上传
anon_mkdir_write_enable=YES // 允许匿名用户创建目录
anon_other_write_enable=YES // 开放其他写入权
dirmessage_enable=YES
  1. 设置白名单用户
vim /etc/vsftpd/user_list

testuser // 只允许该用户登录
  1. 配置文件开启白名单
vim /etc/vsftpd/vsftpd.conf

userlist_enable=YES // 是否启用user_list列表文件
userlist_deny=NO // 是否禁用user_list中的用户
  1. 重启服务
systemctl restart vsftpd

vim /etc/hosts.allow
vsftpd:192.168.12.100

vim /etc/hosts.deny
vsftpd:all

使用虚拟用户登录

  1. 建立虚拟ftp用户的账号数据库文件
vim /etc/vsftpd/vusers.list


test
123.com
user
1234.com
  1. 安装依赖
yum install -y pam
yum install -y libdb-utils
yum install -y libdb
  1. 转换文件
db_load -T -t hash -f /etc/vsftpd/vusers.list /etc/vsftpd/vusers.db
  1. 修改权限
chmod 600 /etc/vsftpd/vusers.*
  1. 创建被映射的用户（该用户不需要密码）
useradd -d /var/ftproot -s /sbin/nologin virtualftp

chmod 755 /var/ftproot
  1. 建立支持虚拟用户的PAM认证文件
vim /etc/pam.d/vsftpd.vu

#%PAM-1.0
auth		required	pam_userdb.so db=/etc/vsftpd/vusers
account		required	pam_userdb.so db=/etc/vsftpd/vusers
  1. 修改认证方式
vim /etc/vsftpd/vsftpd.conf

pam_service_name=vsftpd.vu // 修改pam认证文件
gusest_enable=YES
guest_username=virtualftp
allow_write_chroot=YES // 允许在根目录下面写
  1. 将虚拟用户以及映射用户添加到白名单
vim /etc/vsftpd/user_list

test
user
virtualftp
  1. 如果想要单独给映射账号设置权限（映射账号的权限默认是anon_xxx设置的权限）
cd /etc/vsftpd

mkdir /etc/vsftpd/vusers_dir

touch test // test是虚拟的账号
touch user // user也是虚拟的账号

vim user // 这样user用户会比test用户多两个权限

anon_other_write_enable=YES
anon_mkdir_write_enable=YES
  1. 主配置文件设置目录
vim /etc/vsftpd/vsftpd.conf

user_config_dir=/etc/vsftpd/vusers_dir
  1. 重启服务
systemctl restart vsftpd
