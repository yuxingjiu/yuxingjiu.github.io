Apache

Apache简介

  1. 主要特点
    1. 开放源代码、跨平台应用
    2. 支持多种网页编程语言
    3. 模块化设计、运行稳定、良好的安全性
  2. 软件版本
    1. 1.X
      1. 目前最高版本是1.3，运行稳定
      2. 向下兼容性较好，但缺乏一些较新的功能
    2. 2.X
      1. 具有更多的功能特性
      2. 与1.X相比，配置管理风格存在较大差异
  3. 主要目录和文件
目录位置
服务目录/usr/local/httpd
主配置文件/usr/local/httpd/conf/httpd.conf
网页目录/usr/local/httpd/htdocs/
服务脚本/usr/local/httpd/bin/apachectl
执行程序/usr/local/httpd/bin/httpd
访问日志/usr/local/httpd/log/access_log
错误日志/usr/local/httpd/log/error_log

安装httpd服务器

  1. 安装文件[httpd-2.4.54]
./configure --prefix=/usr/local/httpd --enable-so --enable-rewrite --enable-charset-lite --enable-cgi

如果缺什么环境直接yum安装

yum install -y apr*
yum install -y pcre*

编译

make && make install
  1. 启动服务
/usr/local/httpd/bin/apachectl start

netstat -anpt | grep 80
  1. 修改配置文件
vim /usr/local/httpd/conf/httpd.conf

找到ServerName，将注释去掉

  1. 设置系统服务启动
cp /usr/local/httpd/bin/apachectl /etc/init.d/httpd

启动方法为 : service restart httpd

http服务器基础

  1. 虚拟web主机

在同一台服务器中运行多个web站点，其中每一个站点并不独立占用一台真正的计算机

  1. http支持的虚拟主机
    1. 基于域名的虚拟主机
    2. 基于IP地址的虚拟主机
    3. 基于端口的虚拟主机

web站点部署

  1. 创建测试站点
mkdir -p /var/www/html
mkdir /var/www/html/mpya /var/www/html/mpyb
  1. 创建测试网页
vim /var/www/html/mpya/index.html

<h1>mpya</h1>

vim /var/www/html/mpyb/index.html

<h1>mpya</h1>
  1. 修改虚拟主机文件
vim /usr/local/httpd/conf/extra/httpd-vhosts.conf

<VirtualHost 192.168.12.5:80>
 DocumentRoot "/var/www/html/mpya"
 ServerName www.mpya.com
 ServerAlias www.dummy-host.example.com
 ErrorLog "logs/mpya.com-error_log"
 CustomLog "logs/mpya-access_log" common
</VirtualHost>

<VirtualHost 192.168.12.5:80>
 DocumentRoot "/var/www/html/mpyb"
 ServerName www.mpyb.com
 ErrorLog "logs/mpyb-error_log"
 CustomLog "logs/mpyb-access_log" common
</VirtualHost>
  1. 配置DNS
vim /var/named/chroot/etc/named.rfc1912.zones

zone "mpya.com" IN {
     type master;
     file "mpya.com.zone";
     allow-update { none; };
};
zone "mpyb.com" IN {
     type master;
     file "mpyb.com.zone";
     allow-update { none; };
};

cd /var/named/chroot/var/named

cp -p test.com.zone mpya.com.zone
cp -p test.com.zone mpyb.com.zone

vim mpya.com.zone

**$TTL 1D
@       IN SOA  mpya.com. xiaozheng.163.com. (
                                     2022100601      ; serial
                                     1D      ; refresh
                                     1H      ; retry
                                     1W      ; expire
                                     3H )    ; minimum
     IN      NS      ns1.mpya.com.
ns1     IN      A       192.168.12.5
www     IN      A       192.168.12.5**

vim mpyb.com.zone

**$TTL 1D
@       IN SOA  mpyb.com. xiaozheng.163.com. (
                                     2022100601      ; serial
                                     1D      ; refresh
                                     1H      ; retry
                                     1W      ; expire
                                     3H )    ; minimum
     IN      NS      ns1.mpyb.com.
ns1     IN      A       192.168.12.5
www     IN      A       192.168.12.5**

重启服务

systemctl restart named-chroot
  1. 修改主配置文件，开启vhosts的模块
vim /usr/local/httpd/conf/httpd.conf

将注释去掉即可[搜索vhosts] [Include conf/extra/httpd-vhosts.conf]
  1. 重启服务
/usr/local/httpd/bin/apachectl restart
  1. 给目录添加权限
vim /usr/local/httpd/conf/extra/httpd-vhosts.conf

<Directory "/var/www/html">
    require all granted
</Directory>
  1. 重启服务
/usr/local/httpd/bin/apachectl restart
  1. 访问站点[修改DNS为192.168.12.5]

make && make install
ln -s /usr/local/lib/libmhash* /usr/lib

./configure
{

export LD_LIBRARY_PATH=/usr/local/lib: LD_LIBRARY_PATH

make && make install
ln -s /usr/local/lib/

安装php

yum install libxml2*
tar zxf php
./configure --prefix=/usr/local/php5 --with-mcrypt --with-apxs2=/usr/local/httpd/bin/apxs --with-mysql=/usr/local/mysql --with-config-file-path=/usr/local/php5 --enable-mbstring
make && make install
{
// 如果编译失败

index.php

MySQL

MySQL

MySQL安装

确认没有使用RPM方式安装mysql-server、mysql

rpm -qa | grep mysql-server rpm -e mysql-server
rpm -qz | grep mysql rpm -e mysql

安装光盘自带的ncurses-devel包

rpm -ivh ncurses-devel-5.7

安装cmake包

tar zxf cmake
./Configure
gmake && gmake install

源码编译及安装

groupadd -g 1500 mysql
useradd -M -u 49 -s /sbin/nologin -g mysql mysql
tar zxf mysql.tar.gz
cmake -DCMAKE_INSTALL_PREFIX=/usr/local/mysql \
-DSYSCONFDIR=/etc \
-DDEFAULT_CHARSET=utf8 \
-DDEFAULT_COLLATION=utf8_general_ci \
-DWITH_EXTRA_CHARSETS=all
make
make install

安装后的其他调整

chown -R mysql:mysql /usr/local/mysql/
cp support-files/my-medium.cnf /etc/my.cnf

/usr/local/mysql/scripts/mysql_install_db --user=mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data/

echo "PATH=$PATH:/usr/local/mysql/bin" >> /etc/profile
chmod 777 /etc/profile
. /etc/profile

又或者（上面3行和下面1行意义一样都是可以执行mysql命令的）

ln -s /usr/local/mysql/bin/* /usr/bin/

添加为系统服务

cp support-files/mysql.server /etc/rc.d/init.d/mysqld
chmod +x /etc/rc.d/init.d/mysqld
chkconfig --add mysqld

MySQL使用

  1. 连接并登录到MySQL操作环境
mysql -u 用户名 [-p]
  1. 设置数据库用户的密码
mysqladmin -u 账号 [-p] password
  1. shell下使用sql语句
mysql -u root -p123.com -e sql语句

MySQL结构

  1. 查看数据列信息
show databases;
  1. 查看数据表信息
use 数据库;

show tables;
  1. 显示数据表(字段)结构
describe 数据表;
  1. 创建新的数据库
create database 数据库名;
  1. 创建新的数据表
create tables 表名 (字段定义……);
  1. 删除指定的数据库
drop database 数据库名;
  1. 删除指定的数据表
drop table 数据表;
  1. 指定数据表添加数据
insert into login (user, pass) values ("mpy", "123.com"),("mmm", "123.com");
  1. 查询表中所有数据
select * from login;
  1. 修改[更新]数据表中的数据
update 表名 set 字段名1=值1[,字段名2=值2] where 条件表达式
  1. 删除数据表中指定的数据记录
delete from 表名 where 条件表达式
  1. 设置用户权限[用户不存在时，则新建用户]
grant all on 数据库.* to '用户名'@'连接地址(localhost)' IDENTIFIED BY '密码';

基于域名的虚拟web主机

  1. 准备网站目录及测试文件
mkdir -p /var/www/html/test1
echo "<h1>test1</h1>" >> /var/www/html/test1/1.html

mkdir -p /var/www/html/test2
echo "<h1>test2</h1>" >> /var/www/html/test2/1.html
  1. 调整nginx.conf配置文件
vim /usr/local/nginx/conf/nginx.conf

// 找到http{，里面的每个server{}相当于一个主机
server { // 主机
  listen 81;
  server_name www.test1.com;
  access_log logs/test1.access.log main;
  location / { // 目录和权限
    root /var/www/html/test1;
    index index.html;
  }
}
server {
  listen 82;
  server_name www.test2.com;
  access_log logs/test2.access.log main;
  charset utf-8;
  location / {
    root /var/www/html/test2;
    index index.html;
  }
}

将http开头的几行注释给取消

重启nginx

nginx -s reload
nginx
  1. 测试web虚拟主机

根据IP访问

vim /usr/local/nginx/conf/nginx.conf

在server1中添加配置allow 192.168.0.1; deny all;

server { // 主机
  listen 81;
  server_name www.test1.com;
  access_log logs/test1.access.log main;
  location / { // 目录和权限
    root /var/www/html/test1;
    index index.html;
      allow 192.168.0.1;
      deny all;
  }
}

根据用户访问

vim /usr/local/nginx/conf/nginx.conf

在server2中添加配置auth_basic="ID"; // 这个是提示信息 auth_basic_user_file /usr/local/nginx/conf/.nginxpwd;

server {
  listen 82;
  server_name www.test2.com;
  access_log logs/test2.access.log main;
  charset utf-8;
  auth_basic="ID";
  auth_basic_user_file /usr/local/nginx/conf/.nginxpwd;
  location / {
    root /var/www/html/test2;
    index index.html;
  }
}

tar zxf tomcat

Tomcat站点搭建

mkdir -p /var/www/html/text/

vim index.jsp

<%@ page language="java" import="java.util.*" pageEncoding="UTF-8"%>
<html>
<body>
<h1>123</h1>
<% out.println("server-1");%>

vim /usr/local/tomcat/conf/server.xml

<Context docBase="/var/www/html/test3/" path="" reloadable="false"></Context>
  1. 配置值nginx
vim /usr/local/nginx/conf/nginx.conf

在http里面，server外面进行配置，去声明一个反向代理组的组名

upstream tomcat_server{

location / {

Squid缓存服务

web代理的工作机制 : 缓存网页对象，减少重复请求

squid安装

  1. 编译程序
./configure --proxfi=/usr/local/squid --sysconfdir=/etc/ --enable-arp-acl --enable-linux-netfilter --enable-linux-tproxy --enable-async-io=100 --enable-err-language="Simplify_Chinese" --enable-nuderscore --enable-gnuregex
  1. 创建一个squid的账户
useradd -M -s /sbin/nologin squid
  1. 设置权限
chown -R squid:squid /usr/local/squid
chmod -R 755 /usr/local/squid
  1. 启动服务
cd /usr/local/squid/sbin
./squid
  1. 编辑配置文件
vim /etc/squid.conf

replay_body_size 10MB
http_access allow all  // 允许所有的链接都需经过
  1. 重启服务
netstat -anptl | grep squid
kill -9 pid

/usr/local/squid/sbin/squid // 重启
  1. 设置透明代理
vim /etc/squid.conf

http_port 本机内网IP地址:3128 transparent
  1. 设置防火墙
systemctl start firewalld
iptables -F
iptables -t nat -I PREROUTING -i eth0 -s 192.168.3.0/24 -p tcp --dport 80 -j REDIRECT --to-ports 3128
