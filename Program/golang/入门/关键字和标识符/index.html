# 标识符

# 标识符的组成

在编程语言中标识符就是程序员定义的具有特殊意义的词，比如变量名、常量名、函数名等等

  1. 标识符由数字、字母、下划线组成
  2. 只能以字母和下划线开头
  3. 标识符区分大小写（根据首字母的大小写来确定可以访问的权限，如果首字母大写，则可以被其他的包访问，如果首字母小写，则只能在本包中使用）

# 命名规范

命名规则涉及变量、常量、全局函数、结构、接口、方法等的命名（任何需要外部调用的必须大写字母开头，不对外调用的小写字母开头），即命名如果以小写字母开头，对包外是不可见的，但是它们在整个包内是可见并且可用的，如果以大写字母开头，对保外是可见的，都能调用

# 包名称

保持 package 的名字和目录一致，尽量采取有意义的包名，简短，有意义，包名为小写单词，不使用下划线或混合大小写

# 文件命名

尽量采取有意义的文件名，应该为小写单词，使用下划线分隔各个单词

# 结构体命名

采用驼峰命名法，首字母根据访问控制大写或小写（如果是大写，则内部成员也应该是大写，反之一样）

# 接口命名

命名规则基本和上面的结构体类型一样

单个函数的结构体名以 "er" 作为后缀，像 Reader、Writer

# 变量命名

和结构体类似，变量名称一般遵循驼峰法，首字母根据访问控制原则大写或小写

如果变量为私有，且特有名字为首个单词，则使用小写，如 appService 若类型为 bool 类型，则名称应以 Has、Is、Can 或 Allow 开头

# 常量命名

常量均需使用全部大写字母组成，并使用下划线分词

如果是枚举类型的常量，需要创建相应类型
