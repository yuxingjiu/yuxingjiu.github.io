Go 语言中支持函数、匿名函数、闭包，并且函数在 Go 语言中属于 "一等公民"

# 函数定义

Go 语言中定义函数使用 func 关键字

函数名 ：由字母、数字、下划线组成，但函数名的第一个字母不能是数字，在同一个包内，函数名也不能重名

参数 ：参数由参数变量和参数变量的类型组成，多个参数之间使用，分隔

返回值 ：返回值由返回值变量和其变量类型组成，也可以只写返回值的类型，多个返回值必须使用 () 包裹，用，隔开

函数体 ：实现指定功能的代码块

# 可变参数函数

用 变量 ...数据类型 表示 str ...string

  1. 可变参数，在一个方法中只能有一个
  2. 并且可变参数必须放在函数声明参数列表最后

那么如果将这个可变参数作为参数进行传递会怎么样 ？

会直接报错 cannot use args (variable of type []int) as int value in argument to add 由于 add 函数接受的可变参数是 int 类型，不是切片类型，所以 calc 函数在传递可变可变参数时，需要进行解操作，即对传递的参数进行添加 ... [ add(args...) ]

这样才会输出执行结果，当然不执行 解操作 也可以，只需要将 add 函数接受的可变参数类型修改为 ...[]int 即可

# init 函数

golang 有一个特殊的函数 init 函数 ，先于 main 函数 执行，实现包级别的一些初始化操作

主要特点

  1. init 函数先于 main 函数自动执行，不能被其他函数调用
  2. init 函数没有输入参数、返回值
  3. 每个包可以有多个 init 函数
  4. 包的每个源文件也可以有多个 init 函数（比较特殊）
  5. 同一个包的 init 执行顺序，golang 没有明确定义，编程时要注意程序不要依赖这个执行顺序
  6. 不同包的 init 函数按照包导入的依赖关系决定执行顺序

初始化顺序

变量初始化 -> init 函数 -> main 函数

# defer

Go 语言中 defer 语句会将其后面跟随的语句进行延迟处理（即后定义先执行）

defer 在注册的时候会将当前变量的状态进行保存【a:=1 => defer fmt.Println (a) => a=2 最终输出 a 的值为 1】

在 defer 归属的函数即将返回时，将延迟处理的语句按 defer 定义的逆序进行执行，也就是说，先被 defer 的语句最后执行，最后被 defer 的语句，最先执行

由于 defer 语句延迟调用的特性，所以 defer 语句能非常方便的处理资源释放问题【资源清理、文件关闭、解锁及记录时间等】

# defer 语句执时机

在 Go 语言函数中的 return 语句在底层并不是原子操作，分为给返回值赋值和 RET 指令两步，而 defer 语句执行的时机就在返回赋值操作后和 RET 执行前

# 闭包

闭包可以理解为定义在一个函数内部的函数，在本质上，闭包是将函数内部和函数外部连接起来的桥梁，或者说是函数和其引用环境的组合体

闭包指的是一个函数和与其相关的引用环境组合而成的实体，即 闭包 = 函数 + 引用环境

变量 f1 是一个函数并且它引用了其外部作用域中的 x 变量，此时 f1 就是一个闭包，在 f1 的声明周期内，变量 x 也一直有效

在 add1、sub1 的声明周期内，base 值会一直有效，同理 add2、sub2 一样

# 递归

函数内部调用函数自身的函数为递归函数

  1. 递归就是自己调用自己
  2. 必须先定义函数的退出条件，没有退出条件，就会称为死循环
  3. Go 语言递归函数很可能产生一大堆 goroutine ，也可能出现栈空间内存溢出问题
