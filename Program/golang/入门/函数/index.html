函数

Go语言中支持函数、匿名函数、闭包，并且函数在Go语言中属于”一等公民”

函数定义

Go语言中定义函数使用func关键字

func 函数名 (参数)(返回值) {
    函数体
}

函数名 : 由字母、数字、下划线组成，但函数名的第一个字母不能是数字，在同一个包内，函数名也不能重名

参数 : 参数由参数变量和参数变量的类型组成，多个参数之间使用,分隔

返回值 : 返回值由返回值变量和其变量类型组成，也可以只写返回值的类型，多个返回值必须使用()包裹，用,隔开

函数体 : 实现指定功能的代码块

package main

import "fmt"

// 函数定义
func sum(x int, y int) (ret int) {
    ret = x + y
    return ret
}
// 没有返回值的
func f1(x int, y int){
    fmt.Println(x +y)
}
// 没有参数没有返回值
func f2(){
    x := 10
    y := 20
    fmt.Println(x + y)
}
// 没有参数但有返回值的
func f3(ret int){
    x := 10
    y := 20
    return x + y
}
// 返回多个参数
func f4() (int x, int y){
    x = 1
    y = 1
    return x, y
}
// 参数类型的简写[当参数中连续两个参数的类型一致时，可以将前面哪个参数的类型省略]
func f5(x, y int) int { // x和y都是int类型
    return x + y
}
// 可变长参数[可变长参数必须放在函数的最后]
func f6(x string , y ...int) (){
    fmt.Println(x)
    fmt.Println(y) // y的类型时int类型的切片，y可以不传值，也可以传多个值
}

func main() {
    num := sum(10, 20)
    num1, num2 := f4(2, 3)
    fmt.Println(num)
    f6("hello") // 输出 hello []
    f6("hello", 1,2,3,4,5) // 输出 hello [1 2 3 4 5]是个切片类型
}

参数

  1. 接受单个参数
func f1(n1 int){
    ...
}
  1. 接受多个参数
func f1(n1, n2, n3 int, s1, s2, s3 string){
    ...
}
  1. 接受的参数不确定
func f1(args ...int){
    ...
}
func f1(args ...[]int){
    ...
}
  1. 函数作为参数
func f1(x func()){ // 这是接收一个没有返回值的函数
    ...
}

func f2(x func()int){ // 这是接收一个存在返回值的函数
    ...
}

返回值

  1. 返回单个参数
func f1() int {
    ...
}
  1. 返回多个参数
func f1() (int, float, bool) {
    ...
}
  1. 命名返回值（这样写的好处是，当写return时，可以直接写return，而不用再次return xx, xx, xx, ……）
func f1() (n1 int, f1 float, b1 bool) {
    ...
}
  1. 函数作为返回值
func f1(x func()int) func(x, y int)int{
    ret := func(a, b int)int{
        return a + b
    }
    return ret
}

func f2(x func()int) func(){
    ...
}

函数类型

可变参数函数

用变量 ...数据类型表示str ...string

  1. 可变参数，在一个方法中只能有一个
  2. 并且可变参数必须放在函数声明参数列表最后
package main

import "fmt"

func test(str ...string) {
    fmt.Printf("%T, %#v\n", str, str)
}

func main() {
    test() // []string, []string(nil)
    test("1") // []string, []string{"1"}
    test("1", "2", "3") // []string, []string{"1", "2", "3"}
}

那么如果将这个可变参数作为参数进行传递会怎么样 ？

package main

import "fmt"

func add(args ...int) int {
    nums := 0
    for _, v := range args {
        nums += v
    }
    return nums
}

func calc(n1, n2 int, args ...int) int {
    nums := n1 + n2
    nums += add(args)
    return nums
}

func main() {
    fmt.Println(calc(1, 2, 3, 4, 5, 6))
}

会直接报错cannot use args (variable of type []int) as int value in argument to add由于add函数接受的可变参数是int类型，不是切片类型，所以calc函数在传递可变可变参数时，需要进行解操作，即对传递的参数进行添加...[ add(args…) ]

package main

import "fmt"

func add(args ...int) int {
    nums := 0
    for _, v := range args {
        nums += v
    }
    return nums
}

func calc(n1, n2 int, args ...int) int {
    nums := n1 + n2
    nums += add(args...)
    return nums
}

func main() {
    fmt.Println(calc(1, 2, 3, 4, 5, 6)) // 21
}

这样才会输出执行结果，当然不执行解操作也可以，只需要将add函数接受的可变参数类型修改为...[]int即可

package main

import "fmt"

func add(args ...[]int) int {
    nums := 0
    for _, arg := range args {
        for _, v := range arg {
            nums += v
        }
    }
    return nums
}

func calc(n1, n2 int, args ...int) int {
    nums := n1 + n2
    nums += add(args)
    return nums
}

func main() {
    fmt.Println(calc(1, 2, 3, 4, 5, 6))
}

init函数

golang有一个特殊的函数 init函数 ，先于 main函数 执行，实现包级别的一些初始化操作

主要特点

  1. init函数先于main函数自动执行，不能被其他函数调用
  2. init函数没有输入参数、返回值
  3. 每个包可以有多个init函数
  4. 包的每个源文件也可以有多个init函数（比较特殊）
  5. 同一个包的init执行顺序，golang没有明确定义，编程时要注意程序不要依赖这个执行顺序
  6. 不同包的init函数按照包导入的依赖关系决定执行顺序

初始化顺序

变量初始化 -> init函数 -> main函数

package main

import "fmt"


var intVar int = initVar()

func init() {
    fmt.Println("init……")
}

func initVar() int {
    fmt.Println("initVar……")
    return 100
}

func main() {
    fmt.Println("main……")
}
/*
initVar……
init……
main……
*/

定义函数类型

我们可以使用type关键字来定义一个函数类型

type calculation func (int, int) int

上面语句定义了一个calculation类型，它是一种函数类型，这种函数类型可以接收两个int类型参数并返回一个int类型的返回值

简单来说就是凡是满足条件的函数都是calculation类型的函数

作用域

全局变量

定义在函数外部的变量，它在整个运行周期内有效，在函数中可以访问到全局变量

package main

import "fmt"

// 作用域
var (
    num int = 10
)

func printNum(x int) {
    fmt.Println(x)
}

func main() {
    printNum(num)
}

局部变量

分为两种

  1. 函数内定义的变量无法在该函数外使用
  2. 局部变量和全局变量重名，优先考虑局部变量
package main

import "fmt"

// 作用域
var (
    num int = 10
)

func printNum(x int) {
    num2 = 20
    fmt.Println(x)
}

func main() {
    fmt.Println(num2) // 报错
    printNum(num)
}

defer

Go语言中defer语句会将其后面跟随的语句进行延迟处理（即后定义先执行）

defer在注册的时候会将当前变量的状态进行保存【a:=1 => defer fmt.Println(a) => a=2 最终输出a的值为1】

在defer归属的函数即将返回时，将延迟处理的语句按defer定义的逆序进行执行，也就是说，先被defer的语句最后执行，最后被defer的语句，最先执行

package main

import "fmt"

// 函数

func main() {
    fmt.Println("start")
    defer fmt.Println(1)
    defer fmt.Println(2)
    defer fmt.Println(3)
    fmt.Println("end")
}

start
end
3
2
1

由于defer语句延迟调用的特性，所以defer语句能非常方便的处理资源释放问题【资源清理、文件关闭、解锁及记录时间等】

defer语句执时机

在Go语言函数中的return语句在底层并不是原子操作，分为给返回值赋值和RET指令两步，而defer语句执行的时机就在返回赋值操作后和RET执行前

package main

import "fmt"

// defer
func f1() int {
    x := 5
    defer func() {
        x++ // 修改的是x的值，而不是返回值的值
    }()
    return x // 返回5
}
func f2() (x int) {
    defer func() {
        x++ // 定义的是命名变量返回值，先返回值赋值[x=5]，在执行defer[x=6]，最后return返回，此时的返回值是x[x=6]
    }()
    return 5 // 返回x的值6
}
func f3() (y int) {
    x := 5
    defer func() {
        x++ // 先返回值赋值y=x=5，在执行defer[x=6]，最后return返回返回值y=5
    }()
    return x // 返回5
}
func f4() (x int) {
    defer func(x int) {
        x++ // 改变的是这个函数中的值
    }(x)
    return 5 // 返回值5
}

func main() {
    fmt.Println(f1())
    fmt.Println(f2())
    fmt.Println(f3())
    fmt.Println(f4())
}

闭包

闭包可以理解为定义在一个函数内部的函数，在本质上，闭包是将函数内部和函数外部连接起来的桥梁，或者说是函数和其引用环境的组合体

闭包指的是一个函数和与其相关的引用环境组合而成的实体，即 闭包 = 函数 + 引用环境

package main

import "fmt"

func add() func(int) int {
    var x int
    return func(y int) int {
        x += y
        return x
    }
}

func test7() {
    f1 := add()
    fmt.Printf("f1(10): %v\n", f1(10))
    fmt.Printf("f1(20): %v\n", f1(20))
    fmt.Printf("f1(30): %v\n", f1(30))
    f2 := add()
    fmt.Printf("f2(40): %v\n", f2(40))
    fmt.Printf("f2(50): %v\n", f2(50))

/*
f1(10): 10
f1(20): 30
f1(30): 60
----------
f2(40): 40
f2(50): 90
*/
}

变量 f1 是一个函数并且它引用了其外部作用域中的 x 变量，此时 f1 就是一个闭包，在 f1 的声明周期内，变量 x 也一直有效

package main

import "fmt"

func cal(base int) (func(int) int, func(int) int) {
    add := func(x int) int {
        base += x
        return base
    }
    sub := func(x int) int {
        base -= x
        return base
    }
    return add, sub
}

func main() {
    add1, sub1 := cal(100)
    fmt.Printf("add1(10): %v\n", add1(10))
    fmt.Printf("sub1(5): %v\n", sub1(7))
    add2, sub2 := cal(10)
    fmt.Printf("add2(5): %v\n", add2(5))
    fmt.Printf("sub2(2): %v\n", sub2(2))
/*
add1(10): 110
sub1(5): 103
add2(5): 15
sub2(2): 13
*/
}

在add1、sub1的声明周期内，base值会一直有效，同理add2、sub2一样

package main

import "fmt"

// 闭包
// 实现f1调用f2执行


func f1(f func()) {
    fmt.Println("this is f1")
    f()
}

func f2(x, y int) {
    fmt.Println(x + y)
    fmt.Println("this is f2")
}
func f3(x func(a, b int), y, z int) func() {
    tmp := func() {
        x(y, z)
    }
    return tmp
}

func main() {
    ret := f3(f2, 100, 200)
    ret() // 实现了功能
    f1(ret)
}

递归

函数内部调用函数自身的函数为递归函数

  1. 递归就是自己调用自己
  2. 必须先定义函数的退出条件，没有退出条件，就会称为死循环
  3. Go语言递归函数很可能产生一大堆 goroutine ，也可能出现栈空间内存溢出问题

阶乘

package main

import "fmt"

func jc(n int) int {
    if n == 1 {
        return 1
    }
    return n * jc(n-1)
}

func main() {
    fmt.Printf("jc(5): %v\n", jc(5)) // jc(5): 120
}

斐波那契数列

package main

import "fmt"

func fbnq(n int) int {
    if n == 1 || n == 2 {
        return 1
    }
    return fbnq(n-1) + fbnq(n-2)
}

func main() {
    fmt.Printf("fbnq(10): %v\n", fbnq(10)) // fbnq(10): 55
}

汉诺塔

package main

import "fmt"

func tower(start, temp, end string, len int) {
    if len == 1 {
        fmt.Println(start, "->", end)
        return
    }
    tower(start, end, temp, len-1)
    fmt.Println(start, "->", end)
    tower(temp, start, end, len-1)
}

func main() {
    tower("塔1", "塔2", "塔3", 3)
}
/*
塔1 -> 塔3
塔1 -> 塔2
塔3 -> 塔2
塔1 -> 塔3
塔2 -> 塔1
塔2 -> 塔3
塔1 -> 塔3

*/
