在 Go 编程中，数据类型用于声明函数和变量，数据类型的出现是为了把数据分成所需内存大小不同的数据，编程的时候需要用大数据的时候才申请大内存，就可以充分利用内存

# 值类型

值类型包括 : int 、 float 、 指针 、 数组 、 结构体

当 n1 将值赋给 n2 时，n2 将值改变，那么 n1 的值会不会也发生改变 ？ 不会

具体过程 ：值拷贝，在内存中申请新的内存空间，将 n1 的值重新赋给 n2，两者的内存地址是不一样的

# 引用类型

引用类型包括 : 切片 、 映射 、 接口

# 整形

整形分为两大类，按长度分为 : <font color="#00b050">int8（又称 byte）、int16、int32（又称 rune）、int64</font > 对应的无符号整形 : <font color="#00b050">uint8、uint16、uint32、uint64</font>

其中 uint8 就是我们熟知的 byte 型，int16 对应 C 语言的 short 型，int64 对应 C 语言的 long 型

类型描述
uint8无符号 8 位整形 [0~255]
uint16无符号 16 位整形 [0~65535]
uint32无符号 32 位整形 [0~4294967295]
uint64无符号 64 位整形 [0~18446744073709551615]
int8有符号 8 位整形 [-128~127]
int16有符号 16 位整形 [-32768~32767]
int32有符号 32 位整形 [-2147483648~2147483647]
int64有符号 64 位整形 [-9223372036854775808~9223372036854775807]

# byte 和 rune 类型

组成每个字符串的元素叫做 "字符"，可以通过遍历或者单个获取字符串元素获取字符

字符用单引号包裹

Go 语言的字符有以下两种

uint8 类型，或叫做 byte 型，代表了 ASCII 的一个字符

rune 类型，代表了一个 UTF-8 字符

当处理中文、日文或者其他符合字符时，则需要用到 rune 类型，rune 类型实际是一个 int32

Go 使用了特殊的 rune 类型来处理 Unicode，让基于 Unicode 的文本处理更为方便，也可以使用 byte 型进行默认字符串的处理

# 指针类型

Go 语言中不存在指针操作，只记住两个符号

  1. & : 取地址
  2. * : 根据地址取值

取地址操作符 & 和取值操作符 * 是一对互补操作符，& 去除地址，* 根据地址取出地址指向的值

  • 变量、指针地址、指针变量、取地址、取值的关系
    • 对变量进行取地址操作 [&] 可以获取这个变量的指针变量
    • 指针变量的值是指针地址
    • 对指针变量进行取值操作 [*] 可以获取指针变量指向的原变量的值

# new

new 是一个内置的函数

  • <font color="#00b050">Type</font > 表示类型，new 函数只接收一个参数，这个参数是一个类型
  • <font color="#00b050">*Type</font > 表示类型指针，new 函数返回一个指向该类型内存的指针

new 函数不常用，使用 new 函数得到的一个类型的指针，并且该指针对应的值为该类型的零值

指针作为引用类型需要初始化后才能拥有内存空间，才可以给它赋值，使用内置的 new 函数就可以对 b 进行初始化后可以正常赋值

# make

make 也是用于内存分配的，区别于 new，它只用于 slice、map 以及 chan 的内存创建，而且它返回的类型就是这三个类型本身，而不是他们的指针类型，因为这三种类型就是引用类型，所以就没有必要返回它们的指针

make 函数的函数格式

make 函数是无可替代的，我们在使用 slice、map 以及 channel 的时候，都需要使用 make 进行初始化，然后才可以对其键值进行赋值

# make 和 new 的区别

  1. make 和 new 都是用来分配内存的
  2. new 很少用，一般用来给 基本数据类型分配内存，string/int。。。。，返回的是对应类型的指针【<font color="#00b050">*string、*int</font>】
  3. make 是用来给 slice、map、channel 分配内存的，返回的是对应的这三个类型的本身

# 没有 max

当利用切片或数组造出另一个切片时，如果没有指定其 max 值，那么默认就会以原先的切片或数组的 cap（容量）来进行计算

如上程序中，当 a1 利用切片构造出另一个切片后，想要给新的切片 追加数据 时，那么原先的切片或数组也会发生改变，这是因为在构造切片时，没有指定其 max 值，而是直接使用原先的 cap - 起始位置 即上程序中的 a1 的 cap 不是 2，而是 5，当追加数据时，发现长度不够（长度为 2），但是容量够，所以便直接将数据追加在新的切片后面，而这一追加便将原先的数据发生了改变

# 有 max

而添加 max 值后，再次查看 cap，就会发现容量不够，而会生成一个新的数组，将原先的数据进行复制，在追加，从而不会影响原先的数据，这时，改变新切片的值将不会对原先的数组或切片产生任何影响

# 注意事项

切片是不能直接比较的

不能使用 == 操作符来判断两个切片是否含有全部相等元素

切片唯一合法的比较操作符是和 nil 比较，一个 nil 值的切片并没有底层数组，一个 nil 值的切片的长度和容量都是 0，但是我们不能说一个长度和容量都是 0 的切片一定是 nil

因此，判断一个切片是否为空，要使用 <font color="#00b050">len (s) == 0</font > 来判断，不能使用 < font color="#00b050">s == null</font > 判断
