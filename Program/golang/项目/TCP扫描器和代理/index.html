# io.Reader&io.Writer

无论使用的是 TCP、HTTP、文件系统还是任何其他方式，都需要使用两种类型 io.Reader 和 io.Writer ，对于所有输入 / 输出（I/O）任务来说，这两种数据类型都是必不可少的，它们都是 Go 内置 io 包的一部分，是所有本地或网络数据传输的基础

这两种类型都被定义为接口类型，意味着不能直接被实例化，每种数据类型包含单个导出函数的定义，分别是 Read、Write，将这种函数理解为抽象方法，该抽象方法必须在某种数据类型上得到实现才能被视为 Reader 或 Writer

实现 io.Reader&io.Reader

该程序定义了两种自定义类型 ：FooReader、FooWriter，在每种类型上，为 FooReader 定义了函数 Read (b [] byte)，为 FooWriter 定义了 Write (b [] byte)，在这里两个函数都从 stdin 读取并写入 stdout

# TCP 代理服务器

学习如何在套接字中读写数据，需要用到 net 包中的 Conn 接口（Go 的面向流的网络连接）

Conn 实现了针对接口 Reader 和 Writer 定义的函数 Read 和 Write，所以可以既是 Reader 又可以是 Writer，因为 TCP 连接是双向的，因此可以用来写入或发送数据

创建 Conn 实例后，可以通过 TCP 套接字发送和接收数据，但是 TCP 服务器不能简单地创建一个连接，该连接必须是客户端发起建立

在 Go 中，首先可以使用 net 包中的 Listen 函数，在特定的端口上打开 TCP 监听器

客户端连接后，net 包中的 Listen 函数返回一个 Listener 接口，其中 Accept 函数将创建并返回一个 Conn 对象，可以使用该对象收发数据

# 回显服务器

  1. 首先定义了一个名为 echo (net.Conn) 的函数，该函数接收 Conn 实例作为参数，充当执行所有必要的 I/O 连接处理程序，使用缓冲从连接读取数据以及向连接写入数据进行无线循环，数据会被读入一个 b 的变量中，在返回该连接
  2. 主函数设置一个监听器，调用处理函数（服务器无法建立连接，只能监听），net,Listen 在所有接口上绑定 TCP 端口 10086，返回 listener 和 err
  3. 接下来无线循环确保即使已经收到一个连接服务器仍将继续监听新的连接请求，在循环中，嗲用 listener.Accept 接收客户端连接，成功后会返回 Conn 实例对象
  4. 之后将 Conn 实例传递给处理函数 echo (net.Conn)，以 go 开头，实现并发，以便在等待处理函数完成时其他连接不会被阻塞

# NC 复现

使用 PipeReader 和 PipeWriter 可以避免显示刷新 writer 并同步连接 stdout 和 TCP 连接

调用 io.Pipe () 会同时创建同步连接的一个 reader 和一个 writer，任何被写入 writer 的数据都会被 reader 读取，因此，需要将 writer 分配给 cmd.Stdout ()，然后使用 io.Copy () 将 PipeReader 连接到 TCP 连接，使用 go 来防止代码阻塞，命令的任何标准输出豆浆发送到 writer，通过管道传到 reader 并通过 TCP 连接输出
