# 无密码认证

# 二层网络

同样在 用户认证与管理 => 认证策略 中配置

选择 不需要认证 用户名可以选择 以 IP 或者以 MAC

这里设置自动录入绑定关系 （ 即第一个上线的 IP 和对应电脑的 MAC 地址进行绑定 ）

之后在访问百度就会出现对应的 IP 绑定 MAC

# 三层网络（跨三层取 MAC）

先配置一个无密码认证策略

之后在在线用户管理查看 PC 显示的 MAC 地址

登录 AF 查看其 MAC 地址

再次回到 AC，配置跨三层取 MAC 用户认证与管理 => 认证高级选项 => 跨三层取 MAC => 启用跨三层 MAC 识别 => 新增

下拉把 AF 对应的 MAC 地址去掉

网络 => 接口 / 区域 => 区域中选择互联网区，勾选 SNMP

之后在去 AF 配置 SNMP

配置完成后回到 AC 查看其 MAC 地址

# 本地密码认证

查看本机用户

用户认证与管理 => 用户认证 => 认证策略 【配置完成后记得在认证高级选项中勾选 https】

用户认证与管理 => 用户管理 => 组 / 用户

先增加一个组，组内存放需要认证的机器

点击 <mark style="background: #FF5582A6;"> 办公组 </mark> 进入添加用户 【账号密码都是 test1】

可以勾选 初次认证修改密码

之后在认证高级选项 => 认证选项 勾选 https 重定向

点击提交

再次访问百度就会跳转到认证页面

输入 test1 /test1

修改完成后再次输入账号密码认证

可以访问百度了

# 外部密码认证

使用 AD 域服务器来进行认证

  1. 登录 AD 域服务器，配置用户

  1. 点击 AD 用户和计算机，在 cit.sangfor.com 中新建用户组织

之后在新建的组中创建用户

用户名为 test

密码为 123.com

  1. AC 上创建认证服务器

  1. 配置认证服务器

[!ob-info] 认证服务器配置
方法一 ：

需要登录到 AD 找到创建的对应用户，右键属性中找到属性编辑器中的 distinguishedName 的值，密码是刚刚创建用户时设置的密码

方法二 ：

直接使用登录用户名 test@cit.sangfor.com 密码 123.com

或者使用管理员 administrator@cit.sangfor.com 密码 sangfor@123

方法一

  • 使用管理员账号

  • 使用创建的用户

方法二

  • 使用管理员账号

  • 使用创建的用户

<span style="background:#b1ffff"> 注意这里配置完成后查看这里 </span>

点进来之后选择新建的组织单位

如果不选的话，提交后在 用户 / 组 中查看会显示其他组织的信息

选择完后查看就会只显示当前创建的组织中的账号信息

配置完成后点击提交，服务器配置完成后，配置策略

再此之前可以先创建一个用户组

之后的非本地 / 域用户登录就会以员工组的身份登录

这里创建一个新的策略（不做任何认证，只做认证后处理），用来验证这个认证后处理是否可行

测试完成后把该策略删掉，继续执行外部密码认证

测试访问百度

这是认证失败后的界面显示

认证成功
