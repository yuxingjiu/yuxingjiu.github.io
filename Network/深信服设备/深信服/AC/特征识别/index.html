# 传统检测

传统的网络设备根据数据包的五元组（源目 IP、源目端口、协议）这些特征等来识别应用并进行操作

# 深度检测

深度行为检测技术又分为两种 深度包检测（DPI） 和 深度流检测（DFI）

特点

  1. 可视化全网
  2. 流量精细化管理
  3. 减少或延迟带宽投入，降低网络运营成本
  4. 即使发现和抑制异常流量
  5. 透视全网服务质量，保障关键业务质量
  6. 丰富的 QoS 提供能力

# 深度包检测

深度包检测不仅检测五元组，还增加了应用层分析，另外识别各种应用及其内容

# 基于特征字

  1. 不同的应用通常依赖于不同的协议，而不同的协议都由其特殊的特征，这些特征可能是特定的端口、特定的字符串或者特定的 Bit 序列
  2. 对业务流中特定数据报文中的特征信息的检测以确认业务流承载的应用和内容
  3. 通过对应用特征信息的审计（ http 数据包中的 UA 字段位置 ）基于特征的识别技术可以很方便的进行功能扩展，实现对新协议的检测

# 基于应用网关

  1. 需要先识别出控制流，根据对应的协议，对控制流进行解析，从协议内容中识别处响应的业务流
  2. 某些应用的控制流和数据流是分离的，数据流没有任何特征

# 基于行为模式

  1. 对终端已经实施地行为进行分析，判断出用户正在进行的动作或者即将实施地动作
  2. 行为模式识别技术通常用于无法根据协议判断的业务识别（垃圾邮件）

# 深度流检测

基于流量行为的应用识别技术，即不同的应用类型体现在会话连接或数据流上的状态等各有不相，基于流的行为特征，建立流量特征模型，与已建立的应用数据流的数据模型对比，分析会话连接流的包长、连接速率、传输字节量、包与包之间的间隔等信息来与流量模型对比，从而实现鉴别应用类型

# http/http 控制

# http 封堵

  1. 获取 http 三次握手后的 GET 请求包中的 host 字段来识别服务器
  2. 发送重定向包，在发送 RST

# https 封堵

  1. https 在四次握手的第一阶段，获取客户机发送的 Client hello 包中的 Servername 字段来识别服务器
  2. 直接发送 RST 结束 TCP 连接，不会发送重定向包

# 自定义对象

AC 内置规则库已经覆盖常见的应用和网站，并以半个月为周期的持续更新，但难免有部分不常见的应用和网站没有更新，这种情况，只要我们提供应用的特征或 url 就可以通过自定义进行识别和控制

对象定义 => URL 分类库 可以自定义 URL（支持匹配 URL 和域名关键字）自行定义要匹配的网站或 IP
