部署方式

发表于 2023-07-26
  • 余星酒
首页 网络 深信服设备 AC

部署模式是指设备以什么样的工作方式部署到客户网络中，不同的部署模式对客户原有网络的影响各不同，设备在不同模式下支持的功能也各不一样

  1. AC设备支持路由、网桥、旁路部署
  2. SG设备支持路由、网桥、旁路、单臂部署
  3. 12.0之后支持认证部署模式
  4. 13.0版本之后认证模式功能以认证口的形式融入到普通模式中

3种模式

  1. 路由模式可以实现设备所有功能，网桥其次，旁路模式多用于审计，只能对TCP应用控制，控制功能最弱
  2. 路由模式对客户原有网络改造影响最大，网桥模式其次，旁路模式对客户原有网络改造无影响，即使设备宕机也不影响原有网络
  3. 设备路由模式最多支持8条外网线路，网桥模式最多支持8对网桥，旁路模式除了管理口外，其他均可作为监听口，可以同时选择多个网口作为监听口

路由部署

设备以路由模式部署时，AC的工作方式与路由器相当，具备基本的路由转发及NAT功能，一般在客户还没有响应的网关设备或用户的网络环境规模比较小，需要将AC做网关使用时，推荐使用路由部署 [路由模式下支持AC所有的功能]

如果需要使用NAT、VPN、DHCP等功能时，AC必须以路由模式部署，其他工作模式没有这些功能

配置思路

  1. 网口配置，配置各网口地址，如果是固定IP，则填写运营商给的IP地址及网关，如果是ADSL拨号上网，则填写运营商给的拨号账号和密码，确定内网口的IP
  2. 确定内网是否为多网段网络环境，如果是的话需要添加相应的会暴露有，将到内网个网段的数据回指给设备下接的三层设备
  3. 用户是否需要通过AC设备上网，如果是的话，需要配置NAT规则
  4. 检查并放行防火墙规则
  5. 点提交

网桥部署

设备以网桥模式部署时对客户原有的网络基本没有改动，网络模式部署AC时，对客户来说AC就是个透明的设备，如果因为AC自身的原因导致网络中断时可以开启硬件bypass功能恢复网络通信 [网桥模式部署时AC不支持NAT(代理上网和端口映射)、VPN、DHCP等]

配置思路

  1. 配置设备网桥地址、网关、DNS
  2. 确定内网是否为多网段网络环境，这里是三层环境，需要添加相应的回包路由，将到内网个网段的数据回指给设备下接的三层设备
  3. 检查并方通防火墙规则
  4. 提交

旁路部署

旁路模式主要用于实现行为审计功能，不需要改变用户的网络环境，通过把设备的监听口接在交换机的镜像口上同时镜像交换机上下行的数据，从而实现对上网数据的监控与控制，这种模式对用户的网络环境没有影响，即使设备本身宕机也不会对用户的网络造成中断 [只能对TCP应于做控制，对UDP的应用无法控制，也不支持流量管理、NAT、VPN、DHCP等]

旁路模式部署还可用于旁路重定向认证，再不改变网络原有架构的情况下对入网用户进行认证 （旁路portal重定向认证+审计场景、旁路802.1x认证+基线核查+审计场景）

配置思路

  1. 交换机设置镜像口，并接到AC监听口
  2. 配置需要审计的内网网段和服务器网段
  3. 配置管理口地址，用于管理AC设备
  4. 提交
