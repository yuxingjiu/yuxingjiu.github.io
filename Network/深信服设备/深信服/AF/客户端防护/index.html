# 病毒的工作步骤

  1. 潜伏阶段

休眠状态，直到特定的程序被执行

  1. 传染阶段

将自身程序赋复制给其他程序或磁盘

  1. 触发阶段

执行特定功能达到特定的功能

  1. 发作阶段

破坏程序感染文件

# 网关杀毒实现

# 代理扫描

将所有经过网关的需要进行病毒检测的数据报文透明的转角给网关自身的协议栈，通过网关自身的协议栈将文件全部缓存下来后，在送入病毒检测引擎进行病毒检测

# 流扫描方式

依赖于状态检测技术以及协议解析技术，简单的提取文件的特征与本地签名进行匹配

# AF 网关杀毒

  1. 提取（描述信息 + 文件缓存路径）传递
  2. 人工智能检测引擎 SAVE（本地病毒库匹配）
  3. 查杀失败时上传云脑，上传 MD5 值
  4. 文件 MD5 云查（安全云脑）
  5. 文件云查（安全云脑）
  6. 反馈结果，本地 SAVE 引擎学习
  7. 等待检测结果
  8. 根据返回的检测结果进行操作，记录日志

# 勒索防护

  1. 事前加固 ：勒索病毒风险评估，精准评估勒索病毒进入点缝隙，配置勒索病毒专项策略，全面防护勒索风险
  2. 事中积极防御 ：通过配置的勒索专项策略，全面防护勒索风险
  3. 事后快速响应与处置 ：隔离识别已失陷的主机，专项工具进行杀毒

# 僵尸网络

僵尸网络是指骇客利用自己编写的分布式拒绝攻击程序将数万个机器沦陷，即僵尸电脑或肉鸡组织成一个个控制节点来发送伪造包或垃圾数据包，使预定攻击目标瘫痪（蠕虫也可以说是被利用组成僵尸网络）

# 异常流量检测

通过对当前的网络层及应用层行为与安全模型进行偏离角度分析，能够发现隐藏的网络异常行为，并根据行为特征确定攻击类型，发现特征匹配无法发现的攻击

外发流量异常功能是一种启发式的 DOS 攻击检测手段，能够检测源 IP 不变的各种 DOS 攻击

当特定协议的外发包 PPS 超过设置的阈值时，基于 5 分钟左右的抓包样本检测数据包是否为单向流量、是否正常响应内容，得出结论，将发现的攻击提交日志显示

# 其他检测

  1. 与僵尸网络连接是最基础的判定方式，信息来源包括：上万台在线设备收集、与 google 等机构合作共享
  2. 对于未知的僵尸网络 (存在大量 DGA 生成的 c&c 域名)，通过模拟 DGA 算法总结特征 / 总结一般正常域名的构成方式来判定未知的僵尸网络
  3. 危险的外联方式检测，如使用已知的（IRC、HFS) 与僵尸网络进行通讯
  4. 使用标准端口传输非标准协议 (如：在 80 端口中传输 RDP 协议)
  5. 对外发起 cc 攻击
  6. 对外传播恶意文件
  7. 对外发送 shellcode
  8. 检测出下载恶意文件、恶意 PDF 等行为
  9. 检测出下载文件与后缀名不符
  10. 上下行流量不符

# AF 防御

利用规则库，超 100w 条僵尸网络规则

# 误判排除

  1. 发现某个终端流量被 AF 僵尸网络规则误判，可以在僵尸网络功能模块下排除指定 IP
  2. 发现某个规则引起的误判拦截所有内网终端流量，可以在 安全防护对象 => 僵尸网络规则库 找到指定规则禁用，所有僵尸网络策略都不对此规则做任何拦截
  3. 也可以直接在内置数据中心中，查询僵尸网络日志后使用 例外添加 排除
