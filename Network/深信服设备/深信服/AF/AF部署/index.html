AF部署

发表于 2023-07-28
  • 余星酒

接口

AF的部署模式是由各个接口的属性决定的

根据接口属性

  1. 物理接口
  2. 子接口
  3. VLAN接口
  4. 聚合接口

物理口

  1. 路由口
  2. 透明口
  3. 虚拟网线口
  4. 旁路镜像口

接口不同工作层面

  1. 二层区域（透明口、旁路口）

  2. 三层区域

  3. 虚拟网线区域

  4. 设备支持配置多个WAN属性的路由接口连接多条外网线路，但是需要开通多条线路的授权

  5. 管理口不支持设置成透明模式或虚拟网线接口，如果2对或2对以上的虚拟网线接口，则必须要求设备不少于5个物理接口，预留一个专门的管理口Eth0

  6. 一个路由接口下可添加多个子接口，路由接口的IP地址不能与子接口的IP地址在同网段

物理接口

物理接口与AF设备面板上的接口一一对应（eth0为manage口）根据网口数据转发特性的不同，可以选择路由、透明、虚拟网线（这三种接口可以设置为WAN口也可以是非WAN口）和旁路镜像四种类型

物理接口无法删除或新增，物理接口的数目由硬件决定（个别平台支持扩展）

路由接口

路由口

如果设置为路由接口，则需要给该接口配置IP地址，且该接口具有路由转发功能

管理口

Eth0位固定的管理口，接口类型为路由口，无法修改，Eth0可以增加管理IP地址，默认的管理IP为10.251.251.251/24，如果需要修改，则在AF8.0.13版本后

通过 系统 ⇒ 系统设置 ⇒ 通用配置 ⇒ 网络参数 中修改

透明接口

透明口相当于普通的交换网口，不需要配置IP地址，不支持路由转发，根据MAC地址表转发数据

虚拟网线接口

  1. 虚拟网线接口也是普通的交换接口，不能配置IP地址，不支持路由转发，转发数据时也无需检查MAC地址表，直接从虚拟网线配对的接口转发
  2. 虚拟网线接口的转发性能高于透明接口，单进单出，双进双出等成对出现的网桥的环境下，推荐使用虚拟网线接口部署

旁路镜像口

  1. 旁路镜像口不能配置IP地址，也不支持数据转发，只是用来接收从外部镜像过来的镜像数据
  2. 镜像接口可以配置多个，可以根据现场实际业务场景需要接收的数据情况进行选择

聚合接口

将多个以太网接口捆绑在一起形成的逻辑接口，创建的聚合接口成为一个逻辑接口，而不是底层的物理接口

  1. 接口名称 ：最多支持4个

  2. 类型 ：不支持旁路镜像

  3. 工作模式

  4. 负载均衡-hash ：按数据包源目IP/MAC的hash值均分

  5. 负载分担 -RR ：直接按数据包轮转均分到每个接口

  6. 主备模式 ：取Eth最大号为主接收口收发包，其余为备接口

子接口

  1. 子接口应用于路由解耦需要启用VLAN或Trunk的场景
  2. 子接口是逻辑接口，只能在路由口下添加子接口
  3. 子接口的下一跳网关和链路故障检测根据实际环境进行配置

vlan接口

为vlan定义IP地址，则会产生vlan接口，vlan接口也是逻辑接口

区域

区域分类

  1. 二层区域
  2. 三层区域
  3. 虚拟网线区域

区域作用

  1. 用于定义和归类接口，以供各类安全策略等模块调用
  2. 是本地逻辑安全区域的概念
  3. 一个或多个接口所连接的网络

定义区域时，需要根据控制的需求来规划，可以将一个接口划分到一个区域，或将几个相同需求的接口划分到同一区域

一个接口只能属于一个区域，而一个区域可以有多个接口

可以在区域中选择接口，也可以预先设置好区域名称，在接口中选择区域

部署

路由模式组网

  1. 路由组网方式，防火墙位于内部网络和外部网络之间，负责在内部网络、外部网络中进行路由寻址，相当于路由器，其余内部网络、外部网络相连的上下型业务接口均工作在三层 ，需要分别配置不同段的IP地址
  2. 此组网方式支持更多的安全特性 ：NAT、策略路由选择、动态路由协议等
  3. 需要修改原网络拓扑，对现有环境改动较大
  4. 一般部署在需要进行路由转发的位置（出口路由器、替换已有的路由器或防火墙等）
  5. 使用AF替换现有的防火墙部署在出口，实现对内网用户和服务器的安全防护

单臂路由

单臂路由是指路由器的一个接口上通过配置子接口（逻辑接口，并不存在的真正物理接口）的方式，实现原来相互隔离的不同vlan（虚拟局域网）之间的互联互通

透明模式组网

部署一台AF设备进行安全防护，但是又不改动现有的网络环境

虚拟网线部署

  1. 路由器和交换机聚合口对接，要求透明部署防火墙设备，即从1、3号口进来的数据，只能从2、4号口出去，不在转发到其他接口，同时防火墙可以被日常管理
  2. 由于该需求是不考虑MAC表直接以对应关系进行转发，选择虚拟网线接口

混合模式组网

  1. 用户内网有服务器群，服务器均配置公网IP地址，提供所有用户直接通过公网IP地址接入访问
  2. 内网用户使用私有地址，通过NAT转换代理上网，希望将AF部署在公网出口上，实现内外部数据通信的同时保护内部服务

AF设备连接公网和服务器群的2个接口使用透明access口，连接内网网段使用路由接口

旁路模式组网

  1. 设备旁挂在现有的网络设备上，不影响现有的网络结构，通过端口镜像技术把流量镜像到AF上，实现对数据的分析和处理

  2. 需要另外单独设置管理接口才能对设备进行管理

  3. 启用管理口reset功能

  4. 旁路模式支持的功能仅有

  5. APT（僵尸网络）

  6. PVS（实时漏洞分析）

  7. WAF（web应用防护）

  8. 入侵防护系统

  9. DLP（数据泄密防护）

  10. 网站防篡改部分功能（客户端保护）

使用AF来实现对各区域之间数据交互进行安全分析，同时不改动现有的环境

策略路由

  1. 策略路由是路由的一种
  2. 静态路由的匹配条件相对较少 ：匹配目的IP、子网掩码与下一跳网关
  3. 策略路由可以弥补静态路由的不足，更灵活多样的匹配条件，根据相应策略来进行匹配，包括 ：匹配源、目的、协议、应用等，出口在外网多条线路情况下，建议配置策略路由
  4. 源地址策略路由 —— 指定内网哪些IP走指定线路出公网
  5. 多线路负载路由 —— 指定多条线路进行负载选路
