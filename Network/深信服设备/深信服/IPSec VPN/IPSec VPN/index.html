# IPSec 简介

IPSec ：是一组基于网络层的，应用密码学的安全通信协议族，IPSec 不是具体指哪个协议，而是一个开放的协议族

IPSec VPN ：是基于 IPSec 协议族构建的在 IP 层实现的安全虚拟专用网，通过在数据包中插入一个预定义头部的方式来保障 OSI 上层协议数据的安全，主要用于保护 TCP、UDP、ICMP 和隧道的 IP 数据包

IPSec 协议族

# IPSec 工作模式

# 传输模式

应用场景 ：经常用于主机和主机之间端到端通信的数据保护

封装方式 ：不改变原有的 IP 包头，在原数据包头后面插入 IPSec 包头，将原有的数据封装成被保护的数据

# 隧道模式

应用场景 ：经常用于私网与私网之间通过公网进行通信，建立安全 VPN 通道

封装模式 ：增加新的 IP（外网 IP）头，然后是 IPSec 包头，之后在将原来的整个数据包封装

# IPSec 通信协议

# AH

AH ：认证报头

提供的安全服务

  1. 无连接数据完整性（ 通过哈希函数产生的校验来保证 ）
  2. 数据源认证 （ 通过在计算验证码时加入一个共享秘钥来实现 ）
  3. 抗重放服务 （ AH 报头中的序列号可以防止重放攻击 ）

不提供任何保密性服务

它不加密所保护的数据包

不论是传输模式还是隧道模式下，AH 提供对数据包的保护时，它保护的是整个 IP 数据包（易变的字段除外，如 IP 头部的 TTL 和 TOS 字段）

# ESP

ESP ：封装安全有效载荷

ESP 提供的安全服务

  1. 无连接数据完整性
  2. 数据源认证
  3. 抗重放服务
  4. 数据保密
  5. 有限的数据流保护

保密服务通过使用密码算法加密 IP 数据包的相关部分来实现

数据流保密有隧道模式下的保密服务提供

ESP 通常使用 DES、3DES、AES 等加密算法实现数据加密，使用 MD5 或 SHA1 来实现数据完整性认证

# IPSec 建立阶段

  1. SA ：通信对等体间对某些要素的约定，通信的双方符合 SA 约定的内容

SA 由三元组来唯一标识 => 安全参数索引、目的 IP 地址、安全协议号

  1. 用 IPSec 保护一个 IP 包之前，必须先建立安全联盟（SA）
  2. IPSec 的安全联盟可以通过手工配置的方式建立，但是当网络中节点较多时，手工配置将非常困难，而且难以保证安全性，这就可以使用 IKE 自动进行安全联盟建立与秘钥交换的过程，而 Internet 秘钥交换（IKE）就用于动态建立 SA，代表 IPSec 对 SA 进行协商

# IKE

IKE 为 IPSec 协商生成秘钥，供 AH/ESP 加解密和验证使用

# 工作过程

# 第一阶段

通信各方彼此间建立了一个已通过身份验证和安全保护的通道，此阶段的交换建立了一个 ISAKMP 安全联盟，即 ISAKMP SA（IKE SA），而第一阶段有两种协商模式 主模式 和 野蛮模式

# 主模式

主模式下 IKEv1 采用 3 个步骤 6 条 ISAKMP 消息建立 IKE SA

  1. 第一个消息由隧道的发起者发起，携带了一些参数（加密机制 - DES、散列机制 - MD5-HMAC、Diffie-Hellman 组 - 2、认证机制 - 预共享等）
  2. 应答方对发送方信息的应答，当应答方查找 SPD 查找到发送方相关的策略后，将自己的信息同样发送给对端，当然，应答方在发送传输集时将会生成自己 Cookie 并添加到数据包中
  3. 当完成了第一步骤双方的策略协商后，则开始进行第二步骤 DH 公共值交换，随数据包发送的还包含辅助随机数，用户生成双方的加密秘钥
  4. 应答方同样将本端的 DH 公共值和 Nonce 随机数发送给对端
  5. 发起者向响应者发送，主要是为了验证对端就是自己想要与之通信的对端，这样就可以通过预共享、数字签名、加密临时值来实现，双方交换 DH 公共值后，结合随机数生成一系列的加密秘钥，用户双方加密、校验，同时生成秘钥后，将公共秘钥和本端神扥信息等进行 hash，hash 值传输给对端进行验证设备身份
  6. 最后由响应者向发起者发送，主要目的和前面一样

在六条消息过后，一旦验证通过，就进入了 IKE 第二阶段 ：快速模式

# 野蛮模式

野蛮模式同样包含三个步骤，但仅通过三个包进行传输

  1. 发起者发送数据包，其包含了 SA 载荷（策略协商信息。秘钥交换载荷、随机数载荷、身份验证载荷等）
  2. 应答方收到发起者发送的消息后，通过自身查看 SPD 是否存在与发起方身份匹配的相关策略，若存在，则利用发起者发送的信息与自身配置进行计算，生成身份验证 hash 值后，将自身配置策略信息和 hash 值传给发起者
  3. 发起者接收到应答方的策略信息和 hash 值后，同样进行验证，若匹配，则将自身的 hash 值用计算出的秘钥加密后，传输给应答方
主模式野蛮模式
消息交换6 个消息3 个消息
身份 ID以 IP 地址作为身份 ID，自动生成本端身份 ID 和对端身份 ID可以以多种形式（IP、字符串等）手动或自动的生成本端和对端的身份 ID
域共享秘钥只能基于 IP 地址来确定基于 ID 信息（主机名和 IP 地址）
安全性较高 <br><br > 前 4 个消息以明文传输，最后两个消息加密，对对端身份进行了保护较低 <br><br > 前两个消息以明文传输，最后一个消息进行加密，不保护对端身份
速度较慢较快

# 第二阶段

用已经建立的安全联盟（IKE SA）为 IPSec 协商安全服务，为 IPSec 协商具体的安全联盟，建立 IPSec SA，产生真正可以用来加密数据流的秘钥，IPSec SA 用于最终的 IP 数据安全传输

ISAKMP/IKE 阶段 2 只有一种信息交换模式 —— 快速模式，定义了受保护数据连接时如何在两个 IPSec 对等体之间构成的，深信服的标准 IPSec 有几对的出站入站，在 DLAN 运行状态里面就会显示几条连接，每一对快速模式交互的包都是三个，第一个包由主链接发起方发起响应

快速模式的两个主要功能

  1. 协商安全参数来保护数据连接
  2. 周期性的对数据连接更新秘钥信息

数据传输阶段是通过 AH 或 ESP 通信协议进行数据的传输，建立在网络层

而如果建立隧道后，如果其中一段的设备异常重启，导致 SA 不一致，会发生什么 ？

VPN 黑洞

对端的 VPN 连接已经断开，而我方还处在 SA 有效生存期内，从而形成了 VPN 隧道黑洞

DPD 解决 VPN 隧道黑洞

DPD ：死亡对等体检测，检查对端的 ISAKMP SA 是否存在，当 VPN 隧道异常时，能检测到并重新发起协商，来维持 VPN 隧道

DPD 只对第一阶段生效，主要是为了防止标准 IPSec 出现隧道黑洞

DPD 包并不是连续发送，而是采用空闲计时器机制，每接收到一个 IPSec 加密的包就重置这个包对应的空闲定时器，而检测主要靠超时计时器，判断是否再次发起请求，默认是发出 5 次请求（请求 -> 超时 -> 请求 -> 超时 -> 请求 -> 超时）都没有收到任何 DPD 应答就会删除 SA
