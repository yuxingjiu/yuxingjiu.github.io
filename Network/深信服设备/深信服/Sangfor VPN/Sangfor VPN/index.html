Sangfor VPN

深信服设备自身互联两种 VPN 类型，IPSec VPN 和 Sangfor VPN

# sangfor vpn 功能

# 和 IPSec VPN 相比

与标准 IPSec VPN 相比，Sangfor VPN 的专利技术的优势

  1. 支持两端都为非固定 IP 的公网环境 -> 通过 webagent 实现
  2. 更细致的权限粒度

与标准 IPSec VPN 相比，Sangfor VPN 的特殊场景

  1. 更细致的权限粒度
  2. 隧道间路由技术，分支用户通过总部上网，实现总部的统一管控
  3. 隧道内 NAT 技术，解决多个分支网段 IP 冲突的问题

# webagent

webagent 寻址过程 ：用于 Sangfor VPN 互联时，分支与移动用户寻找总部的地址，从而建立 VPN 连接（寻址过程中，所有信息均使用 DES 加密）

分支 SSL VPN 是固定的向主 webagent 发起访问请求（每隔多少秒发起一次请求）

# 线路探测

当总部存在多条线路时，可以建立多条线路，这样可以实现一个带宽的叠加，还能做到一个链路的备份

# sangfor vpn 术语

# 总部

提供 vpn 接入服务，为其他 vpn 用户提供接入账户校验的设备，一般将服务端所在的网络做总部

# 分支

即接入总部端的设备，一般将客户端所在的网络做分支

# 移动

sangfor vpn 软件客户端，又称为 PDLAN

一个 vpn 设备既可以当总部，又可以当分支，也可以同时当总部和分支的角色

# webagent 格式

sangfor vpn 是基于 TCP/UDP 的 4009 端口传输的

4 中格式

  1. IP: 端口 => 100.100.100.100:4009

适用于总部 vpn 设备有固定公网 IP 地址的环境

  1. IP1#IP2: 端口 => 100.100.100.100#200.200.200.200:4009

适用于总部 vpn 设备有多条固定 IP 的线路，且需要做 vpn 的线路备份的环境

  1. 网址的形式 => webagent.xxx.com/webagent/index.php

适用于总部 vpn 设备没有固定公网 IP 的环境，如 ADSL 线路

  1. 域名：端口形式 => www.webagent.com:4009

适用于总部已存在动态域名指向他们出口的公网 IP 环境

# 本地子网

IPSec VPN 一般通过 ACL 抓取感兴趣流，而 sangfor vpn 是通过本地子网宣告自身内网网段给对端的形式，来让对端具备访问本端网段的路由，从而实现数据的互访（设备默认只宣告设备直连网段）

# VPNTUN 接口

vpn 数据的虚拟路由口，用来引导数据发往 vpn 隧道，从而封装报文

# 数据传输

# 特殊场景

# 禁止访问其他服务

  1. 分支和总部部署了两台 vpn 设备，现总部的 vpn 设备作为 von 总部域分支建立了 vpn 连接，用户要求对分支访问总部的服务器进行权限控制，比如只允许分支网络的 PC 访问总部的 web 服务器，禁止访问其他任何服务器，包括 PC 客户端

  1. 通过 vpn 内网权限（两端都会收到限制）
  2. 防火墙过滤规则

# 分支通信

  1. 总部和两个分支分别建立 vpn 连接，分支一与分支二均能访问总部内网，现在用户要求分支一和分支二之间能相互访问（两个分支分别在总部建立 vpn 隧道，但分支一和分支二没有任何线路相连，也没有 vpn 隧道）

通过分在在分支一和分支二的设备中配置隧道间路由实现

# 分支通过总部上网

  1. 总部与分支建立 vpn 连接，总部希望审计分支的上网行为，因此总部要求分支通过总部实现上网

通过在分支一种配置隧道间路由实现通过总部上网
