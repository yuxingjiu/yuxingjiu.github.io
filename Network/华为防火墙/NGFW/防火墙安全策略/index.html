防火墙简介

防火墙主要用于保护一个网络区域免受来自另一个网络区域的网络攻击和网络入侵行为

防火墙与交换机、路由器的对比

  1. 路由器与交换机工作在数据链路层与网络层，而防火墙工作在物理层（2 到 7 层）
  2. 路由器与交换机的本质是转发，防火墙的本质是控制

防火墙发展史

其中状态检测防火墙是跨时代的设备，下一代防火墙就是基于状态检测防火墙研发

UTM 防火墙是单核单行处理防火墙，下一代防火墙是多核并行处理防火墙

包过滤防火墙

缺点

  1. 无法关联数据包之间的关系（即来 100 个数据包只会一个一个检查数据）
  2. 无法使用多通道协议（像 ftp 存在两个端口）
  3. 通常不检查应用层数据

代理防火墙

缺点

  1. 处理速度慢（伪装成服务器对目标数据进行处理后在对服务器进行转发）
  2. 升级困难

状态检测防火墙

优点

  1. 处理后续包速度快（即查看确认序列号）
  2. 安全性高

防火墙特征

  1. 逻辑区域过滤器

区域 ：华为防火墙默认区域（Local、DMZ、Trust、Untrust）
每个区域都有唯一一个级别值（范围是 1-100） ：Local（100）、DMZ（85）、Trust（50）、Untrust（5） 安全级别数值越大越安全

  1. 隐藏内网网络结构

防火墙放在内网网络出口，设置 NAT 转换，隐藏内部网络

  1. 自身安全保障

登录防火墙需要身份验证，达到一定上限会锁死

  1. 主动防御攻击

经过防火墙的流量，只要是带有攻击性的就会被丢弃

防火墙安全区域

安全区域（Security Zone）简称区域（Zone）

  1. Zone 是本地逻辑安全区域的概念
  2. Zone 是一个或多个接口所连接的网络
  3. 一个区域可以同时包含多个接口，一个接口同时只能属于一个区域
  4. 一个接口划入某个安全区域，表示接口所连接的网段属于此安全区域，接口本身属于 local

区域

  1. Trust 区域
    该区域内网络的受信任程度高，通常来定义内部用户所在的网络（内网）
  2. Untrust
    该区域代表的是不受信任的网络，通常来定义 Internet 等不安全的网络（外网）
  3. DMZ（非军事区）
    该区域内网络的受信任程度中等，通常来定义内部服务器（公司 OA 系统、ERP 系统等）所在的网络（连接服务器）
  4. Local
    防火墙上提供了 Local 区域，代表防火墙本省（防火墙主动发起的报文以及抵达防火墙自身的报文）（防火墙自身）

防火墙安全策略

作用

  1. 默认防火墙禁止不同安全区域之间互访，同一区域内部允许
    控制单播（一对一）、组播（一对一部分）、广播（一对多）
  2. 管理员想放行不同区域之间的流量移动，需要通过安全策略放行

定义

  1. 安全策略是按一定顺序控制设备对流量转发以及对流量进行内容安全一体化安全检测的策略
  2. 规则的本质是包过滤

使用场景

  1. 对跨防火墙的网络互访进行控制（不同区域之间）
  2. 对设备本身的访问进行控制（其他区域访问的防火墙）

根据定义的规则对经过防火墙的流量进行筛选，并根据关键字确定筛选出的流量如何进行下一步操作

即策略靠前，匹配成功后直接进行后续包流程，不往下看其他策略

注意事项

  1. 先匹配最上面的策略（先配置的），策略与策略之间为 “或” 关系，所有用户自定义的策略都不能匹配，则匹配默认策略（默认策略是拒绝所有）
  2. 没有写的条件表示任意，写了的条件严格匹配（什么条件都不给表示默认，给了条件就会以条件为标准）
  3. 相同条件之间是 “或” 关系，不同关系是 “与” 关系
  4. 协议（service）应用（application）
  5. 条件惊喜的策略放在最上面，条件宽泛的策略放在下面
  6. 如果匹配协议时发现存在未定义的(service http => service protocol tcp destination-port 80)可以使用协议来替代，像 http 是基于 tcp 的 80 端口，而 ospf 是基于 IP 协议的（ip 协议直接跟上协议号(ospf 是 89)) (service ospf => service protocol 89) (service icmp => service protocol 1)

防火墙会话表

  1. 会话表中存放的是会话，会话简称为 session，一个会话表中包含多条 session（会话）
  2. 每条 session 都有老化时间
  3. 会话表中的 session 都是正向的，但系统中还存放着反向的 session
  4. 正向的 session 用于数据包首次访问发出，反向的 session 用于回来的数据进入

查看会话表

[FW]display firewall session table // 简单的session表
[FW]display firewall session table verbose // 详细信息

由于 session 表中存放的是一个正向 session，系统中还有一个反向 session

 icmp // 正向
 Zone: trust --> dmz  TTL: 00:00:20  Left: 00:00:11 // 正向传输区域，传送时间，等待时间
 Recv Interface: GigabitEthernet1/0/1 // 流量入端口
 Interface: GigabitEthernet1/0/3  NextHop: 10.1.3.100  MAC: 5489-9815-0a44 // 流量出端口，下一跳，MAC地址
 <--packets: 1 bytes: 60 --> packets: 1 bytes: 60 // 正向数据
 10.1.1.100:59845 --> 10.1.3.100:2048 PolicyName: trust_to_dmz_untrust // 会话由10.1.1.100产生去往10.1.3.100，匹配策略是trust_to_dmz_untrust

还有一个反向的，和正向的相反

icmp 反向
 Zone: dmz --> trust  TTL: 00:00:20  Left: 00:00:11 // 正向传输区域，传送时间，等待时间
 10.1.3.100:2048 --> 10.1.1.100:59845 PolicyName: trust_to_dmz_untrust // 会话由10.1.1.100产生去往10.1.3.100，匹配策略是trust_to_dmz_untrust

这里就思考一个问题，设置了允许 trust 区域访问 dmz，dmz 可以访问我们吗？

这就是因为会话表的存在，才会产生这种奇特的现象

具体原理（首包会产生正向 session）

首包 ：是第一个数据包，像 ping ip 的时候会有发送包和接收包，而第一个发送包就是首包

这就说明了为什么 trust 可以访问 dmz，而 dmz 不能访问 trust

因为 dmz 主动发起请求会发送首包，创建会话时无法通过安全策略，会话创建失败，所以无法 ping 通
