智能选路背景

在网络出口经常会部署多条出口链路，以增加出口的带宽和可靠性，如果不能够将流量合理的分配到链路上，可能会导致网络出现拥塞，访问速度变慢，链路资源得不到充分利用以及跨运营商访问等

当FW作为网络出口设备拥有多条出口链路时，智能选路功能可以根据管理员设置的带宽、权重、优先级自动探测链路质量并动态地选择出接口，保证链路资源得到充分利用，提升用户的上网体验（多出口智能选路不一定是防火墙上才有的技术，实际中可能是防火墙，也可能是负载均衡或设备，不同查那个上实现原理会有区别，配置也有一定差异）

多出口选路存在的问题

​

在多出口选路的场景中，通常需要考虑很多场景

  1. 如何让链路带宽、权重、优先级或自动探测到的链路质量动态的选择出接口，实现链路资源的合理利用和用户体验的提升
  2. 如何在路由表已经产生的情况下，不按照现有的路由表进行转发，而是根据用户制定的策略进行路由选择，来决定报文如何转发
  3. 如何对服务可用性、链路可用性或链路时延进行探测，并根据探测结果调整业务流量的分配，为网络服务质量提供必要保障
  4. 如何在设备连接多个ISP网络时可以保证流量转发使用最短路径（电信走电信，联通走联通，避免绕路）

智能选路解决问题

  1. 全局选路策略
    当FW上存在多条等价路由时（多条缺省路由负载），全局选路策略带宽、链路质量、权重和优先级动态地选择出接口，保证链路资源得到充分利用，提升用户的上网体验
  2. 策略路由
    策略路由能够依据用户指定的策略进行数据的转发
  3. ISP选路（基于ISP路由的选路）
    当FW作为出口网关设备连接多个ISP网络时，通过批量生成ISP路由，使访问特定ISP网络的流量从相应的出接口转发出去，保证流量转发使用最短路径
  4. 健康检查（探测，和ip-link类似）
    FW可以感知服务器或链路的状态变化，保证流量传输不受服务器或链路故障的影响

智能选路原理

智能选路的功能主要分为选路、探测

​​

全局选路策略

​​

在多出口场景下，当到达目的网络有多条等价路由或者缺省路由时，FW默认按照逐流负载分担模式进行转发，使用源IP地址和目的IP地址进行HASH计算选择出接口，即由报文的源IP和目的IP决定选择哪条路，不会考虑各条链路的实际宽带或链路的实时状态，当转发流量较大时，很可能出现一部分链路拥塞、另一部分闲置的情况，造成链路资源的浪费，当某些链路的传输质量比较差时，可能造成访问失败，影响用户的体验，用户无法选择优先使用某些链路转发流量，可能产生额外的费用，全局选择策略可以是FW按照不同的选路方式转流量到各条链路上，并根据各条链路的实时动态调整分配结果，以此提高链路资源的利用率和用户体验

防火墙负载分担方式针对数据流进行负载分担，在防火墙上对于每个会话计算出一个哈希值，根据管理员配置的负载分担比例将哈希值范围分为数块，对应各个出接口，哈希因子默认为源IP，这样可以尽量见啥使用同一源IP上网的用户发生链路切换，哈希因子可以配置（可以是源IP、目的IP、源端口、目的端口的任意组合）

用户可以根据具体需求设置链路的过载保护阈值，当某条链路承载的流量超过用户所设置的阈值时，过载保护会将超出的流量在未超出阈值的链路间负载，提高链路利用率和用户体验

模式

根据链路带宽负载分担

​​

FW有3条出接口链路，其中与 ISP1 相连的链路带宽为200M，与 ISP2 和 ISP3 相连的链路带宽均为100M，如果根据链路带宽负载分担，当FW转发一段时间流量后，各链路上累计传输的流量将分别占到总流量的50%、25%、25%

根据链路带宽负载分担是默认的智能选路方式，当企业从不同ISP处获得多条带宽不等的链路时，为了充分利用每个链路的带宽，提高链路的利用率，可以选择根据链路带宽负载分担

带宽是指管理员在FW上为各个接口指定的宽带，一般来说，管理员需要根据实际链路带宽设置合理的带宽值，FW按照带宽比例将流量分配到各条链路上，所以带宽大的链路转发较多的流量，带宽小的链路转发较少的流量，所有链路都会被充分利用，不会有链路闲置的情况

为了保证链路不会过载，管理员设置了过载保护阈值，各链路均为90%，当某条链路的带宽使用率高达90%时，已建立会话的流量仍从该链路转发，但是后续新建立会话的流量不在通过此链路转发，FW会在未过载的链路中智能选路，后续流量按照未过载链路之间的带宽比例进行负载分担，如果所有链路均已过载，那么FW将继续按照个链路的带宽比例分配流量

FW是根据各接口指定带宽的比例来分流的，而不是根据流量的实时流速，所以实际上个借口链路上分配的流量比例很难和设置的带宽比例一致 (各链路传输流量的比例和带宽的比例成正比)

根据链路质量负载分担

​​

如果采用链路质量负载分担，FW向各个ISP内的指定设备发送5个探测报文，其中ISP1、ISP2和ISP3链路丢爆率为0%、40%、100%，所以FW判定ISP1的质量最高，优先使用ISP1链路转发流量，只要探测表项没有老化，FW将一致使用ISP1转发流量，不会使用ISP2和ISP3 (相当于主备备份)

当企业从不同ISP处获得多条链路时，为了使用户获得最佳的访问体验，需要FW能够根据各链路的实时传输质量动态调整流量的分配，可以选择根据链路质量负载分担

FW优先使用链路质量高的链路转发流量

  1. 丢包率（FW发送若干个探测报文后，将统计丢包的个数，并计算丢包率)
  2. 时延（回应报文的接收时间减去探测报文的发送时间即为时延）
  3. 时延抖动(相邻两次探测的时延之差取绝对值即为时延抖动)

是FW衡量链路质量的3个参数，管理员可以根据实际需要选择其中的一个或多个参数，三个质量参数中，丢包率是最重要的参数，如果两条链路的丢包率、时延、时延抖动都各不相同，那么FW判定丢包率小的链路质量高

FW自动向目的IP发送链路质量探测报文，获取各链路的传输质量信息，并将链路质量探测结果保存在链路质量探测表中，当有流量达到FW时，FW首先根据报文的目的IP去匹配探测表，如果匹配则根据表中记录的出接口转发流量，如果未匹配，则自动向目的IP发起质量探测选择最优的链路转发流量，并将探测结果记录在链路质量探测表中，当质量探测表项老化后，新的流量触发智能选路时需要重新进行链路质量探测

缺省情况下，链路质量探测报文的协议类型是 tcp-simple (FW使用TCP报文检查网络的连接性，只要目的设备回应第一个探测报文，即认为链路时可用的，无需完成三次握手)

FW针对TCP业务流量使用 tcp-simple协议 进行质量探测
FW针对非TCP业务流量都是用 ICMP协议 进行质量探测

探测协议的报文类型还可以修改为 ICMP，就对所有业务流量都是用 ICMP协议 进行探测

根据链路权重负载分担

​​

FW拥有3条出接口链路属于不同的ISP，其中管理员设置ISP的权重分别为5/3/2，所以权重比例为5:3:2，当FW转发一段时间流量后，各链路上累计传输的流量将分别占到总流量的50%、30%、20%

当企业从不同ISP处获得多条性能不等的链路时，为了优化使用转发性能最优的链路，保证大多数用户的访问体验的同时，且不浪费其他性能稍差的链路，可以使用根据链路权重负载分担

管理员在FW上为各个接口指定权重时，一般情况需要考虑链路的带宽、转发时延、链路租借费用等等因素，而前面说的转发性能最优的链路不一定是转发速度最快的链路，而是最符合企业利益的链路，所以管理员需要根据实际情况设置合理的权重

根据链路优先级主备备份

​​

FW有3条出接口链路属于不同的ISP，其中优先级分别为8/6/2，ISP1优先级最高，FW优先使用ISP1链路转发流量，当ISP1的链路使用率超过过载阈值时，将启用优先级次高的链路路转发流量

当企业从不同ISP处获得多条链路时，如果各链路的带宽、转发时延、链路租借费用等因素存在较大差异，那么可以优先使用某些链路传输流量，并利用其他链路作为备份链路或负载分担链路，提高业务的可靠性，就可以选择根据链路优先级主备备份

管理员为各接口指定合理的优先级后，优先级最高的接口称为主接口，其他优先级的接口统称为备份接口，FW优先使用主接口转发流量，如果没有为主接口链路指定过载保护阈值，那么即使链路过载，FW也不会使用其他链路传输流量，只有当主接口链路发生故障后，优先级次高的备份接口才启用以替代主接口，而其他优先级更低的备份接口则仍未启用

会话保持

智能选路接口可以配置过载保护阈值，当接口链路的带宽利用率达到过载保护阈值时，FW对新流量进行只能选路时将排除该过载链路，在其他未过载的链路中进行选路，这样可能会导致用户上网流量在接口链路过载前选择了该接口链路，而新建会话流量因为原接口链路过载而被FW从其他接口转发出去，从而出现已经登录的网站在刷新后需要重新登录，网络游戏在链路切换后掉线，甚至某些网上银行业务因检测到IP地址变化而拒绝用户访问等现象

​​

FW支持在四种智能模式选路方式中配置会话保持功能

开启会话保持后，流量进行首次智能选路选择某接口后，FW会生成相应的会话保持表项，新流量如果命中了该会话保持表项，FW按照会话保持表项中记录的出接口转发流量，这样能保证该用户的流量始终使用同一接口链路转发（直接使用会话保持表项中记录的出接口转发该流量，这样就保证了此用户的流量始终同一出接口转发）

​​

用户A的上网流量进行首次智能选路选路后，会生成一个会话保持销项，其中包含了源IP、匹配的智能选路策略ID和首次选路的出接口，当该用户再次发起连接时，FW会根据新流向中的源IP和匹配的只能选路策略ID查找相应的会话保持表项，并直接使用会话保持表项中记录的出接口转发该流量

完成全局选路策略的配置后，后续经过FW的流量将按照选路策略被转发，而之前的部分流量由于会话没有老化，所以不会立即按照选路策略被转发，可以手动输入 rest firewall session table 手动清除会话信息或者等待会话老化，而清除回话表象信息将导致连接终端，所以必须确认对业务不会造成不良影响才能清除会话表信息

策略路由

策略路由是一种依据用户指定的策略进行转发的机制（可以控制路由）

​​

FW转发数据报文时，会查找路由表，并根据目的地址来进行报文的转发，在这种机制下，只能根据报文的目的地址为用户提供转发服务，无法提供有差别的服务

策略路由是在路由表已经产生的情况下，不按照现有的路由表进行转发，而是根据用户制定的策略进行路由选择的机制，从更多的方面（入接口、源安全区域、源/目IP、用户、服务、应用）来决定报文如何转发，增加了在报文转发控制上的灵活度

策略路由并没有替代路由表机制，而是优先于路由表生效，为某些特殊业务指定转发方向（不能匹配策略路由的，匹配路由表中的路由）

ISP路由

​​

FW拥有两条属于不同ISP网络的出口链路，当内网用户访问ISP2中的Server2时，如果FW上存在等价路由，则FW可以通过两条不同的路径达到Server2，其中路径3显然不是最优路径，路径1才是用户所期望的路径

当FW作为出口网关设备连接多个ISP网络时，通过ISP选路功能可以使访问特定ISP网络的流量从相应出接口转发出去，保证流量转发使用最短路径，配置ISP选路功能后，当内网用户访问Server1或Server2时，FW会根据目的地址所在ISP网络选择响应的出接口，从而使访问流量通过最短路径到达服务器（路径1和路径3）

健康检查

使用健康检查功能后，防火墙探测到ISP1链路故障，当流量触发智能选路时，ISP1链路不在参与智能选路，防火墙将从ISP2链路和ISP3链路中选择提交进行流量转发（和IP-Link类似）

​

​

当防火墙拥有多个出接口时，通过只能选路功能可以使流量按照链路带宽、质量、权重或优先级动态地选择出接口，保证链路资源得到充分利用

为提高流量转发的可靠性，智能选路功能可以配合健康检查功能一起使用，保证流量不被转发到故障链路上，当健康检查的结果显示链路故障时，对应的接口链路将不载参与智能选路，流量也就避免被转发到故障连路上，当链路状态恢复正常时，对应的接口链路重新参与智能选路，并转发分配到的流量，健康检查是单独的功能，和链路质量探测不同，链路质量探测主要是探测链路质量（丢包、时延等），而健康检查是确保链路正常，所以只要参加智能选路的链路建议都配置一条健康检查与之对应

​​

未使用健康检查前由于不能感知到ISP2链路发生故障，所以访问Server的流量继续按照ISP路由从ISP2链路转发，导致访问失败，使用健康检查功能后，FW无法访问Server，对应的ISP路由也被删除，如果ISP1和ISP2之间存在互通路由，则访问流量可以通过ISP1网络到达Server，虽然不是最优访问路径，但是可以保证流量转发的可靠性，提升用户体验

健康检查原理

  1. 防火墙可以根据不同类型的目的设备发送响应协议的探测报文，通过分析应答报文即可判断链路的可用性
  2. 防火墙提供了丰富的健康检查机制，可以提高链路和服务的质量，提升用户的使用体验
  3. 健康检查可以对服务的可用性、链路可用性或链路时延进行探测，并根据探测结果调整业务流量的分配，为网络服务提供必要的保障
  4. 防火墙通过健康检查结果实时感知网络中发生的变化，并立即做出响应地调整，保证所使用的服务器和链路时可用的，提高服务的稳定性和可靠性
  5. 当多个服务器或链路可用时，防火墙可以根据服务类型选择性能最优的服务器处理业务流量或根据链路时延大小选择网络时延最小的链路传输业务流量
探测协议探测原理
DNS使用NDS协议向指定设备发起请求如果应答报文中的标识字段与请求报文一致，即认为该链路可用
HTTP完成TCP三次握手后，使用HTTP协议向指定设备发送获取目的根目录请求，收到响应后即认为该链路可用，防火墙发送RST终止TCP连接
ICMP向指定设备发送ICMP报文，如果ICMP应答报文中的标识符和序列号字段与请求报文一致，即认为该链路可用
RADIUS使用RADIUS协议向指定服务器发起认证请求，用户名为guestguest，密码为空，如果应答报文中的表示符与发送报文一致，即认为该服务可用
TCP使用TCP协议向指定设备发送TCP连接请求，如果连接建立成功，即认为该链路可用，随后防火墙会发送RST终止TCP连接
TCP简单模式使用TCP报文检查网络的连通性，只要目的设备回应第一个探测报文，即认为链路可用，无需完成三次握手

DNS透明代理

  • 也叫智能DNS

当内网用户通过域名访问Web服务器时，需要发送DNS请求报文给DNS服务器进行域名解析，如果企业租用多个ISP链路作为网络出口，每个ISP内都有提供相同服务的web服务器，一般来讲，企业内网用户的客户端都会配置一个相同的DNS服务器地址，而DNS服务器通常会将域名解析成自己所在ISP内的Web服务器地址，这将导致内网用户的上网流量都将集中到一个ISP的链路上转发，最终导致链路拥塞，影响用户上网体验，同时由于其他ISP的链路资源没有被使用，也造成了资源的浪费

​​

DNS透明代理

NGFW的DNS透明代理功能可以修改部分DNS请求报文的目的地址，将其修改为其他ISP内的DNS服务器地址（去放ISP1的请求修改为去往ISP2的DNS服务器地址），DNS请求被转发到不同的ISP，解析后的Web服务器地址就属于不同的ISP，所以上网流量将通过不同的ISP链路转发，就不会造成链路拥塞，充分利用了所有链路资源​

​​

DNS代理过程

  1. 当DNS请求和报文到达NGFW时，NGFW首先会检查DNS服务器地址是否需要做透明代理，只有管理员指定的DNS服务器才会做DNS透明代理（管理员可以排除一些域名，当用户访问这些域名时，即使DNS服务器地址需要做透明代理，NGFW也不会捉处理，而是直接转发）
  2. DNS透明代理功能通过替换DNS请求报文的目的地址，达到在不同ISP链路上分担流量的目的，而替换后的DNS服务器地址是什么值由出接口决定
  3. NGFW在每个出接口绑定2个DNS服务器，一个首选，一个备用，它们属于该出接口直连的ISP网络，当NGFW决定DNS请求报文的出接口后，DNS透明代理功能优先使用首选DNS服务器的地址替换DNS请求报文的目的地址，只有首选DNS服务器的状态为down，才使用备用DNS地址进行替换
  4. DNS服务器将解析后的web服务器地址返回给用户，此web服务器和DNS服务器属于同一个ISP
  5. 用户根据返回的地址访问web服务器，此时需要使用运营商地址库选路功能，使流量通过响应的出接口转发，经由目的地址所在ISP的网络到达web服务器

​​

DNS透明代理功能需要和智能选路功能（策略路由或全局选路策略）、运营商地址库选路功能一起配合使用，才能起到分担流量的作用效果，其中，智能选路功能用于选择转发DNS请求报文的出接口，运营商地址选路功能用于保证业务流量经由目的地址所在ISP的网络达到web服务器，单独配置DNS透明代理功能没有意义，因为配置下发后并不生效
