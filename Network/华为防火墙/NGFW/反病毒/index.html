# 计算机病毒

# 病毒

病毒的主要目的是破坏信息，入侵的主要目的是窃取信息

<font color="#ff0000"> 病毒是一种恶意代码 </font>，可感染或附着在应用程序或文件中，一般通过邮件或文件共享等协议进行传播，威胁用户主机和网络的安全

有些病毒会耗尽主机资源、占用网络宽带，有些病毒会控制主机权限，窃取用户数据，有些病毒还会对主机硬件造成破坏

# 病毒传播途径

电子邮件

  1. HTML 正文可能被嵌入恶意脚本
  2. 邮件附件携带病毒压缩文件
  3. 利用社会工程学进行伪装，增大病毒传播机会
  4. 快捷传播特性

网咯共享

病毒会搜索本地网络中存在的共享，包括默认共享，如 ADMIN$、IPC$、E$ 等，通过 \\x.x.x.x\E$ 进行访问

  1. 通过空口了或弱口令猜测
  2. 病毒自带口令猜测列表
  3. 将自身复制到网络共享文件夹中

P2P 共享软件

  1. 将自身复制到 P2P 共享文件夹
  2. 通常以游戏、CDKEY 等相关名字命名
  3. 通过 P2P 软件共享给网络用户

# 杀毒软件

  1. 特征码技术
    杀毒软件存在病毒特征库，包含了各种病毒的特征码（一段特殊的程序，从病毒样本中抽取出来，与正常程序不同），把被扫描的信息与特征库匹配
  2. 行为查杀技术
    病毒在运行时会有各种行为特征（系统中增加有特殊后缀的文件、监控用户行为等）

# 网关反病毒

FW 上配置反病毒功能后，正常文件可以顺利进入内部网络，包含病毒的文件则会被检测出来，并采取阻断或告警等手段进行干预

# 工作原理

支持对使用一下协议传输的文件进行病毒检测

  1. FTP ：文件传输协议
  2. HTTP ：超文本传输协议
  3. POP3 ：邮件协议的第三个版本
  4. SMTP ：简单邮件传输协议
  5. UMAP ：因特网信息访问协议
  6. NFS ：网络文件系统
  7. SMB ：文件共享服务器

NGFW 支持对不同传输方向上的文件进行病毒检测

上传 ：指客户端向服务器发送文件
下载 ：指服务器向客户端发送文件

# 实现技术

代理扫描方式

将所有经过防火墙的需要进行病毒检测的数据报文透明的转交给防火墙自身的协议栈，通过防火墙自身的协议栈将文件全部缓存下来后，在送入病毒检测引擎中

流扫描

依赖于状态检测技术以及协议解析技术，提取文件的特征与本地签名库进行匹配

区别

基于代理的反病毒网关可以进行更多，如解压、拓客等高级操作，检测率高、但是由于进行了文件全缓存，性能、系统开销会比较大，而基于流扫描的反病毒网关性能高、开销小，但检测率优先
