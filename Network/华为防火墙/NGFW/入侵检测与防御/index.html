# 入侵检测与防御

传统的信息安全方法采用严格地访问控制和数据加密策略来防护但在复杂系统中，这些策略是不充分的，不能完全保证系统的安全

<font color="#ff0000"> 入侵检测时对入侵行为的发掘 </font>，通过从计算机网络或计算机系统的关键点收集信息并分析，从中发现网路或系统中是否有违反安全策略的行为和被攻击的迹象

什么是入侵？
入侵是指未经授权而尝试访问信息系统资源、篡改信息系统中的数据，使信息系统不可靠或不能使用的行为

典型的入侵行为

  • 篡改 web 网页
  • 破解系统木马
  • 复制 / 查看敏感数据
  • 使用网络嗅探工具获取用户密码
  • 访问未经允许的服务器
  • 其他特殊硬件获得原始网络包
  • 向主句植入特洛伊木马程序

# 入侵检测 [ID]

通过监视各种操作，分析、审计各种数据和现象来实时检测入侵行为的过程，入侵检测的内容覆盖了授权和非授权的各种入侵行为

# 入侵检测系统 [IDS]

入侵检测的内容覆盖了授权和非授权的各种入侵行为，违反安全策略行为、冒充其他用户、泄露系统资源、恶意行为、非法访问、以及授权者滥用权力等，入侵检测系统可以通过网络和计算机动态地搜集大量关键信息资料，能及时分析和判断整个系统环境的目前状态，一旦发现违反安全策略的行为或系统存在被攻击的痕迹等会立即启动有关安全机制进行应对（主要是上报或记录，不会做处理 —— 类似摄像头，只做记录）

# 技术实现

  1. 异常检测模型 （Anomaly Detection）
    首先总结正常操作应该具有的特征（通过对系统审计数据的分析建立起来的系统主体的正常行为特征轮廓），当用户活动与正常行为有重大偏离时，即被认为是入侵

  1. 误用检测模型（Misuse Detection）—— 特征检测（Signature-based Detection）
    收集非正常操作的行为特征，建立相关的特征库，当检测的用户或系统行为与库中的记录匹配时，也认为是入侵

# 入侵防御

# 入侵防御系统 [IPS]

在发现入侵行为时能实时阻断的入侵检测系统（华为的 NIP 设备时专门的 IPS 设备）
IPS 使得 IDS 和防火墙走向统一

# 部署方式

  1. 旁路

SPAN ：接在交换机上，通过交换机做端口镜像
TAP ：通过专用的流量镜像设备，部署在网络边界

  1. 直路

Inline ：串接在网络边界，在线部署，在线阻断

# 入侵检测与防御区别

IDS 的主要作用是监控网络状况，侧重于风险管理

IPS 的主要作用是实时阻断入侵行为，侧重于风险控制

# 作用不同

IDS 入侵检测系统，IDS 主要检测系统内部，运行在被监控的主机上，对主机的网络行为、系统日志、进程和内存等指标进行监控
IPS 是检测在系统的防火墙和外网之间，针对流向内部的流量进行分析，监视网络或网络设备的网络资料传输行为的计算机网络安全设备，能够即时的终端、调整或隔离一些恶意行为

# 部署方式

IDS 通常采用旁路接入，在网络中的位置一般选为尽可能靠近攻击源、尽可能靠近受保护资源，这些位置通常在服务器区域的交换机上，外网接入路由器后的第一台交换机上，重点保护网段的局域网交换机上
IPS 通常采用直路接入，在办公网络中，至少需要在办公网与外部网络的连接部位、重要服务器集群前端、办公网内部接入层

# 工作机制不同

IDS 主要针对已发生的攻击事件或异常行为进行处理，属于被动防护（对所监测的网络数据进行获取、还原），根据签名进行模式匹配或进行一系列统计分析，根据结果对有问题的会话进行阻断，或者和防火墙产生联动（<mark style="background: #ABF7F7A6;">IDS 致命缺点在于阻断 UDP 会话不太灵，对加密的数据流也束手无策 </mark>）
IPS 针对攻击事件或异常行为可提前感知及预防，属于主动防护，根据设置的过滤器，分析相对应的数据包，通过检查的数据包可以继续前进，包含恶意内容的数据包会被丢弃，被怀疑的数据包需要接收进一步的检查

IPS 的阻断方式比 IDS 更为可靠，可以终端拦截 UDP 会话，也可以做到确保符合签名规则的数据包不漏发到被保护区域
