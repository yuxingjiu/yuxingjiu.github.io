配置双机

#双机热备-上下接交换机-主备模式

  1. 配置VRRP备份组

在FW1下行业务接口g1/0/1和FW2下行业务接口g1/0/1配置VRRP组1，其FW1为Active，FW2为Standby
在FW1上行业务接口g1/0/2和FW2上行业务接口g1/0/2配置VRRP组2，其FW1为Active，FW2位Standby

FW1配置

interface GigabitEthernet1/0/1
 vrrp vrid 1 virtual-ip 10.1.1.254 active
 
interface GigabitEthernet1/0/2
 vrrp vrid 2 virtual-ip 202.96.137.88 255.255.255.0 active

FW2配置

interface GigabitEthernet1/0/1
 vrrp vrid 1 virtual-ip 10.1.1.254 standby

interface GigabitEthernet1/0/2
 vrrp vrid 2 virtual-ip 202.96.137.88 255.255.255.0 standby
  1. 指定心跳接口并启动双机热备

FW1配置

hrp interface GigabitEthernet1/0/6 remote 192.168.255.2
hrp enable

FW2配置

hrp interface GigabitEthernet1/0/6 remote 192.168.255.1
hrp enable

配置完成后进入同步状态 (主动端配置会同步到被动端，而被动端无法进行手动配置)

配置策略（安全策略 + NAPT，只需要在FW1（主）端配置，会同步到FW2（备））

FW1配置

security-policy
 rule name u_t
  source-zone trust
  destination-zone untrust
  source-address 10.1.1.0 mask 255.255.255.0
  action permit

nat address-group 202
 section 202.96.137.88

nat-policy
 rule name t_ut
  source-zone trust
  destination-zone untrust
  action source-nat address-group 202

双机配置

#双机热备-上下接交换机-负载分担

两台FW的业务接口都在三层，上下行分别连接二层交换机，可以通过VRRP来监测链路，创建VRRP备份组

  1. VRRP备份组配置

FW1配置

interface GigabitEthernet1/0/1
 vrrp vrid 1 virtual-ip 10.1.1.254 active

interface GigabitEthernet1/0/2
 vrrp vrid 2 virtual-ip 10.1.2.254 standby

interface GigabitEthernet1/0/3
 vrrp vrid 3 virtual-ip 202.96.137.88 255.255.255.0 active

interface GigabitEthernet1/0/4
 vrrp vrid 4 virtual-ip 61.69.100.88 255.255.255.0 standby

interface GigabitEthernet1/0/2

interface GigabitEthernet1/0/3

interface GigabitEthernet1/0/4
 vrrp vrid 4 virtual-ip 61.69.100.88 255.255.255.0 active
  1. 心跳线配置，开启双机热备

FW1配置

 hrp mirror session enable // 开启快速会话备份
 hrp interface GigabitEthernet1/0/6 remote 192.168.255.2
 hrp enable
 hrp nat resource primary-group // 配置NAT端口范围，主设备是65535的前一半，配置后备设备自动配置为65535后一半

FW2配置

 hrp mirror session enable
 hrp interface GigabitEthernet1/0/6 remote 192.168.255.2
 hrp enable
 hrp nat resource primary-group

当链路探测发现故障时会将优先级-2，切换主备，但是当接口恢复后，还是备，因为FW1成为备墙后不存在缺省路由，一直探测失败

想要恢复原先主墙，必须让现在的主墙先退出，等待原先的主墙链路探测变为up，在重新启动即可
