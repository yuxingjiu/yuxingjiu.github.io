[FW]interface GigabitEthernet 1/0/2
[FW-GigabitEthernet1/0/2]ip address 10.1.2.1 255.255.255.0

[FW]interface GigabitEthernet 1/0/3
[FW-GigabitEthernet1/0/3]ip address 10.1.3.1 255.255.255.0

[FW]interface GigabitEthernet 1/0/4
[FW-GigabitEthernet1/0/4]ip address 10.1.4.1 255.255.255.0

[FW]interface GigabitEthernet 1/0/5
[FW-GigabitEthernet1/0/5]ip address 10.1.5.1 255.255.255.0

[FW]interface GigabitEthernet 1/0/6
[FW-GigabitEthernet1/0/6]ip address 100.100.100.1 255.255.255.0

# 防火墙区域配置

[FW]firewall zone trust
[FW-zone-trust]add interface GigabitEthernet0/0/0
[FW-zone-trust]add interface GigabitEthernet1/0/1
[FW-zone-trust]add interface GigabitEthernet1/0/2

[FW]firewall zone dmz
[FW-zone-dmz]add interface GigabitEthernet1/0/3

[FW]firewall zone untrust
[FW-zone-untrust]add interface GigabitEthernet1/0/4

[FW]firewall zone name server
[FW-zone-server]set priority 90
[FW-zone-server]add interface GigabitEthernet1/0/6

测试

测试同一区域内的可以访问，但是访问不同区域的就无法 ping 通，这是因为防火墙默认拒绝所有区域互相访问（包括网关，因为网关对应 Local）

# 防火墙策略

实验要求

  1. 内网 Trust 区域客户端可以访问 DMZ/Untrust 区域的所有业务（FTP/PING）
[FW]security-policy
[FW-policy-security]rule name trust_to_dmz_untrust
[FW-policy-security-rule-trust_to_dmz_untrust]source-zone trust
[FW-policy-security-rule-trust_to_dmz_untrust]destination-zone dmz
[FW-policy-security-rule-trust_to_dmz_untrust]destination-zone untrust
[FW-policy-security-rule-trust_to_dmz_untrust]source-address 10.1.1.0 mask 255.255.255.0
[FW-policy-security-rule-trust_to_dmz_untrust]source-address 10.1.2.0 mask 255.255.255.0
[FW-policy-security-rule-trust_to_dmz_untrust]service ftp
[FW-policy-security-rule-trust_to_dmz_untrust]service icmp
[FW-policy-security-rule-trust_to_dmz_untrust]action permit

  1. 内网 Trust 区域客户端只允许 10.1.2.0/24 网段可以访问 Server 区域 www 服务器（HTTP 可以，PING 禁止）
[FW]security-policy
[FW-policy-security]rule name trust_to_server
[FW-policy-security-rule-trust_to_server]source-zone trust
[FW-policy-security-rule-trust_to_server]destination-zone server
[FW-policy-security-rule-trust_to_server]source-address 10.1.2.0 mask 255.255.255.0
[FW-policy-security-rule-trust_to_server]service http
[FW-policy-security-rule-trust_to_server]action permit

  1. 禁止 Trust 区域 10.1.2.100/24 这个地址访问 Untrust 区域
[FW-policy-security]rule name un_trust_to_untrust
[FW-policy-security-rule-un_trust_to_untrust]source-zone trust
[FW-policy-security-rule-un_trust_to_untrust]destination-zone untrust
[FW-policy-security-rule-un_trust_to_untrust]source-address 10.1.2.0 mask 255.255.255.0
[FW-policy-security-rule-un_trust_to_untrust]action deny

// 将策略移到最上边，因为第一条规则是允许Trust区域访问Untrust
[FW-policy-security]rule move un_trust_to_untrust top

  1. 允许 Untrust 区域访问 Server 区域的 www 服务器
[FW]security-policy
[FW-policy-security]rule name untrust_to_server
[FW-policy-security-rule-untrust_to_server]source-zone untrust
[FW-policy-security-rule-untrust_to_server]destination-zone server
[FW-policy-security-rule-untrust_to_server]destination-address 100.100.100.100 mask 255.255.255.255
[FW-policy-security-rule-untrust_to_server]action permit

# 命令配置


IP 配置

[FW]int g1/0/1
[FW-GigabitEthernet1/0/1]ip a 10.1.1.1 24
[FW]int g1/0/2
[FW-GigabitEthernet1/0/2]ip a 10.1.2.1 24
[FW]int g1/0/6
[FW-GigabitEthernet1/0/6]ip a 100.100.100.1 24
[FW]int g1/0/5
[FW-GigabitEthernet1/0/5]ip a 200.200.200.1 24
[FW]int g1/0/3
[FW-GigabitEthernet1/0/3]ip a 10.1.3.1 24

区域划分

[FW]firewall zone trust
[FW-zone-trust]add interface g1/0/1
[FW-zone-trust]add interface g1/0/2
[FW]firewall zone name server1
[FW-zone-server1]set priority 20
[FW-zone-server1]add interface g1/0/6
[FW]firewall zone name server2
[FW-zone-server2]add interface g1/0/5
[FW-zone-server2]set priority 21
[FW]firewall zone untrust
[FW-zone-untrust]add interface g1/0/3


策略配置

[FW-policy-security]rule name t_ut_to_s1
[FW-policy-security-rule-t_ut_to_s1]source-zone trust
[FW-policy-security-rule-t_ut_to_s1]source-zone untrust
[FW-policy-security-rule-t_ut_to_s1]destination-zone server1
[FW-policy-security-rule-t_ut_to_s1]destination-address 100.100.100.100 32
[FW-policy-security-rule-t_ut_to_s1]service http
[FW-policy-security-rule-t_ut_to_s1]action permit
[FW-policy-security]rule name t_to_s2
[FW-policy-security-rule-t_to_s2]source-zone trust
[FW-policy-security-rule-t_to_s2]destination-zone server2
[FW-policy-security-rule-t_to_s2]service ftp
[FW-policy-security-rule-t_to_s2]destination-address 200.200.200.200 24
[FW-policy-security-rule-t_to_s2]action permit
[FW-policy-security]rule name un_t_100101_to_ut
[FW-policy-security-rule-un_t_100101_to_ut]source-zone trust
[FW-policy-security-rule-un_t_100101_to_ut]destination-zone untrust
[FW-policy-security-rule-un_t_100101_to_ut]source-address 10.1.1.100 32
[FW-policy-security-rule-un_t_100101_to_ut]source-address 10.1.1.101 32
[FW-policy-security-rule-un_t_100101_to_ut]uaction deny
[FW-policy-security]rule name t_to_ut
[FW-policy-security-rule-t_to_ut]source-zone trust
[FW-policy-security-rule-t_to_ut]destination-zone untrust
[FW-policy-security-rule-t_to_ut]action permit
[FW-policy-security]rule name s1_to_ut
[FW-policy-security-rule-s1_to_ut]source-zone server1
[FW-policy-security-rule-s1_to_ut]destination-zone untrust
[FW-policy-security-rule-s1_to_ut]service dns http https
[FW-policy-security-rule-s1_to_ut]action permit
[FW-policy-security]rule name s2_to_t
[FW-policy-security-rule-s2_to_t]source-zone server2
[FW-policy-security-rule-s2_to_t]destination-zone trust
[FW-policy-security-rule-s2_to_t]service icmp
[FW-policy-security-rule-s2_to_t]action permit

Trust、 untrust 区域客户端都可以访问 servre1 区域的 www 服务

Trust 区域可以访问 server2 的 FTP 服务

Trust 区域可以访问 untrust 区域的任何服务，但是 10.1.1.100-101 两个地址禁止访问 untrust

Servre1 区域可以访问 untrust 区域的 http、https、 dns 服务用于系统升级激活 (直接配置即可，不需要测试)

Server2 区域服务器不可以访问 untrust 区域 (禁止上网), 但是可以访问 Trust 区域的 icmp (ping 测试)
