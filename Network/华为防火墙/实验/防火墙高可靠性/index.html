# IP-Link

ISP 配置

[ISP]interface GigabitEthernet 0/0/2
[ISP-GigabitEthernet0/0/2]ip address 202.96.137.1 24
[ISP]interface GigabitEthernet 0/0/3
[ISP-GigabitEthernet0/0/3]ip address 113.100.1.1 24
[ISP]int LoopBack 0
[ISP-LoopBack0]ip address 100.100.100.100 32

FW 配置

[FW]interface GigabitEthernet 1/0/0
[FW-GigabitEthernet1/0/0]ip address 10.1.1.254 24
[FW]interface GigabitEthernet 1/0/2
[FW-GigabitEthernet1/0/2]ip address 202.96.137.88 24
[FW]interface GigabitEthernet 1/0/3
[FW-GigabitEthernet1/0/3]ip address 113.100.1.99 24
[FW]firewall zone trust
[FW-zone-trust]add interface GigabitEthernet 1/0/0
[FW]firewall zone untrust
[FW-zone-untrust]add interface GigabitEthernet 1/0/2
[FW-zone-untrust]add interface GigabitEthernet 1/0/3
[FW]security-policy
[FW-policy-security]rule name t_ut
[FW-policy-security-rule-t_ut]source-zone trust
[FW-policy-security-rule-t_ut]destination-zone untrust
[FW-policy-security-rule-t_ut]action permit
[FW]nat-policy
[FW-policy-nat]rule name t_ut
[FW-policy-nat-rule-t_ut]source-zone trust
[FW-policy-nat-rule-t_ut]destination-zone untrust
[FW-policy-nat-rule-t_ut]action source-nat easy-ip
[FW]ip route-static 0.0.0.0 0 202.96.137.1
[FW]ip route-static 0.0.0.0 0 113.100.1.1 preference 61


可以 ping 通，但是这条链路故障掉，防火墙就无法正常通信

再次 ping

#IP-Link 配置

[FW]ip-link check enable // 开启IP-Link
[FW]ip-link name LT // IP-Link配置名称
[FW-iplink-LT]destination 100.100.100.100 interface g1/0/2 mode icmp
// destination表示探测的地址，interface表示从哪个接口发数据 mode表示使用哪种协议探测，默认ICMP

[FW]ip route-static 0.0.0.0 0 202.96.137.1 track ip-link LT // 关联路由

IP-Link 初始化和现在状态为 Init 和 Up，现在继续把链路断掉

成功切换路由并且链路状态变为了 down，再次把链路恢复

# BFD

基础配置和 IP-Link 一致

当链路 down 掉，无法正常通信

#BFD 配置

FW 配置

[FW]bfd
[FW]bfd 1 bind peer-ip 202.96.137.1
[FW-bfd-session-1]discriminator local 10
[FW-bfd-session-1]discriminator remote 20
[FW-bfd-session-1]commit

ISP 配置

[ISP]bfd
[ISP]bfd 1 bind peer-ip 202.96.137.88
[ISP-bfd-session-1]discriminator local 20
[ISP-bfd-session-1]discriminator remote 10
[ISP-bfd-session-1]commit


配置关联路由

[FW]ip route-static 0.0.0.0 0 202.96.137.1 track bfd 1


成功切换路由并且链路状态变为了 down

还是比 IP-Link 块一点切换的

# 主备模式

R1 配置

[R1]interface GigabitEthernet0/0/0
[R1-GigabitEthernet0/0/0]ip address 192.168.255.1 30
[R1]interface GigabitEthernet0/0/1
[R1-GigabitEthernet0/0/1]ip address 10.1.1.252 24
[R1]ospf router
[R1]ospf router-id 1.1.1.1
[R1-ospf-1]area 0
[R1-ospf-1-area-0.0.0.0]network 10.1.1.0 0.0.0.255
[R1-ospf-1-area-0.0.0.0]network 192.168.255.0 0.0.0.3

R2 配置

[R2]interface GigabitEthernet0/0/0
[R2-GigabitEthernet0/0/0]ip address 192.168.255.5 30
[R2]interface GigabitEthernet0/0/1
[R2-GigabitEthernet0/0/1]ip address 10.1.1.253 24
[R2]ospf router-id 2.2.2.2
[R2-ospf-1]area 0
[R2-ospf-1-area-0.0.0.0]network 10.1.1.0 0.0.0.255
[R2-ospf-1-area-0.0.0.0]network 192.168.255.4 0.0.0.3

R3 配置

[R3]int LoopBack 0
[R3-LoopBack0]ip address 100.100.100.100 24
[R3]interface GigabitEthernet0/0/0
[R3-GigabitEthernet0/0/0]ip address 192.168.255.2 30
[R3]interface GigabitEthernet0/0/1
[R3-GigabitEthernet0/0/1]ip address 192.168.255.6 30
[R3]ospf router-id 3.3.3.3
[R3-ospf-1]area 0
[R3-ospf-1-area-0.0.0.0]network 192.168.255.0 0.0.0.3
[R3-ospf-1-area-0.0.0.0]network 192.168.255.4 0.0.0.3
[R3-ospf-1-area-0.0.0.0]network 100.100.100.0 0.0.0.255


可以正常通信，但这是配置了两个不同的网关，想要实现一个网关 down 掉，不影响这个网段正常通信，需要配置主备模式

#VRRP - 主备模式配置

R1 配置

[R1-GigabitEthernet0/0/1]vrrp vrid 1 virtual-ip 10.1.1.254

R2 配置

[R2-GigabitEthernet0/0/1]vrrp vrid 1 virtual-ip 10.1.1.254


修改网关地址为虚拟 IP 地址

再次 ping

查看交换机 MAC 地址表变化

修改优先级来改变主备路由器

修改 R1

[R1-GigabitEthernet0/0/1]vrrp vrid 1 priority 101


依旧是 R1 位 Master

修改 R2

[R2-GigabitEthernet0/0/1]vrrp vrid 1 priority 102

R2 成功称为 Master，查看交换机 MAC 地址表

# 同网段负载均衡

主备模式实现了一个网关 down 掉后不影响网段内的设备正常通信，但是另外一个设备一直空着，有点浪费，所有的流量都从一台路由器经过，为了使资源利用最大化，配置负载均衡，两台路由器都进行利用，当设备或链路发生故障时还能正常通信

基础配置 R1、R2、R3 一样

#VRRP - 负载均衡

R1 配置

[R1-GigabitEthernet0/0/1]vrrp vrid 1 virtual-ip 10.1.1.254
[R1-GigabitEthernet0/0/1]vrrp vrid 1 priority 101
[R1-GigabitEthernet0/0/1]vrrp vrid 2 virtual-ip 10.1.1.1
[R1-GigabitEthernet0/0/1]vrrp vrid 2 priority 102

R2 配置

[R2-GigabitEthernet0/0/1]vrrp vrid 1 virtual-ip 10.1.1.254
[R2-GigabitEthernet0/0/1]vrrp vrid 1 priority 102
[R2-GigabitEthernet0/0/1]vrrp vrid 2 virtual-ip 10.1.1.1	
[R2-GigabitEthernet0/0/1]vrrp vrid 2 priority 101

<font color="#ff0000">R1</font>
<font color="#ff0000">10.1.1.254 => Backup</font>
<font color="#ff0000">10.1.1.1 => Master</font>

<font color="#ff0000">R2</font>
<font color="#ff0000">10.1.1.254 => Master</font>
<font color="#ff0000">10.1.1.1 =>Backup</font>

修改 PC 网关

当上行链路出现故障时

因为 R1 存在去放 R2 的路由，所以当上行链路出现故障时，它会选择从 R1 的 G0/0/1 接口到 SW1 的 G0/0/1 再去 R2 访问 R3

当下行链路出现故障时

R2 会切换为 Master

# 不同网段负载均衡

基础配置

  1. SW4 配置
vlan batch 10 20

interface GigabitEthernet0/0/1
 port link-type trunk
 port trunk allow-pass vlan 10 20

interface GigabitEthernet0/0/2
 port link-type trunk
 port trunk allow-pass vlan 10 20

interface Ethernet0/0/1
 port link-type access
 port default vlan 10

interface Ethernet0/0/2
 port link-type access
 port default vlan 20
  1. SW2 配置
vlan batch 10 20 192

interface Vlanif10
 ip address 10.1.1.252 255.255.255.0

interface Vlanif20
 ip address 10.1.2.252 255.255.255.0

interface Vlanif192
 ip address 192.168.255.1 255.255.255.252

interface GigabitEthernet0/0/1
 port link-type trunk
 port trunk allow-pass vlan 10 20

interface GigabitEthernet0/0/2
 port link-type access

interface GigabitEthernet0/0/24
 port link-type access
 port default vlan 192
  1. SW3 配置
vlan batch 10 20 192

interface Vlanif10
 ip address 10.1.1.253 255.255.255.0
 vrrp vrid 1 virtual-ip 10.1.1.254
 vrrp vrid 1 priority 101

interface Vlanif20
 ip address 10.1.2.253 255.255.255.0
 vrrp vrid 2 virtual-ip 10.1.2.254
 vrrp vrid 2 priority 102

interface Vlanif192
 ip address 192.168.255.5 255.255.255.252

interface GigabitEthernet0/0/1
 port link-type trunk
 port trunk allow-pass vlan 10 20

interface GigabitEthernet0/0/24
 port link-type access
 port default vlan 192
  1. R4 配置
interface GigabitEthernet0/0/1
 ip address 192.168.255.2 255.255.255.252

interface GigabitEthernet0/0/2
 ip address 192.168.255.6 255.255.255.252

interface LoopBack0
 ip address 100.100.100.100 255.255.255.255

OSPF 配置

  1. R4 配置
ospf 1 router-id 4.4.4.4
 area 0.0.0.0
  network 192.168.255.0 0.0.0.3
  network 192.168.255.4 0.0.0.3
  network 100.100.100.100 0.0.0.0
  1. SW3 配置
ospf 1 router-id 2.2.2.2
 area 0.0.0.0
  network 192.168.255.4 0.0.0.3
  network 10.1.1.0 0.0.0.255
  network 10.1.2.0 0.0.0.255
  1. SW2 配置
ospf 1 router-id 3.3.3.3
 area 0.0.0.0
  network 192.168.255.0 0.0.0.3
  network 10.1.1.0 0.0.0.255
  network 10.1.2.0 0.0.0.255
