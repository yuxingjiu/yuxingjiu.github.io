[!ob-info] 全局选路策略
基于链路带宽负载分担 —— 充分利用链路的带宽

基于链路质量负载分担 —— 为了使用户上网体验最优，FW能够根据链路质量实时地分配流量

基于链路权重负载分担 —— 在保证上网质量的同时，还要保证不对链路造成浪费

  1. 流量经过FW的安全策略匹配顺序？SNAT怎么配置？Easy-IP是什么？

[!ob-info] 安全策略
安全策略匹配顺序是自上而下的进行匹配，先设置的在上面，后设置的在下面，如果一旦匹配成功就会停止往下匹配，根据策略进行下一步操作，如果没有匹配成功，则看默认策略，默认策略一般为 禁止

条件精细的策略放在最上面，条件宽泛的策略放在最下面

[!ob-info] 地址转换
SNAT

源NAT地址转换，用于内网访问外网，数据发出时转换源IP，返回时转换目的IP，实现技术包括 NAPT、Easy-IP

NAPT需要配置一个地址池，这个地址池中的地址是公网IP，且是固定的，如果防火墙是作为出口设备配置的公网IP接口，可以将其IP添加到地址池中

Easy-IP是直接利用自身的公网接口IP地址进行转换，不需要明确自身的接口IP地址是多少，但如果不是作为出口设备，不存在公网IP地址，只能配置NAPT

DNAT

目的NAT地址转换，用于外网访问内网，数据发出时转换目的IP，接收时转换源IP，实现技术Nat Server

Nat-Server是使用一个公网地址来代内网服务器地址

  1. VGMP、HRP、VRRP是什么？有什么作用？相互之间有什么关系？
  1. VGMP是一种虚拟网关协议，可以在多个网关设备之间实现冗余，保证主网关出现故障时，备份网关可以立刻切换流量
  2. VRRP是虚拟路由冗余协议，
  3. HRP是一种热备份路由协议，基于VRRP的扩展协议

VGMP、HRP、VRRP都是网络协议，它们主要用于提高网络的可用性和冗余性。

VGMP（Virtual Gateway Protocol）是一种虚拟网关协议，它可以在多个网关设备之间实现冗余。VGMP可以确保在主网关故障时，备份网关可以立即接管网络流量，从而实现网络的高可用性和冗余性。

HRP（Hot Standby Router Protocol）是一种热备份路由器协议，它用于实现局域网（LAN）中的路由器冗余。HRP可以确保在主路由器故障时，备份路由器可以接管路由器的职责，从而保证网络的连通性和稳定性。

VRRP（Virtual Router Redundancy Protocol）是一种虚拟路由器冗余协议，它可以在多个路由器之间实现冗余。VRRP可以确保在主路由器故障时，备份路由器可以接管路由器的职责，从而实现网络的高可用性和冗余性。

这三种协议都可以实现网络设备的冗余，提高网络的可用性和稳定性。它们之间的关系是，VGMP和VRRP都可以实现路由器的冗余，但VGMP更加适合在较大的网络中使用;而HRP主要用于实现局域网中的路由器冗余。

  1. 策略路由与安全策略的区别

[!ob-info] 策略路由与安全策略
策略路由时用户控制流量的走向，而安全策略是FW控制流量可不可以通过

策略路由即使指定要去网的目的能到达，没有安全策略也是无法达到的

  1. DNS透明代理？

[!ob-info] DNS透明代理
在智能选路中，当配置了选路策略后，由于向一些网站可能在各大运营商都存在对应的服务器，当DNS正好配置了是某一个运营商的DNS服务器时，那么解析回来的IP也是对应运营商的地址，根据策略都会选择同一条链路，造成了一条链路拥挤，一条链路空闲，这时就可以配置一个防火墙指定的的DNS IP地址，当FW发现是该IP发出的DNS请求，就会把流量分别进行转发，这样就避免了上面的问题
