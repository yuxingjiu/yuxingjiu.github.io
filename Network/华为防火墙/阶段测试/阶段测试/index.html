  1. 智能选路的全局选路有哪些方式，举例说明

[!ob-info] 全局选路策略
基于链路带宽负载分担 —— 充分利用链路的带宽

基于链路质量负载分担 —— 为了使用户上网体验最优，FW 能够根据链路质量实时地分配流量

基于链路权重负载分担 —— 在保证上网质量的同时，还要保证不对链路造成浪费

基于链路优先主备备份 —— 当链路各种参数存在很大差异时，可以设置优为主，差为备

可以根据用户指定的策略进行转发

[!ob-info] ISP 路由

  1. 流量经过 FW 的安全策略匹配顺序？SNAT 怎么配置？Easy-IP 是什么？

[!ob-info] 安全策略
安全策略匹配顺序是自上而下的进行匹配，先设置的在上面，后设置的在下面，如果一旦匹配成功就会停止往下匹配，根据策略进行下一步操作，如果没有匹配成功，则看默认策略，默认策略一般为 <mark style="background: #CACFD9A6;"> 禁止 </mark>

<mark style="background: #FF5582A6;"> 条件精细的策略放在最上面，条件宽泛的策略放在最下面 </mark>

[!ob-info] 地址转换
<mark style="background: #ABF7F7A6;">SNAT</mark>

源 NAT 地址转换，用于内网访问外网，数据发出时转换源 IP，返回时转换目的 IP，实现技术包括 <mark style="background: #FF5582A6;">NAPT</mark>、<mark style="background: #FF5582A6;">Easy-IP</mark>

NAPT 需要配置一个地址池，这个地址池中的地址是公网 IP，且是固定的，如果防火墙是作为出口设备配置的公网 IP 接口，可以将其 IP 添加到地址池中

Easy-IP 是直接利用自身的公网接口 IP 地址进行转换，不需要明确自身的接口 IP 地址是多少，但如果不是作为出口设备，不存在公网 IP 地址，只能配置 NAPT

<mark style="background: #ABF7F7A6;">DNAT</mark>

目的 NAT 地址转换，用于外网访问内网，数据发出时转换目的 IP，接收时转换源 IP，实现技术 Nat Server

Nat-Server 是使用一个公网地址来代内网服务器地址

  1. VGMP、HRP、VRRP 是什么？有什么作用？相互之间有什么关系？
  1. VGMP 是一种虚拟网关协议，可以在多个网关设备之间实现冗余，保证主网关出现故障时，备份网关可以立刻切换流量
  2. VRRP 是虚拟路由冗余协议，
  3. HRP 是一种热备份路由协议，基于 VRRP 的扩展协议

VGMP、HRP、VRRP 都是网络协议，它们主要用于提高网络的可用性和冗余性。

VGMP（Virtual Gateway Protocol）是一种虚拟网关协议，它可以在多个网关设备之间实现冗余。VGMP 可以确保在主网关故障时，备份网关可以立即接管网络流量，从而实现网络的高可用性和冗余性。

HRP（Hot Standby Router Protocol）是一种热备份路由器协议，它用于实现局域网（LAN）中的路由器冗余。HRP 可以确保在主路由器故障时，备份路由器可以接管路由器的职责，从而保证网络的连通性和稳定性。

VRRP（Virtual Router Redundancy Protocol）是一种虚拟路由器冗余协议，它可以在多个路由器之间实现冗余。VRRP 可以确保在主路由器故障时，备份路由器可以接管路由器的职责，从而实现网络的高可用性和冗余性。

这三种协议都可以实现网络设备的冗余，提高网络的可用性和稳定性。它们之间的关系是，VGMP 和 VRRP 都可以实现路由器的冗余，但 VGMP 更加适合在较大的网络中使用；而 HRP 主要用于实现局域网中的路由器冗余。

  1. 策略路由与安全策略的区别

[!ob-info] 策略路由与安全策略
策略路由时用户控制流量的走向，而安全策略是 FW 控制流量可不可以通过

策略路由即使指定要去网的目的能到达，没有安全策略也是无法达到的

[!ob-info]
不能，因为路由器工作在三层，而防火墙可以在 2-7 层，路由器的本质是转发，防火墙的本质是控制

  1. 一条流量经过防火墙，匹配的动作是 permit 的安全策略，那么该流量是否一定可以通过？原因？

[!ob-info]
不一定，因为防火墙可能设置了更加详细的策略，而不只是单纯的区域匹配，像只允许 http 或者 http 的经过，而流量是一个 ftp 的，很明显，不让通过

  1. DNS 透明代理？

[!ob-info] DNS 透明代理
在智能选路中，当配置了选路策略后，由于向一些网站可能在各大运营商都存在对应的服务器，当 DNS 正好配置了是某一个运营商的 DNS 服务器时，那么解析回来的 IP 也是对应运营商的地址，根据策略都会选择同一条链路，造成了一条链路拥挤，一条链路空闲，这时就可以配置一个防火墙指定的的 DNS IP 地址，当 FW 发现是该 IP 发出的 DNS 请求，就会把流量分别进行转发，这样就避免了上面的问题
