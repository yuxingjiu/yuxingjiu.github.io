VLAN简介

在典型交换网络中，在某台主机发送一个广播帧或未知单播帧时，该数据帧会被泛洪，甚至传递到整个广播域

广播域越大，产生的网络安全问题、垃圾流量问题就越严重

而虚拟局域网（VLAN）可以隔离广播域（不受地域限制、同一VLAN内的设备才能进行二层通信）

特点

  1. 一个VLAN就是一个广播域，所以在同一个VLAN内部，计算机可以直接进行二层通信，而不同的VLAN内的计算机，无法直接进行二层通信，只能进行三层通信来传递信息，即广播报文被限制在一个VLAN内，现实网络中一个VLAN、一个部门、一个网段，不同部门、不同VLAN、不同网段
  2. VLAN的划分不受地域的限制

优势

  1. 灵活构建虚拟虚拟工作组，用VLAN划分不同的用户到不同的工作组，同一工作组的用户也不必局限于某一固定的物理范围，网络构建维护更加方便
  2. 限制广播域（隔离广播域），广播域被限制在一个VLAN内，节省了带宽，提高了网络处理能力
  3. 增强局域网的安全性，不同VLAN内的报文在传输时是相互隔离的，即一个VLAN内的用户不能和其他VLAN内的用户直接通信
  4. 提高了网络的健壮性，故障被限制在一个VLAN内，本VLAN内的故障不会影响其他VLAN的正常工作

VLAN 原理

VLAN 标签

要使交换机能够分辨不同VLAN的报文，需要在报文中添加标识VLAN信息的字段，IEE 802.1Q协议规定，在以太网数据帧中加入4个字节的VLAN标签，又叫VLAN Tag简称Tag，满足这种格式的数据帧称为IEEE 802.1Q数据帧，也称为VLAN数据帧

VLAN数据帧

在一个VLAN交换网络中，以太网帧主要有两种形式

  1. 有标记帧（Tagged帧）IEE 802.1Q协议规定，在以太网数据帧的目的MAC和源MAC地址字段后，协议烈性字段前加入4个字节的VLAN标签的数据帧
  2. 无标记帧（Untagged帧），原始的、未加入4字节VLAN标签的数据帧（计算机无法识别Tagged数据帧，因此计算机处理和发出的都是Untagged数据帧，为了提高处理效率，交换机内部处理的数据帧一律都是Tagged帧）

VLAN数据帧中的主要字段

  1. TPID ：2Byte（标签协议标识符），表示数据帧类型，取值0x8100时表示IEEE 802.1Q的VLAN数据帧
  2. PRI ：3bit（数据帧的优先级），用于QoS，取值范围0~7，值越大越优先（网络阻塞时，交换机优先发送优先级更高的数据帧）
  3. CFI ：1bit（标准格式指示位），表示MAC地址在不同的传输介质中是否以标准格式进行封装，用于兼容以太网和令牌环网（CFI取值为0表示MAC地址以标准格式进行封装，为1表示以非标准格式封装，在以太网中，CFI的值为0）
  4. VID ：12bit（VLAN ID）表示该数据帧所属的VLAN编号（VLAN ID的取值范围是04095，由于0和4095为协议保留取值，所以VLAN ID 有效取值范围是14094）

划分方式

  1. 基于接口

  2. 根据交换机的接口来划分VLAN

  3. 网络管理员预先给叫花鸡的每个接口配置不同的PVID，讲解欧划入PVID对应的VLAN

  4. 当一个数据帧进入交换机时，没有带VLAN标签，该数据帧就会被打伤接口指定PVID的Tag，然后数据帧将在指定PVID中传输

  5. 缺省VLAN1，PVID

  6. Port VLAN ID，是接口上的缺省VLAN

  7. PVID默认是根据接口类型来区分的，Access接口PVID=VLAN ID，在Trunk接口PVID默认=1，在Hybird接口PVID默认是1

  8. 取值1~4094

  9. 基于MAC地址

  10. 基于IP子网划分

  11. 基于协议划分

  12. 基于策略

接口类型

Access接口

交换机上常用来连接用户PC、服务器等终端设备的接口（不能识别Tag的用户终端），Access接口所连接的这些设备的网卡往往只收发无标记帧，Access接口只能加入一个VLAN

接收帧

接收到的是Untagged帧

接收该数据帧，并打上改接口的PVID的Tag

接收到的是Tagged帧

当该帧的VLAN ID与该接口的PVID相同时，接收该帧

当该镇的VLAN ID与该接口的PVID不同时，丢弃该帧

发送帧

帧的VLAN ID与接口PVID相同

先剥离该帧的Tag，然后再将其从该接口转发

帧的VLAN ID与接口PVID不同

禁止该帧从接口发出

Trunk接口

Trunk接口允许多个VLAN的数据帧通过，这些数据帧通过802.1Q、Tag实现区分，Trunk接口常用于交换机之间的互联，也用于连接路由器、防火墙等设备的子接口（可以同时收发Tagged帧和Untagged帧的终端）

接收帧

接收到的是Untagged帧

该帧打上改接口的PVID的Tag，当PVID在该接口允许通过的VLAN列表里时接收该帧，若不在，丢弃该帧

接收到的是Tagged帧

当该帧的VLAN ID在该接口允许通过的VLAN列表里时，接收该帧，否则丢弃该帧

发送帧

帧的VLAN ID与接口PVID相同

当该帧的VLAN ID在该接口允许通过的VLAN列表中，则该帧的Tag剥离，然后将其从该接口发送，如果VLAN ID不在雷暴中，则禁止将该帧从该接口转发出去

帧的VLAN ID与接口PVID不同

当该帧的VLAN ID在该接口允许通过的VLAN列表中，保留该帧的Tag，将其从该接口转发，如果VLAN ID不在允许通过的VLAN列表中则禁止该帧从接口转发出去

Hybird接口

Hybrid接口与Trunk接口类似，也允许多个VLAN的数据帧通过，这些数据帧通过802.1Q、Tag实现区分，用户可以灵活指定Hybrid接口在发送某个（或某些）VLAN的数据帧时是否携带Tag（华为设备默认接口类型是Hybrid）

接收帧

接收到的是Untagged帧

打上PVID，当PVID在该接口允许通过的VLAN列表时接收该帧，当PVID不在允许通过的VLAN列表时，丢弃该帧

接收到的是Tagged帧

当该帧的VLAN ID在该接口允许通过的VLAN列表里时，接收该帧，否则丢弃该帧

发送帧

帧的VLAN ID是该接口允许剥离Tag通过的VLAN ID

当管理员通过命令设置发送该VLAN的帧时不携带Tag，则将该帧的Tag剥除，然后将其从接口发送出

帧的VLAN ID是该接口允许携带Tag通过的VLAN ID

当管理员通过命令设置发送该VLAN的帧时携带Tag，则保留该帧的Tag，然后将其从该接口发送出去

对于Hybrid接口，除了要配置PVID外，还存在两个允许通过的VLAN ID列表，一个是Untagged VLAN ID列表，另一个是Tagged VLAN ID列表，其中VLAN 1默认在Untagged VLAN 列表中，这两个允许通过列表中的所有VLAN的帧都是允许通过这个Hybrid接口的

其中Tagged VLAN ID列表可以通过display port vlan查看的
