# VLAN 简介

在典型交换网络中，在某台主机发送一个广播帧或未知单播帧时，该数据帧会被泛洪，甚至传递到整个广播域

广播域越大，产生的网络安全问题、垃圾流量问题就越严重

而虚拟局域网（VLAN）可以隔离广播域（不受地域限制、同一 VLAN 内的设备才能进行二层通信）

# 特点

  1. 一个 VLAN 就是一个广播域，所以在同一个 VLAN 内部，计算机可以直接进行二层通信，而不同的 VLAN 内的计算机，无法直接进行二层通信，只能进行三层通信来传递信息，即广播报文被限制在一个 VLAN 内，现实网络中一个 VLAN、一个部门、一个网段，不同部门、不同 VLAN、不同网段
  2. VLAN 的划分不受地域的限制

# 优势

  1. 灵活构建虚拟虚拟工作组，用 VLAN 划分不同的用户到不同的工作组，同一工作组的用户也不必局限于某一固定的物理范围，网络构建维护更加方便
  2. 限制广播域（隔离广播域），广播域被限制在一个 VLAN 内，节省了带宽，提高了网络处理能力
  3. 增强局域网的安全性，不同 VLAN 内的报文在传输时是相互隔离的，即一个 VLAN 内的用户不能和其他 VLAN 内的用户直接通信
  4. 提高了网络的健壮性，故障被限制在一个 VLAN 内，本 VLAN 内的故障不会影响其他 VLAN 的正常工作

# VLAN 原理

# VLAN 标签

要使交换机能够分辨不同 VLAN 的报文，需要在报文中添加标识 VLAN 信息的字段，IEE 802.1Q 协议规定，在以太网数据帧中加入 4 个字节的 VLAN 标签，又叫 VLAN Tag 简称 Tag，满足这种格式的数据帧称为 IEEE 802.1Q 数据帧，也称为 VLAN 数据帧

# VLAN 数据帧

在一个 VLAN 交换网络中，以太网帧主要有两种形式

  1. 有标记帧（Tagged 帧）IEE 802.1Q 协议规定，在以太网数据帧的目的 MAC 和源 MAC 地址字段后，协议烈性字段前加入 4 个字节的 VLAN 标签的数据帧
  2. 无标记帧（Untagged 帧），原始的、未加入 4 字节 VLAN 标签的数据帧（计算机无法识别 Tagged 数据帧，因此计算机处理和发出的都是 Untagged 数据帧，为了提高处理效率，交换机内部处理的数据帧一律都是 Tagged 帧）

VLAN 数据帧中的主要字段

  1. TPID ：2Byte（标签协议标识符），表示数据帧类型，取值 0x8100 时表示 IEEE 802.1Q 的 VLAN 数据帧
  2. PRI ：3bit（数据帧的优先级），用于 QoS，取值范围 0~7，值越大越优先（网络阻塞时，交换机优先发送优先级更高的数据帧）
  3. CFI ：1bit（标准格式指示位），表示 MAC 地址在不同的传输介质中是否以标准格式进行封装，用于兼容以太网和令牌环网（CFI 取值为 0 表示 MAC 地址以标准格式进行封装，为 1 表示以非标准格式封装，在以太网中，CFI 的值为 0）
  4. VID ：12bit（VLAN ID）表示该数据帧所属的 VLAN 编号（VLAN ID 的取值范围是 0~4095，由于 0 和 4095 为协议保留取值，所以 VLAN ID 有效取值范围是 1~4094）

# 划分方式

  1. 基于接口

  2. 根据交换机的接口来划分 VLAN

  3. 网络管理员预先给叫花鸡的每个接口配置不同的 PVID，讲解欧划入 PVID 对应的 VLAN

  4. 当一个数据帧进入交换机时，没有带 VLAN 标签，该数据帧就会被打伤接口指定 PVID 的 Tag，然后数据帧将在指定 PVID 中传输

  5. 缺省 VLAN1，PVID

  6. Port VLAN ID，是接口上的缺省 VLAN

  7. PVID 默认是根据接口类型来区分的，Access 接口 PVID=VLAN ID，在 Trunk 接口 PVID 默认 = 1，在 Hybird 接口 PVID 默认是 1

  8. 取值 1~4094

  9. 基于 MAC 地址

  10. 基于 IP 子网划分

  11. 基于协议划分

  12. 基于策略

# 接口类型

# Access 接口

交换机上常用来连接用户 PC、服务器等终端设备的接口（不能识别 Tag 的用户终端），Access 接口所连接的这些设备的网卡往往只收发无标记帧，Access 接口只能加入一个 VLAN

# 接收帧

接收到的是 Untagged 帧

接收该数据帧，并打上改接口的 PVID 的 Tag

接收到的是 Tagged 帧

当该帧的 VLAN ID 与该接口的 PVID 相同时，接收该帧

当该镇的 VLAN ID 与该接口的 PVID 不同时，丢弃该帧

# 发送帧

帧的 VLAN ID 与接口 PVID 相同

先剥离该帧的 Tag，然后再将其从该接口转发

帧的 VLAN ID 与接口 PVID 不同

禁止该帧从接口发出

# Trunk 接口

Trunk 接口允许多个 VLAN 的数据帧通过，这些数据帧通过 802.1Q、Tag 实现区分，Trunk 接口常用于交换机之间的互联，也用于连接路由器、防火墙等设备的子接口（可以同时收发 Tagged 帧和 Untagged 帧的终端）

# 接收帧

接收到的是 Untagged 帧

该帧打上改接口的 PVID 的 Tag，当 PVID 在该接口允许通过的 VLAN 列表里时接收该帧，若不在，丢弃该帧

接收到的是 Tagged 帧

当该帧的 VLAN ID 在该接口允许通过的 VLAN 列表里时，接收该帧，否则丢弃该帧

# 发送帧

帧的 VLAN ID 与接口 PVID 相同

当该帧的 VLAN ID 在该接口允许通过的 VLAN 列表中，则该帧的 Tag 剥离，然后将其从该接口发送，如果 VLAN ID 不在雷暴中，则禁止将该帧从该接口转发出去

帧的 VLAN ID 与接口 PVID 不同

当该帧的 VLAN ID 在该接口允许通过的 VLAN 列表中，保留该帧的 Tag，将其从该接口转发，如果 VLAN ID 不在允许通过的 VLAN 列表中则禁止该帧从接口转发出去

# Hybird 接口

Hybrid 接口与 Trunk 接口类似，也允许多个 VLAN 的数据帧通过，这些数据帧通过 802.1Q、Tag 实现区分，用户可以灵活指定 Hybrid 接口在发送某个（或某些）VLAN 的数据帧时是否携带 Tag（华为设备默认接口类型是 Hybrid）

# 接收帧

接收到的是 Untagged 帧

打上 PVID，当 PVID 在该接口允许通过的 VLAN 列表时接收该帧，当 PVID 不在允许通过的 VLAN 列表时，丢弃该帧

接收到的是 Tagged 帧

当该帧的 VLAN ID 在该接口允许通过的 VLAN 列表里时，接收该帧，否则丢弃该帧

# 发送帧

帧的 VLAN ID 是该接口允许剥离 Tag 通过的 VLAN ID

当管理员通过命令设置发送该 VLAN 的帧时不携带 Tag，则将该帧的 Tag 剥除，然后将其从接口发送出

帧的 VLAN ID 是该接口允许携带 Tag 通过的 VLAN ID

当管理员通过命令设置发送该 VLAN 的帧时携带 Tag，则保留该帧的 Tag，然后将其从该接口发送出去

对于 Hybrid 接口，除了要配置 PVID 外，还存在两个允许通过的 VLAN ID 列表，一个是 Untagged VLAN ID 列表，另一个是 Tagged VLAN ID 列表，其中 VLAN 1 默认在 Untagged VLAN 列表中，这两个允许通过列表中的所有 VLAN 的帧都是允许通过这个 Hybrid 接口的

其中 Tagged VLAN ID 列表可以通过 display port vlan 查看的
