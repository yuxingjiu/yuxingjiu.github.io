# 网络可靠性

网络的可靠性指当设备或者链路出现单点或者多点故障时保证网络服务不间断的能力，网络的可靠性可以从单板、设备、链路多个层面实现

# 提升链路带宽

设备之间存在多条链路时，由于 STP 的存在，实际只会有一条链路转发流量，设备间链路带宽无法得到提升

而以太网链路聚合（Eth-Trunk，简称链路聚合），通过将多个物理接口捆绑成一个逻辑接口，这样可以在不进行硬件升级的条件下，达到增加链路带宽的目的

# 基本术语

<font color="#00b050"> 聚合组 </font> ：若干条链路捆绑在一起形成的逻辑链路，每个聚合组唯一对应着一个逻辑接口，这个逻辑接口又称为链路聚合接口或 Eth-Trunk 接口

<font color="#00b050"> 成员接口和成员链路 </font> ：组成 Eth-Trunk 接口的各个物理接口称为成员接口，成员接口对应的链路称为成员链路

<font color="#00b050"> 活动接口和活动链路 </font> ：活动接口又叫选中（Selected）接口，是参与数据转发的成员接口，活动接口对应的链路称为活动链路（Active link）

<font color="#00b050"> 非活动接口和非活动链路 </font> ：又叫非选中（Unselected）接口，是不参与转发数据的成员接口，非活动接口对应链路称为非活动链路（Inactive link）

<font color="#00b050"> 聚合模式 </font> ：根据是否开启 LACP（链路聚合控制协议），链路聚合可以分为手工模式和 LACP 模式

<font color="#00b050"> 其他概念 </font> ：活动接口上限阈值和活动接口下线阈值

链路聚合接口可以作为普通的以太网接口来使用，与普通以太网接口的差别在于，转发的时候链路聚合组需要从成员接口中选择一个或者多个接口来进行数据转发

一个聚合组内要求成员接口一下参数相同

  1. 接口速率
  2. 双工模式
  3. VLAN 的配置（接口类型都是 Trunk 或 Access，如果为 Access 接口的 default VLAN 需要一致，如果为 Trunk 接口，接口放行的 VLAN、缺省 VLAN 需要一致，聚合接口的配置在聚合口下整体配置，不需要在单独物理接口下配置）

# 链路聚合模式

# 手工模式

  • 运用在设备老旧、低端、不支持 LACP 协议，当所有接口状态为 up 时，都是支持转发，左右两端最多 8 个连接

  1. 手工模式 ：Ethe-Trunk 的建立、成员接口的加入均由手动配置、双方系统之间不适用 LACP 进行协商
  2. 正常情况下所有链路都是活动链路，该模式下所有活动链路都参与数据的转发，平局分担流量，如果某条活动链路故障，链路聚合组自动在剩余的活动链路中平均分摊流量
  3. 当聚合的两端设备中存在一个不支持 LACP 协议时，可以使用手工模式

# 缺陷

  1. 当成员接口接入到其他交换机中


无法识别误连的错误，导致数据丢失，手工模式下，设备间没有报文交互，因此只能通过管理员人工确认

为了是链路聚合接口正常工作，必须保证本端链路聚合接口中所有成员机接口的对端接口，属于同一设备、加入同一链路聚合接口

  1. 当对端接口某些协议发生错误不能接收

SW2 无法通知 SW1 接口 down 掉，导致数据丢失，设备只能通过物理层状态判断对端接口是否正常工作

# LACP 模式

LACP 模式，采用 LACP 协议的一种链路聚合模式，设备间通过链路聚合控制协议数据单元（LACPDU）进行交互，通过协议商确保对端是同一台设备、同一个聚合接口的成员接口

# LACPDU

LACPDU 报文中包含设备优先级、MAC 地址、接口优先级、接口号等

# 系统优先级

LACP 模式下，两端设备所选择的活动接口数目必须保持一致，否则链路聚合组就无法建立，此时可以使其一端称为主动端，另一端为被动端，根据主动端选择活动接口，接口速率一样，通过系统 LACP 优先级确定主动端，值越小优先级越高

# 接口优先级

选出主动端后，两端都会以主动端的接口优先级来选择活动接口，优先级高的接口将优先被选为活动接口，接口 LACP 优先级值越小，优先级越高

# 最大活动接口数

LACP 模式支持配置最大活动接口数目，当成员接口数目超过最大活动接口数目时，会通过比较接口优先级、接口号选举出较优的接口称为活动接口，其余称为备份端口（非活动接口），同时对应的链路分别称为活动链路、非活动链路，交换机只会从活动接口中发送，接收报文（最多同时 8 条传输数据）


当活动链路中出现链路故障时，可以从非活动链路中找出一条优先级最高（接口优先级、接口编号比较）的链路替换故障链路，实现总体宽带不发生变化，业务的不间断转发

# 活动链路的选举

  1. 主动端的选举

SW1、SW2 配置 LACP 模式的链路聚合，两端都设置最大活跃接口数为 2

通过 LACPDU 选举出优先级较高的交换机 SW1 作为 LACP 协商过程的主动端


2. 活动接口的选举

SW1 在本端通过比较接口优先级、接口编号选举出活动接口，其中 1、2 号接口在相同的接口优先级下拥有更小的接口编号，成为活动接口


SW1 通过 LACPDU 将本端活动端口选举结果告知对端

LACP 通过 LACPDU 中的三个 flags 来标识该端口的状态，如果是活跃端口三个 flags 值为 1，如果是非活跃端口，flags 字段值为 0（Synchronization、Collecting、Distributing）

  1. 活动链路的选举

SW2 根据 SW1 的选举结果，明确本端的活动接口，同时对应的链路称为活动链路，至此，Eth-Trunk 的活动链路选举过程完成
