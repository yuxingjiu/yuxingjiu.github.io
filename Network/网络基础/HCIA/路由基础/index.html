路由简介

  1. 路由是指我们报文转发的路径信息，通过路由可以确认IP报文的路径
  2. 路由设备是一句路由转发报文到目的网段的网络设备（路由器、防火墙、三层交换机等）
  3. 路由设备维护着一张路由表保存着路由信息

路由器 => 路由表 => 路由 (路由是路由器转发数据报文的唯一参考依据)

其中路由包括 ：目的网络、掩码、出接口、下一跳IP

路由获取

直连路由（direct）

  • 直连路由自动产生，会把接口地址所属网段以直连路由的形式加入路由器，但并不是所有接口生成的直连路由都会出现在路由表中
  • 前提是该接口的物理状态up（网线两端插上）状态协议都为up（配置IP地址）

查看直连路由

静态路由（static）

  • 静态路由由管理员手动配置，配置方便，对系统要求低，但不能自适应网络拓扑变化

静态路由配置

ip route-static 目的IP段 掩码 下一跳IP
ip route-static 192.168.2.0 255.255.255.0 10.1.1.2

RTA配置

ip route-static 10.1.1.0 255.255.255.0 10.0.0.2

RTC配置

ip route-static 10.0.0.0 255.255.255.0 10.1.1.2

缺省路由

缺省路由一般用于企业网络出口，配置一条缺省路由让出口设备能够转发前往Internet上任意地址的IP报文，整个拓扑中，缺省路由只能往一个方向指（要么进方向，要么出方向），要是不在同一方向很可能会出现环

ip route-static 0.0.0.0 0 下一跳地址
ip route-static 0.0.0.0 0 1.2.3.254

R1和R2都配置了缺省路由，正常情况下，PC1 ping PC2是正常的

但是PC1 ping 其他网段会发生什么？

直到TTL变为0，数据包丢弃才停止

等价路由

来源相同、开销相同的路由都会被加入路由表，形成的路由为等价路由（两条路由条目指向的目的网段相同、但是具有不同的下一跳地址），路由转发会将流量分不到多条路径上

路由表，其中框柱的是等价路由

而当路由表中存在等价路由后，前往该目的网段的IP报文，路由器会通过所有的有效的接口、下一跳转发这种转发行为称为负载分担

负载分担

在负载分担中有两种模式，数据流和数据包进行负载分担（在路由器中默认基于数据流进行负载分担）

  1. 基于数据流，会随机选择一条路由条目，只要选定了路哟条目就不会发生改变

[!info] 配置路由器基于数据流（默认基于数据流，默认情况直接设置会报错）
[Huawei] load-balance flow

只能抓到一条路由条目上的数据包

  1. 基于数据包，会发生改变，每条路由条目都可能会发送数据包

[!info] 配置路由器基于数据包
[Huawei] load-balance packet

改变设置后，两条路由条目都可以抓到数据包

浮动路由

  1. 静态路由支持配置时手动指定优先级，可以通过配置目的地址/掩码相同、优先级不同、下一跳不同的静态路由，实现转发路径的备份
  2. 浮动路由是主用路由的备份，保证链路故障时提供备份路由，主用路由下一跳可达时该备份路由不会出现在路由表
ip route-static 目的网段 掩码 下一跳地址 preference 数值
ip route-static 目的网段 掩码  preference 70

此时只剩下一条主路由，当这条路由出现故障时（down掉），会使用浮动路由

动态路由

ospf(开放式最短路径优先)协议是使用场景非常广泛的冬天路由协议之一，是一种基于链路状态算法的路由协议

  1. 静态路由是通过手动配置和维护的路由条目命令行简单使用小型或稳定的网络
  2. 但随着设备的数量增加，配置量急剧增加，网络发生变化无法自动收敛网络，需要手动修改

OSPF构成

  1. LSA泛洪

运行链路状态路由协议的路由器之间首先会建立一个协议的邻居关系，然后彼此之间交换LSA（酷游状态通信/信息）即每台路由器都产生一个描述自己连接口状态（接口的开销、与邻居路由器之间的关系）开销和跨带有关联，带宽越大，开销越小，反之越大

  1. LSDB组件

每台路由器都会产生LSA，路由器将接收到的LSA发送到自己的LSDB（链路状态数据库）路由器通过LSDB掌握了全网的拓扑

  1. SPF计算

每台路由器都计算出一棵以自己为根的无环的、拥有最短路径的树，而最短路径计算（SPF计算）是根据cost值来计算的

  1. 路由表生成

每台路由器根据SPF计算结果，将路由器加载入路由表

OSPF基础术语

  1. Router-ID

路由器标识符，用于在一个OSPF区域中唯一地标识一台路由器，可以通过手工配置的方式或使用系统自动配置

  1. OSPF-area

用于标识一个OSPF区域，区域是从逻辑上划分为不同的组，每个组用区域号（Area ID）来标识
Area 0 是骨干区域，是必备的

  1. 度量值

OSPF使用cost错位度量值，cost值=100mb/s/÷接口带宽，若计算结果为小数，直接取整（1.9=1）若为0.000001，按1处理

一条OSPF路由的cost值可以理解为是从目的网段到本路由器沿途所有接入口的cost累加

OSPF三大表项

OSPF有三张重要的表项，OSPF邻居表、LSDB表、OSPF路由表

OSPF邻居表

OSPF在传递链路状态信息之前，先建立OSPF邻居关系（邻居关系建立通过交互hello报文建立，邻居表显示了OSPF路由器之间的邻居状态）

display ospf peer // 查看邻居表

LSDB表

LSDB会保存自己产生的及从邻居收到的LSA信息（Type标识LSA类型、AdvRouter标识发送LSA的路由器）

display ospf lsdb // 查看LSDB表

OSPF路由表

OSPF路由表和路由器路由表是两张不同的表项（OSPF路由表包含Destination、Cost、NextHop等指导转发的信息）

display ospf routing // 查看OSPFF路由表

OSPF五大报文

Hello

周期性发送（10s发送一次），用来发现和维护OSPF邻居关系

Database Description(DD)

描述本地LSDB摘要信息，用于两台设备之间进行数据库同步

Link State Request

用于向对方其你去所需要的LSA，设备只有在OSPF邻居双方成功交换DD报文后才会向对方发送LSR报文

Link State Update

用于向对方发送所需的LSA

Link State ACK

用来对收到的LAS进行确认

OSPF七大状态

Down

失效 ：该状态还没有进行Hello报文的发送，一旦进行Hello报文发送就会进入下一状态

Init

初始化 ：接收到的Hello包并不存在本地的Router-ID，但是将发送该Hello包的邻居设置为Init，此时双方还没有建立双向通信，随着双方互相发送Hello报文，于是才会进入下一状态

2-Way

双向通信 ：邻居关系和邻接关系的分界点，接收到的Hello报文中存在本地的Router-ID则双方就会进入2-way（双向通信）状态，该状态也是OSPF最稳定的状态之一，标志着邻居关系的建立，也是邻接关系建立的基础

Exstart

预启动 ：使用类似Hello的DD报文进行主从关系（Master、Slave）选举，Router-ID数值大德为主，优先进入下一状态，此时DD报文是空的

Exchange

准交换 ：使用真正的DD报文进行数据库目录的共享

Loading

加载 ：使用LSR、LSU、LSACK去获取未知的LSA信息（LSDB同步）

Full

全毗邻 ：邻接关系建立的标志

主从选举

MA多路网络访问有两种类型 ：广播型多路访问网络（BMA）和非广播型多路访问网络（NBMA），以太网是一种典型的广播型多路访问网络

在MA网络中如果每台OSPF路由器都与其他的所有路由器建立邻接关系，便会导致网络中存在过多的OSPF邻接关系增加设备负担，也增加了网路中的泛洪的OSPF报文数量，当拓扑发生变化，网络中的LSA泛洪会造成带宽的浪费和设备资源的损耗

为了优化MA网络中OSPF邻接关系，OSPF指定了三种路由器身份，DR（指定路由器）、BDR（备用指定路由器）、DRouter路由器

只允许DR和BDR与其他OSPF路由器建立邻接关系DRouter之间不会建立全毗邻（Full）的OSPF邻接关系，只会停留在2-way状态，BDR会监控DR状态，若当前DR发生故障时，会重新选出新的DR和BDR

选举规则

  1. OSPF DR优先级更高的称为该MA的DR，若优先级相同，则具有更高级的Router-ID的路由器（接口）称为DR，且DR具有非抢占性（若OSPF建立后又连入一个比当前DR的Router-ID更高的路由器，此时新连入的路由器不会抢占当前DR，只是一个普通的DRouter，只有当前DR发生故障重新选举DR时，才会是DR，那故障的DR复原后变为普通的DRouter路由器）
  2. Router-ID选择顺序（手动和自动）
    若没有手动配置Router-ID，则会从loopback地址选取最大，若没有配置，则从接口地址中选出最大IP作为设备的ID号
