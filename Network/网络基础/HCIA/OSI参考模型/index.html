# 网络通信协议

# 协议

决定数据的格式和传输的一组规则

# 协议分层设计的意义

简化了相关的网络操作，提供了不同厂商之间的兼容性，各个层次独立，一层的变化不会影响到邻居

# 协议标准化组织

IETF ：国际互联网工程任务组 —— TCP/IP 协议

IEEE ：电子和电子工程师协会 —— Ethernet、WIFI

ISO ：国际标准化组织 —— OSI 模型

# 应用层

对应用程序提供接口（任何机器交流的接口），是应用程序能够使用网络服务应用层协议会指定使用响应的传输层协议以及传输层使用的端口等，<font color="#00b050"> 单位（PDU）为数据（Data）</font>

# 表示层

对数据格式的转换，以确保一个系统生成的应用层数据能够被另一系统的应用层识别和理解

# 会话层

在通信双方之间建立、管理和终止会话

# 传输层

接收来自应用层协议的数据封装上响应的传输层头部帮助其建立端到端的连接，<font color="#00b050"> 单位为段（Segment）</font>

# TCP/UDP 协议

TCP ：传输控制协议
UDP ：用户数据报协议

# TCP/UDP 报文

  1. TCP 报文

TCP 控制位

  1. URG ：紧急指针有效

  2. ACK ：确认序列号（为了与确认号 ack [三次握手] 区分开），大写表示

  3. PSH ：接收方尽快将这个报文交给应用层

  4. SYN ：发起一个新连接

  5. FIN ：释放一个新连接

  6. UDP 报文

# TCP/UDP 端口

  1. 客户端使用的源端口一般随机分配，目标端口则由服务器应用指定
  2. 源端口号一般为系统中未使用的，且大于 1023
  3. 目的端口号为服务端开启的应用（服务）所侦听的端口（像 HTTP 对应 80）

# TCP 三次握手

  1. 由 TCP 连接发起方（PC1）发送第一个 SYN 位置 1 的 TCP 报文（初始序列号 a 为一个随机生成的数字，因为没收到来自 PC2 的任何报文，所以确认序列号为 0）
  2. 接收方（PC2）接收到合法的 SYN 报文之后，回复一个 SYN 和 ACK 置 1 的 TCP 报文（初始序列号 b 为一个随机生成的数字，同时因为此报文是回复给 PC1 的报文，所以确认序列号为 a+1）
  3. PC1 收到 PC2 发送的 SYN 和 ACK 置位的 TCP 报文后，回复一个 ACK 置位的报文（此时序列号为 a+1，确认号为 b+1）
  4. PC2 收到后，TCP 双向连接建立

# TCP 四次挥手

  1. 由 PC1 发出一个 FIN 字段置 "1" 的不带数据的 TCP 段
  2. PC2 收到 PC1 发来的 FIN 置位的 TCP 报文后，回复一个 ACK 置位的 TCP 报文
  3. 若 PC2 也没有需要发送的数据，则直接发送 FIN 置位的 TCP 报文，假设此时 PC2 还有数据要发送，name 当 PC2 发送完这些数据后，会发送一个 FIN 置位的 TCP 报文去关闭连接
  4. PC1 收到 FIN 置位的 TCP 报文，回复 ACK 报文，TCP 双向连接断开

# 网络层

负责数据从一台主机到另一台主机之间的传递，<font color="#00b050"> 单位为包（Packet）</font>

# IP 协议

网络层经常被称为 IP 层，但网络层协议并不只是 IP 协议，还有 ICMP 协议等等

Internet Protocol 本身是一个协议文件的名称，该协议文件的内容非常少，主要是定义并阐述 IP 报文的格式（经常被提及的 IP，一般不是特指 Internet Protocol 这个协议文件本身，而是泛指直接或间接与 IP 协议相关的任何内容）

# IP 报文

# 协议号

  1. IP 报文头中的协议号字段标识了将会继续处理该报文的协议
  2. 即指出此数据包携带的数据使用何种协议，以便目的主机的 IP 层将数据部分上报给哪个进程处理

# ICMP 协议

Internet 控制消息协议 ICMP 时 IP 协议的辅助协议

ICMP 协议用来在网络设备间传递各种差错和控制信息，对于收集各种网络信息、诊断和排除各种网络故障等方面起着至关重要的作用

# 数据链路层

数据链路层向网络层提供网内通信，负责组帧、物理编址、差错控制等功能，可以向网络层的 IP、IPV6 等协议提供服务，<font color="#00b050"> 单位为帧（Frame）</font>

# ARP 协议

  • 网络层和数据链路层之间

根据已知的对方 IP 地址获取其对应的 MAC 地址

# 报文格式

  1. DMAC 目的 MAC，IPV4 为 6 字节，用于标识帧的接收者
  2. SMAC 源 MAC，IPV4 为 6 字节，用于标识帧的发送者
  3. Type 协议类型（0x0800 表示数据包，0x0806 表示 ARP 数据包）2 字节
  4. FCS 4 字节，用于数据校验

# 物理层

根据物理介质的不同将数字信号转为光信号、电信号或电磁波信号，单位为比特流（Bit stream）

常见的介质有 ：双绞线、光纤、电磁波等
