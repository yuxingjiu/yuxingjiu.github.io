网络通信协议

协议

决定数据的格式和传输的一组规则

协议分层设计的意义

简化了相关的网络操作，提供了不同厂商之间的兼容性，各个层次独立，一层的变化不会影响到邻居

协议标准化组织

IETF ：国际互联网工程任务组 —— TCP/IP协议

IEEE ：电子和电子工程师协会 —— Ethernet、WIFI

ISO ：国际标准化组织 —— OSI模型

网络参考模型

应用层

对应用程序提供接口（任何机器交流的接口），是应用程序能够使用网络服务应用层协议会指定使用响应的传输层协议以及传输层使用的端口等，单位（PDU）为数据（Data）

表示层

对数据格式的转换，以确保一个系统生成的应用层数据能够被另一系统的应用层识别和理解

会话层

在通信双方之间建立、管理和终止会话

传输层

接收来自应用层协议的数据封装上响应的传输层头部帮助其建立端到端的连接，单位为段（Segment）

TCP/UDP协议

TCP ：传输控制协议
UDP ：用户数据报协议

TCP/UDP报文

  1. TCP报文

TCP控制位

  1. URG ：紧急指针有效

  2. ACK ：确认序列号（为了与确认号ack[三次握手]区分开），大写表示

  3. PSH ：接收方尽快将这个报文交给应用层

  4. SYN ：发起一个新连接

  5. FIN ：释放一个新连接

  6. UDP报文

TCP/UDP端口

  1. 客户端使用的源端口一般随机分配，目标端口则由服务器应用指定
  2. 源端口号一般为系统中未使用的，且大于1023
  3. 目的端口号为服务端开启的应用（服务）所侦听的端口（像HTTP对应80）

TCP三次握手

  1. 由TCP连接发起方（PC1）发送第一个SYN位置1的TCP报文（初始序列号a为一个随机生成的数字，因为没收到来自PC2的任何报文，所以确认序列号为0）
  2. 接收方（PC2）接收到合法的SYN报文之后，回复一个SYN和ACK置1的TCP报文（初始序列号b为一个随机生成的数字，同时因为此报文是回复给PC1的报文，所以确认序列号为a+1）
  3. PC1收到PC2发送的SYN和ACK置位的TCP报文后，回复一个ACK置位的报文（此时序列号为a+1，确认号为b+1）
  4. PC2收到后，TCP双向连接建立

TCP四次挥手

  1. 由PC1发出一个FIN字段置”1”的不带数据的TCP段
  2. PC2收到PC1发来的FIN置位的TCP报文后，回复一个ACK置位的TCP报文
  3. 若PC2也没有需要发送的数据，则直接发送FIN置位的TCP报文，假设此时PC2还有数据要发送，name当PC2发送完这些数据后，会发送一个FIN置位的TCP报文去关闭连接
  4. PC1收到FIN置位的TCP报文，回复ACK报文，TCP双向连接断开

网络层

负责数据从一台主机到另一台主机之间的传递，单位为包（Packet）

IP协议

网络层经常被称为IP层，但网络层协议并不只是IP协议，还有ICMP协议等等

Internet Protocol本身是一个协议文件的名称，该协议文件的内容非常少，主要是定义并阐述IP报文的格式（经常被提及的IP，一般不是特指Internet Protocol这个协议文件本身，而是泛指直接或间接与IP协议相关的任何内容）

IP报文

协议号

  1. IP报文头中的协议号字段标识了将会继续处理该报文的协议
  2. 即指出此数据包携带的数据使用何种协议，以便目的主机的IP层将数据部分上报给哪个进程处理

ICMP协议

Internet控制消息协议ICMP时IP协议的辅助协议

ICMP协议用来在网络设备间传递各种差错和控制信息，对于收集各种网络信息、诊断和排除各种网络故障等方面起着至关重要的作用

ICMP报文

数据链路层

数据链路层向网络层提供网内通信，负责组帧、物理编址、差错控制等功能，可以向网络层的IP、IPV6等协议提供服务，单位为帧（Frame）

ARP协议

  • 网络层和数据链路层之间

根据已知的对方IP地址获取其对应的MAC地址

工作原理

Ethernet协议

报文格式

  1. DMAC 目的MAC，IPV4为6字节，用于标识帧的接收者
  2. SMAC 源MAC，IPV4为6字节，用于标识帧的发送者
  3. Type 协议类型（0x0800表示数据包，0x0806表示ARP数据包）2字节
  4. FCS 4字节，用于数据校验

物理层

根据物理介质的不同将数字信号转为光信号、电信号或电磁波信号，单位为比特流（Bit stream）

常见的介质有 ：双绞线、光纤、电磁波等
