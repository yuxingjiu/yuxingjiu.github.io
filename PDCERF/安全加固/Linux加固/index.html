账号与口令安全

超级用户 : 在Unix系统中，只要将用户的UID和GID设置为0就可以将其变成超级用户，但并不是所欲的超级用户都能容易登录到Unix系统中，这是因为Unix系统使用了可插入认证模块（PAM）进行认证登录，PAM要求超级用户只能在指定的终端上进行访问

用户信息

  1. /etc/passwd : 存储的是账号信息 => 其存储格式 用户名:密码:UID:GID:描述信息:用户目录:登录shell 其中密码为X表示代替密码散列，*表示不可登陆

  2. /etc/shadow : 存储的是密码信息 => 其存储格式 用户名:加密口令:上次修改口令日期:口令两次修改最小天数:口令最大使用天数:失效前多少天警告:被禁止登录前的有效天数:账号被禁止登录时间:保留域

组信息

  1. /etc/group : 存储的是组账号信息
  2. /etc/gshadow : 存储的是组账号密码信息

  1. 限制使用su命令的账户

su命令同于在不同账户之间切换

为了增强系统安全性，可以对su命令的使用权限进行控制，只允许root和wheel组的账户使用su命令

vim /etc/pam.d/su

auth	required	pam_wheel.so	use_uid # 将这一行加入到配置文件中
  1. 设置锁定登录失败的次数和时间
vim /etc/pam.d/system-auth

auth	required	pam_faillock.so	preauth audit deny=3 even_deny_root unlock_time=300
auth	[default=die]	pam_faillock.so authfail audit deny=3 even_deny_root unlock_time=300
auth	sufficient	pam_faillock.so authsucc audit deny=3 even_deny_root unlock_time=300

pam_faillock.so配置项说明

​ authfail 捕获用户登录失败的事件

​ deny=3 用户连续登录失败次数超过3次即锁定

​ unlock_time=300 普通用户自动解锁时间为300秒

​ even_deny_root 同样限制root账户

  1. 密码复杂度

口令复杂度通过/etc/pam.d/password-auth和/etc/pam.d/system-auth文件中的pam_pwquality.so和pam_pwhistory.so模块实现

vim /etc/pam.d/system-auth

password	requisite	pam_pwquality.so minlen=8 minclass=3 enforce_for_root try_first_pass local_users_only retry=3 dcredit=0 ucredit=0 lcredit=0 ocredit=0
password	required	pam_pwhistory.so use_authtok remember=5

pam_pwquality.so配置项

配置项说明
minlen=8口令长度至少包含8个字符
minclass=3口令至少包含大写字母、小写字母、数字和特殊字符中的任意3种
ucredit=0口令包含任意个大写字母
lcredit=0口令包含任意个小写字母
dcredit=0口令包含任意个特殊字符
ocredit=0口令包含任意个特殊字符
retry=3每次修改最多可以尝试3次
enforce_for_root本设置对root账户同样有效

pam_pwhistory.so配置项说明

配置项说明
remember=5口令不能修改为过去5次使用过的旧口令
enforce_for_root本设置对root账户同样有效
  1. 口令复杂度

对应的密码策略模块

  1. pam_passwdqc : 对应的是/etc/login.defs配置文件
  2. pam_pwquality : 对应的是/etc/security/pwquality.conf配置文件

/etc/security/pwquality.conf文件 => 改配置文件修改完成后对当前所有用户都有影响

vim /etc/security/pwquality.conf

retry=N;		# 定义登录/修改密码失败时，可以重试的次数
difok=N;		# 定义新密码中必须有几个字符要与旧密码不同
minlen=N;		# 定义用户密码的最小长度
dcredit=N;		# 定义用户密码中必须包含多少个数字
ucredit=N;		# 定义用户密码中必须包含多少个大写字母
lcredit=N;		# 定义用户密码中必须包含多少个小写字母
ocredit=N;		# 定义用户密码中必须包含多少个特殊字符（除数字、字母外）

/etc/login.defs文件 => 该配置文件修改完成后是对新建用户生效

vim /etc/login.defs

PASS_MAX_DAYS 99	# 密码的最大有效期 99999 : 永久有效
PASS_MIN_DAYS 0		# 是否可修改密码 0表示随时修改，非0表示多少天后可修改
PASS_MIN_LEN 5		# 密码最小长度，使用pam_cracklib module 这参数即无效
PASS_WARN_AGE 7		# 密码失效前多少天在用户登录时通知用户修改密码

  1. 文件和目录的权限

Unix系统中的每个文件和目录都有访问许可权限，文件或目录的访问权限分为三种

  1. 可读
  2. 可写
  3. 可执行

有三种不同类型的用户对文件或目录进行访问

  1. 文件所有者
  2. 同组用户
  3. 其他用户

每一文件或目录的访问权限有三组，每组用三位表示

  1. 文件属主的读、写、执行
  2. 文件属组账号成员的读、写、执行
  3. 系统中其他用户的读、写、执行
  1. 文件系统

文件系统类型 : Ext4、Ext3、ReiserFS、XFS、JFS等

文件系统安全 : 访问权限、文件系统加密

日志安全

大部分Linux发型版默认的日志守护进程为rssylog位于/etc/rsyslog或/etc/rsyslog.d配置文件为/etc/rsyslog.conf任何希望生成日志的程序都可以项rsyslog发送信息，为了方便查阅，可以吧内核信息与其他信息分开，单独保存到一个独立的日志文件中，默认配置中日志文件通常都保存在/var/log目录下

日志类型

  1. 系统日志类型
  • 连接时间日志 /var/log/wtmp、/var/log/utmp 由多个程序执行，记录用户登录时间
  • 进程统计 由系统内核执行，为系统基本服务提供命令使用统计
  • 错误日志 /var/log/messages 由syslogd（rsyslog）守护记录
/var/log/boot.log		系统引导日志，可以使用 dmesg、more查看
/var/log/dmesg			内核启动日志
/var/log/messages		系统日志
/var/log/maillog		邮件系统日志
/var/log/xferlog		FTP系统日志
/var/log/secure			安全信息和系统登录与网络连接的信息
/var/log/wtmp			记录所有用户登陆和注销信息，需要使用 last 查看
/var/run/utmp			记录现在登录的用户 who -u 查看
/var/log/lastlog		记录每个最后登录的信息 lastlog 命令查看
/var/log/btmp			记录登录错误信息 last -f btmp 查看
/var/log/auth.log		记录需要身份确认的操作信息
/var/log/spooler		News日志
/var/log/rpmpkgs		RPM软件包
/var/log/cron			cron（定制任务）日志

/etc/logrotate.conf		配置文件，日志轮替

  • 应用程序（HTTP、FTP）创建的日志

/etc/rsyslog.conf设置，使用rsyslog日志服务器

/etc/rsyslog.conf

#### MODULES ####			# 启动/sbin/rsyslogd要加载的模块
$ModLoad module-name

# $ModLoad是关键字 (/sbin/rsyslog程序中定义的变量名)
# module-name 出现在/lib64/rsyslog/中的模块名（不需要带.so）

#### GLOBAL DIRECTIVES #### # 定义日志格式

#### RULES ####
faility.priority	target
# faility : 表示产生日志的设备
# priority : 表示产生日志的级别
# target : 表示日志存放的地方

facility => 日志分类，表示产生日志的设备（日志类型）

auth			# PAM 产生的日志
authpriv		# ssh、ftp等登录信息的验证信息
cron			# 时间任务相关
kern			# 内核
lpr				# 打印
mail			# 邮件
make(syslog)	# rsyslog服务内部的信息，时间标识
news			# 新闻组
user			# 用户程序产生的相关信息
uucp			# unix to unix copy, unix主机之间的通讯
local1~7		# 自定义的日志设备

priority => 日志等级，表示产生日志的级别

7 => debug		# 调试信息、日志信息最多
6 => info		# 一般信息的日志，最常用
5 => notice		# 最具有重要性的普通条件的信息
4 => warning	# 警告级别
3 => err		# 错误级别，某个功能或者模块不能正常工作的信息
2 => crit		# 严重级别，整个系统或者整个软件不能正常工作的信息
1 => alert		# 需要立刻处置的错误
0 => emerg		# 内核崩溃等严重信息
none 			# 什么都不记录

连接符

.xxx => 表示大于等于xxx级别的信息
.=xxx => 表示等于xxx级别的信息
.!xxx => 表示在xxx之外的等级的信息

防火墙和SELinux

firewalld是Centos7开始的Linux的一大特性，最大的两个好处是 支持动态更新，不需要重启服务 和 加入了防火墙 zone 的概念

firewalld

区域

通过将网络划分成不同的区域（通常情况下成为zones）制定出不同区域之间的访问控制策略来控制不同信任程度区域之间转发的数据流

默认区域 : 在配置规则时，不指定区域规则被认定为写入默认区域（puublic）

firewall-cmd --get-zones			# 显示当前支持的区域列表
firewall-cmd --get-default-zone		# 显示默认的区域

常见的预定义区域

丢弃区域（Drop Zone）: 任何进入的数据包都将被丢弃，这个类似 iptables -j
阻塞区域（Block Zone）: 任何进入的网络连接，返回icmp-host-prohibted，只有服务器已经建立的连接会被通过
公共区域（Public Zone）: 只接受哪些被设定的连接
外部区域（External Zone）: 这个区域相当于路由器启用伪装（masquerading）选项，只有指定的连接会被接受
隔离区域（DMZ Zone）: 如果想要只允许给定的部分服务能被外部访问，可以在DMZ区域中定义
工作区域（Work Zone）: 这个区域，只能定义内部网络
家庭区域（Home Zone）: 这个区域专门用于家庭环境
内部区域（Internal Zone）: 这个区域和工作区域（Work Zone）类似，只能通过被选中的连接
信任区域（Trusted Zone）: 该区域中的接口将被允许所有的网络通信

区域操作命令

firewall-cmd --get-zone-of-interface=ens33				# 根据接口获取区域信息
firewall-cmd --zone=home --add-interface=ens33			# 临时将接口加入到区域
firewall-cmd --zone=home --change-interface=ens33		# 临时修改接口所属区域
firewall-cmd --zone=home --remove-interface=ens33		# 临时从区域中删除一个接口
firewall-cmd --zone=home --query-interface=ens33		# 查询区域中是否包含某接口，未指定区域名，则为默认区域
firewall-cmd --zone=public --list-services				# 查询区域中已启用的服务
firewall-cmd --set-default-zone=home					# 设置默认区域
firewall-cmd --get-active-zones							# 查看当前活跃区域
firewall-cmd --list-all-zones							# 列出全部区域及特性
firewall-cmd --zone=public --list-all					# 列出指定区域特性

SELinux

  1. 安全增强Linux
  2. 安全子系统
  3. 强制访问控制（MAC）
  4. 美国国家安全局开发
  5. 以GUN GPL形式开源发布

SELinux实现了强制访问控制MAC，每个进程和系统资源都有一个特殊的安全标签，资源能否被访问除了DAC规定的原则外，还需要判断每一类进程是否拥有对某一类资源的访问权限

SELinux分为三种模式

  1. permissive : SELinux仅打印告警信息而不强制执行
  2. enforcing : SELinux安全策略被强制执行
  3. disabled : 不加载SELinux安全策略
vim /etc/selinux/config
# 关闭SELinux
SELINUX=disabled

getenforce		# 查询SELinux模式
setnforce 1		# 设置SELinux模式，0表示permissive模式，1表示enforcing模式
sestatus		# 查询SELinux系统状态
