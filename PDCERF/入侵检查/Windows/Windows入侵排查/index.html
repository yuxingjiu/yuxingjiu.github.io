Windows入侵排查

发表于 2023-07-07
  • 余星酒
首页 应急响应 入侵排查

对 windows 系统中的文件、进程、系统信息、日志记录等进行检测，挖掘 Windows 系统中是否具有异常情况

# 可疑用户

# 隐藏账号发现与删除

隐藏账号，是指黑客入侵后为了能够持久保持对计算机的访问，在计算机系统中建立的不被轻易发现的计算机账号

# $ 隐藏账号

  1. 最为简单的隐藏账号建立 net user test$ test /add && net localgroup administrators test$ /add
  2. 其中 $ 符号可以导致系统管理员在使用 net user 时无法查看到 test$ 用户

  1. 可以在 计算机管理 => 本地用户和组 => 用户 中可以发现 test$ 用户

# 注册表修改

注册表地址 ： HKEY_LOCAL_MACHINE\SAM\SAM\Domains\Account\Users

  1. 通过修改注册表项来达到计算机管理也无法发现用户被创建

  1. 选择刚刚新建的 test$ 选择导出

  1. 之后删除 test$ 用户，重新导入

  1. 虽然无法看到，但是并不存在于管理员组中，切换用户添加用户拒绝访问

# 影子账户

和修改注册表类似，只不过需要修改一些值后才能进行导出

  1. 复制管理员账户的 F 值

  1. 将其粘贴到 test$ 账号对应的 F 值

修改完成后，依次导出 test和其对应的十六进制值（2个），删除test和其对应的十六进制值（2个），删除test和其对应的十六进制值（2个），删除test 用户后，再次导入这两个文件

  1. 通过 net user 发现可疑用户，直接 net user 用户名 /del 删除即可

  1. 通过 net user 无法发现隐藏账户，需要通过计算机用户和组管理或者 net localgroup users 查看

  1. 克隆账号，即命令和本地用户和组都无法查找到可以账户，可以通过注册表进行查看 HKEY_LOCAL_MACHINE/SAM/SAM/Domains/Account/Users/Names 查看用户信息

修改完成后点击刷新

删除对应的账号以及 16 进制信息

  1. 利用 D 盾进行查杀

# 网络连接

# 手动检测

  1. 网络连接检测
netstat -ano -p [tcp,udp] | findstr "EST"  # 分析已经建立连接tcp或udp连接

如果没有找到对应的连接，可以查看本机的监听端口有哪些

netstat -ano -p [tcp,udp] | findstr "LIST"  # 分析正在监听的tcp或udp连接
  1. 找到对应的连接连接后，根据 PID 查找对应的执行文件，保存其路径
tasklist /scv | findstr "PID"
  1. 找到恶意文件，结束其进程
taskkill /PID "PID"
  1. 复制路径删掉恶意程序
  2. 在服务、开机启动项、计划任务中分别查找有无残留
sc delete "Server-name" # 在知道恶意服务名称的情况下，执行该命令将其删除

# 工具

  5. 火绒剑独立版

# 启动项、服务、计划任务

# 启动项

在 win2008 之前的版本通过 msconfig 来查看启动项，之后任务管理器都是自带的

  1. 通过输入 msconfig 查看有无异常命名的启动项目，有则取消勾选命名异常的启动项目，根据命令中显示的路径删除文件
  2. 通过注册表查看开机启动项是否正常
HKEY_CURRENT_USER\software\micorosoft\windows\currentversion\run
HKEY_LOCAL_MACHINE\software\micorosoft\windows\currentversion\Run
HKEY_LOCAL_MACHINE\software\micorosoft\windows\currentversion\Runonce
  1. 通过安全软件查看启动项、开机时间管理等
PcHunter
HiBitStartupManager
Process Explorer

# 计划任务

在计算机中可以通过设定计划任务，在固定的时间执行固定操作，一般情况下，恶意代码也可能在固定的时间设置执行，可以使用 at 对计划任务进行管理，高版本使用 schtasks

也可以直接在计划任务管理中查看

  1. 通过 计划事件 查看
  2. 通过第三方工具查看
PcHunter
HiBitStartupManager

# 服务

  1. 通过 services.msc 查看
  2. 通过第三方应用查看
HiBitStartupManager
PCHunter

# 目录、文件分析

# 文件时间属性

在 Windows 系统下，文件属性的时间属性有 ：创建时间、修改时间、访问时间（默认情况下禁止），默认是以修改时间作为展示

如果修改时间要早于创建时间，那么这个文件存在很大的可疑，菜刀可疑修改时间（修改的是修改时间，不是其他两种的时间）

# 最近打开文件分析

windows 系统中默认记录系统中最近打开使用的文件信息，可以在目录 C:\Documents and Settings\Administrator\Recent 下查看，也可以运行 %UserProfile%\Recent 查看

  1. Windows 自带的 find 指令，可以辅助工具来实现对文件内容的检索

find /c /n /i "eval" c:\inetpub\wwwroot\1.asp // 寻找文件中存在关键字的行号
  1. 利用 %UserProfile%\Recent 中的过滤条件

# 临时文件夹

临时文件夹，位于 C:\Documents and Settings\Administrator\Local Settings

使用运行，输入 %temp% 快速打开 temp 文件夹

  1. 查看 temp 文件夹是否会发现 PE 文件（exe、dll、sys）或者特别大的 tmp 文件
  2. 如果存在文件上传到 https://www.virustotal.com/gui/ 进行查看是否为恶意代码

# 开机启动文件

一般情况下，各种木马、病毒程序都会在计算机开启过程中自启动

  1. 利用操作系统的启动菜单 C:\Users\Administrator\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup
  2. 利用系统配置 msconfig
  3. 利用注册表 计算机\HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run

# 浏览器分析

在被黑客拿下的服务器，很有可能会使用浏览器进行网站的访问，因此可以查看浏览器记录查看浏览器是否被使用下载恶意代码

  1. 浏览器浏览痕迹查看（BrowsingHistoryView）
  2. 文件下载记录查看（BrowserDownloadsView）
  3. cookie 信息查看

http://launcher.nirsoft.net/downloads/index.html

# 进程分析

# 可疑进程发现与关闭

计算机与外部网络通信是建立在 TCP 和 UDP 协议商的，并且每一次通信都是不同的端口（0~65535）

如果计算机被种木马，肯定会与外部进行网络通信，那么可以通过分析网络连接状态，查找对应进程的 ID，然后关闭进程 ID 就可以关闭连接状态了

netstat -ano | find "ESTABLISHED" 	// 查看网络建立连接状态

tasklist /svc |find "PID" 					// 查看具体PID进程对应的程序

taskkill /PID pid值	/T  [/F 强制]		// 关闭进程

# 恶意进程发现与关闭

对于可执行程序，可疑直接使用杀毒软件进行查杀，但是并非所有的恶意程序都能被查杀，可疑手动查杀，使用工具 psexplore 利用 virustotal.com 进行分析，对恶意程序相关的服务进行关闭

# 日志分析

# windows 审查策略设置

  1. 开启审核策略后，若日后系统出现故障、安全事故则可以查看系统的日志文件、排除故障、追查入侵者的信息等
  2. windows server 2008 R2 ：开始 => 管理工具 => 本地安全策略 => 本地策略 => 审核策略

# windows 系统日志

  1. 开始 => 管理工具 => 事件查看器
  2. 运行 eventvwr.msc ，打开事件查看器

# windows 日志筛选

在 windows 系统中日志管理工具提供了筛选功能以方便用户进行查看（不同的类型的日志具有不同的编号，每一种操作又有不同的日志 ID）
https://www.ultimatewindowssecurity.com/securitylog/encyclopedia/event.aspx?eventid=4732

4624 -> 登录成功
4625 -> 登录失败
4634 -> 注销成功
4647 -> 用户启动的注销
4648 -> 用户登录
4776 -> 用户凭据登录
4672 -> 使用超级用户（管理员）进行登录
4732 -> 成员已添加到启用安全性的本地组
4724 -> 尝试重置帐户密码
4722 -> 已启用用户帐户
