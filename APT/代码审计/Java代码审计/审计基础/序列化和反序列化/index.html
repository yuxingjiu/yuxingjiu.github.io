# 序列化和反序列化

# 认识序列化和反序列化

序列化 ：序列化就是把对象转换成字节流，便于保存在内存、文件、数据库中 ObjectOutputStream ​类的 writeObject() ​方法实现序列化

反序列化：反序列化即逆过程，由字节流还原成对象 ObjectInputStream ​​类的 readObject() ​​方法用于反序列化

对一个类进行序列化操作需要执行 ObjectOutputStream.writeObject() ​方法写入对象

对一个类进行反序列化操作需要执行 ObjectInputStream.readObject() ​方法从输入流中获取字节然后转为对象

​

  1. 一般来说，Java 的序列化和反序列化是指 Java对象序列化规范 ​
  2. 广义上来说，任何将 Java 对象与可持久性的数据进行转换的过程都可以称作是序列化和反序列化

Fastjson、Jackson、XMLEncoder/XMLDecoder、XStream 等

# 实现序列化和反序列化

能够进行序列化或反序列化的对象，需要实现 Serializable ​​和 Externalizable ​​接口

  1. java.io.Serializable ​ 这个接口中什么都没有，只需要继承即可实现其对应的功能

Java 中一个接口没有任何字段和方法称之为 Marker interface ​ 本身无意义，一般用来告知编译器实现它的类，JVM 知道这个标记有什么特殊含义，就会对该类进行一些特殊的处理

​

  1. java.io.Externalizable ​ 它帮助我们实现了 Serializable ​接口，必须实现 writeExternal() ​和 readExtrernal() ​

​

这两个接口有什么区别 ？为什么要区别认识 ？

  1. Serializable 无需重写，它是一个标记接口
  2. Serializable 提供 transient ​控制序列化属性，readObject 和 writeObject 也可以完成部分属性的序列化
  3. Externalizable 需要重写 readExternal ​和 writeExternal ​方法
  4. 如果同时实现了两个 interface 的时候，JVM 只会运行 Externalizable ​接口中的方法对序列化内容进行处理
  5. Externalizable 接口的实现方式一定要有默认的午餐构造函数
  6. Externalizable 无需产生序列化 ID（seriaVersionUID），而 Serializable 接口需要实现序列化 ID
  7. 相比较 Serializable，Externalizable 序列化、反序列化更加快速，占用内存较小
  8. FileInputStream流 ​ 俗称文件输入流，作用是将文件中的数据输入到内存中，是一个字节输入流，是 InputStream ​抽象类的一个子类
  9. FileOutputStream流 ​ 即字节输出流，用于处理原始二进制数据，将数据写入到文件，需要将数据转换成字节并将其保存到文件
  10. 对象的输出流 : ObjectOutputStream ​将指定的对象写入到文件完成对象的序列化过程
  11. ObjectInputStream ​反序列化流，将使用 ObjectOutputStream ​序列化的原始数据恢复为对象，以流的方式读取对象

​

# 反序列化漏洞

Java 反序列化漏洞产生因素

  1. 对不可信数据进行反序列化处理
  2. 系统应用环境里存在能够被利用的反序列化利用链（ Gadget ）

在反序列化漏洞中，将可以执行代码的位置称之为 sink

每种方法都会存在一个调用关系，最后这个方法可能会调用刚刚的 sink，而整个过程就是一个调用链（Gadget）

那么如何查看这个调用链 ？

通过 idea 的 debug 调试，在存在 sink 的地方打入断点
