# XXE

XXE 是 XML External Entity Injection（XML 外部实体注入）

当配置其 XML 解析功能允许外部实体引用时，攻击者可利用这一可引发安全问题的配置方式，实施任意文件读取、内网端口探测、命令执行、拒绝服务攻击等方面的攻击

  1. DTD（Document Type Definition，文档类型定义）部分是 XXE 攻击 的关键，我们可以将 XML 的 外部实体注入 拆分成 外部 、 实体 与 注入 这三部分来看
  2. 实体 意指 DTD 实体 ，它是用于定义引用普通文本或特殊字符的快捷方式的变量
  3. 外部 则与实体的使用方式有关，实体可分为 内部声明实体 和 引用外部实体
    1. 内部声明实体的定义格式形如 <!ENTITY 实体名称 "实体的值">
    2. 引用外部实体的定义格式形如 <!ENTITY 实体名称 SYSTEM "URI/URL"> 或 <!ENTITY 实体名称 PUBLIC "public_ID" "URI">
  4. 外部实体可支持 http、file 等协议。不同编程语言所支持的协议不同，Java 默认提供 对 http、https、ftp、file、jar、netdoc、mailto、gopher 等协议的支持；“注入” 则意 指攻击者的恶意数据可以诱使解析器在没有适当授权的情况下执行非预期命令或访 问数据。

漏洞成因是该 POC 使用了 XML 解析接口 javax.xml.parsers.DocumentBuilder，但未禁用外部实体

# DOS-POC

  1. 当 XML 解析器加载此文档时，可以发现它包含一个根元素 lolz ，其中包含文本 ＆lol9;
  2. 但是 ＆lol9; 是一个定义的实体，扩展为包含 10 个 ＆lol8; 的字符串
  3. 每个 ＆lol8; 字符串是一个定义的实体，扩展为 10 个 ＆lol7; 字符串等
  4. 所有的实体扩展都经过处理后，这个小于 1KB 的 XML 将扩展到 3GB
