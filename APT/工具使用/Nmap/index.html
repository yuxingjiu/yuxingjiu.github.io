# Nmap 基础介绍

# Nmap 端口状态

端口状态描述
Opend端口开放
Closed端口关闭
Filtered端口过滤，数据被防火墙拦截
UnFiltered未被过滤，但不能识别端口当前的状态
Openfiltered开放或者被过滤，发生在 UDP，IP，FIN，NULL 和 Xmas 扫描中
CloseFiltered关闭或者被过滤，发生在 IP，ID，idle 扫描

# Nmap 扫描原理

管理员权限默认使用 -sS ，无管理员权限默认使用 -sT （使用 -sS，系统禁止）

# TCP 完整连接扫描

-sT ：TCP 完整连接扫描的方式，使用系统网络 API 连接向目标主机的端口发起连接，如果无法连接，说明该端口关闭

该方式扫描速度比较慢，而且由于建立完整的 TCP 连接会在目标机上留下记录信息，不够隐蔽
因此 TCP 完整连接扫描是 TCP SYN 无法使用才考虑选择的方式

# TCP-SYN 扫描

-sS 、 -PS
该方式发送 SYN 到目标端口

  • 如果收到 SYN/ACK 回复，那么判断该端口是开发
  • 如果收到 RST 包，那么判断该端口是关闭
  • 如果没收到回复，那么判断该端口是被屏蔽

# TCP-ACK 扫描

-sA 、 -PA ：向目标主机的端口发送 ACK 包

  • 如果收到 RST 包，说明该端口没有被防火墙屏蔽；
  • 没有收到 RST 包，说明被屏蔽。

但是该方式只能用于确定防火墙是否屏蔽某个端口，可以结合 TCP-SYN 的方式来判断目标主机防火墙的状况

# TCP 秘密扫描

-sF 、 -sN 、 -sX ：使用 TCP Null/FIN/Xmas 秘密扫描方式来协助探测对方的 TCP 端口状态。
这三种扫描方式被称为秘密扫描（Stealthy Scan）
原理：FIN 扫描向目标主机的端口发送的 TCP FIN 包或 Xmas tree 包 / Null 包

  • 如果收到对方 RST 回复包，那么说明该端口是关闭的；
  • 没有收到 RST 包说明端口可能是开放的或被屏蔽的（open|filtered）。
    其中 Xmas tree 包是指 flags 中 FIN URG PUSH 被置为 1 的 TCP 包；NULL 包是指所有 flags 都为 0 的 TCP 包。

# UDP 扫描

-sU 、 -PU ：UDP 扫描方式用于判断 UDP 端口的情况
原理：向目标主机的 UDP 端口发送探测包

  • 如果收到回复 ICMP port unreachable 就说明该端口是关闭的
  • 如果没有收到回复，那说明 UDP 端口可能是开放的或屏蔽的

# Nmap 辅助参数

# 调整扫描速度

-T <0-9>
-T0 非常慢，用于逃避 IDS 检测等
-T1 缓慢，用于逃避 IDS 检测等
-T1 降低速度减少带宽占用，在公网上可用于绕过检测
-T3 默认，根据目标反应自动调整时间
-T4 常用的扫描方式
-T5 极速扫描，降低扫描的精确度
--min-rate 速率[数字] 每秒发送数据包的速度不低于
--max-rate 速率[数字] 每秒发送数据包的速度不超过

# 调整版本检测深度

--version-intensity [0-9] 数值越大，检测越深，默认 7
--version-light 相当于 --version-intensity 2
--version-all 相当于 --version-intensity 9
--version-trace 显示详细的版本扫描活动（用于调试）

# 其他辅助参数

--reason 扫描结果的原因
--exclude 去除某个 IP
--excludefile 去除指定文件中的 IP
--exclude-ports 去除指定端口
-oN 扫描结果输出为普通文件
-oG 扫描结果输出为分类保存文件
-iL 导入需要扫描的列表
-n 不解析 DNS
--system-dns 使用系统的 DNS 解析
--traceroute 路由跟踪

# Nmap 脚本目录

/usr/share/nmap/scripts/ 存放的 nmap 脚本
脚本大致分为

# Nmap 扫描

# 主机发现

# 列表扫描

-sL ：列表扫描是主机发现的一种简写形式，它仅列出指定网络中的每个主机，而不发送任何数据包到目标主机

# 无端口扫描

-sn ：告诉 Nmap 在主机发现之后不要执行端口扫描，并且仅打印出响应主机发现探针的可用主机，也称为 ping 扫描，

# 基于 ARP 主机发现

-PR ：通常在局域网中扫描，使用地址解析协议，本地局域网不会禁止 ARP 请求

# 基于 ICMP 主机发现

-PE ：使用 ICMP 扫描方式
-PP ：使用时间戳 Ping 扫描
-PM ：使用 ICMP 地址掩码 ping 扫描

# 基于 IP 主机发现

-PO

# 基于 TCP 主机发现

-PS ：此选项发送一个空的 TCP 数据包，该数据包设置了 SYN 标志（过程和 TCP 半连接扫描类似）
-PA ：设置了 TCP ACK 标志，这种 ACK 数据包声称是通过已建立的 TCP 连接确认数据的，但是不存在这种连接，因此，远程主机应始终以 RST 数据包作为响应，以公开其在进程中的存在，

# 基于 UDP 主机发现

-PU ：将 UDP 数据包发送到给定的端口

# 基于 SCTP 主机发现

-PY ：发送包含最小 INIT 块的 SCTP 数据包，INIT 块向远程系统建议您正在尝试建立关联，通常，目标端口将关闭，并且 ABORT 块将被发回，如果端口恰好是开放的，则目标将采取 SCTP 四次握手的第二步 通过响应一个 INIT-ACK 块

# 端口扫描

# 指定端口

  1. 指定一个端口扫描
  1. 指定多个端口进行扫描
  1. 指定一段端口进行扫描
nmap -p 1-1000 192.168.1.100
  1. --top-ports （默认1000）开放最高的TCP端口

# IP 协议扫描

-sO IP 协议扫描使您可以确定目标计算机支持哪些 IP 协议（TCP，ICMP，IGMP 等）

# TCP 扫描

-sS 半开放扫描，速度较快，隐蔽性好
-sT 完整连接扫描，扫描速度快，准确性高，但容易被防火墙发现
-sA ACK 扫描，用于发现防火墙规则，确定它们是有状态的还是无状态的，哪些端口是被过滤的
-sW TCP Window 扫描，窗口扫描与 ACK 扫描完全相同，当收到 RST 包时，根据 TCP 窗口的值是正数还是 0 来判断目标端口是开放还是关闭的，如果扫描的端口都是开放状态或者只有少数几个是关闭状态就很不准确
-sM TCP Maimon 扫描，与 NULL，FIN 和 Xmas 扫描完全相同，不同之处在于探针为 FIN / ACK。根据 RFC 793（TCP），应响应此探测（端口是打开还是关闭）来生成 RST 数据包。但是，Uriel 注意到，如果端口打开，则许多 BSD 派生的系统只会丢弃该数据包。
-sF 、 -sN 、 -sX 指定使用 TCP Null， FIN， and Xmas scans 秘密扫描方式来协助探测对方的 TCP 端口状态

# UDP 扫描

-sU 扫描存在于 UDP 端口的程序，速度较慢

# SCTP 扫描

-sY SCTP INIT Ping 扫描，流控制传输协议，向目标发送 INIT 包，需要 root 权限，SCTP INIT 扫描与 TCP SYN 扫描的 SCTP 等效。它可以快速执行，在不受限制性防火墙阻碍的快速网络上每秒扫描数千个端口。像 SYN 扫描一样，INIT 扫描也相对不引人注目且隐秘，因为它从未完成 SCTP 关联
-sZ SCTP COOKIE ECHO 扫描是更高级的 SCTP 扫描，SCTP 实现应在打开的端口上静默丢弃包含 COOKIE ECHO 块的数据包，但在端口关闭时发送 ABORT，这种扫描类型的优点在于，<font color="#ff0000"> 端口扫描不像 INIT 扫描那样明显 </font>，同样，可能有非状态防火墙规则集阻止 INIT 块，但不会阻止 COOKIE ECHO 块，一个好的 IDS 也将能够检测 SCTP COOKIE ECHO 扫描，缺点是 SCTP COOKIE ECHO 扫描无法区分 open 和 filtered 端口，open|filtered 在两种情况下都处于状态

系统信息、服务信息等一起扫描

-A 是一个混合参数

# 脚本使用

-sC 相当于 --script = default
--script=Lua脚本 Lua 脚本是逗号分隔的列表目录，脚本文件或脚本类别
--script-args=n1 = v1，[n2 = v2，...] 提供脚本参数
--script-args-file=文件名 在文件中提供 NSE 脚本 args
--script-trace 显示所有发送和接收的数据
--script-updatedb 更新脚本数据库
--script-help=Lua脚本 ：显示有关脚本的帮助
