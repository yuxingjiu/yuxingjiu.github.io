  4. TFP 是文件传输协议

FTP ：文件传输协议，上层协议是 TCP

端口号 ：<mark style="background: #ABF7F7A6;">20 （传输数据）和 21 （传输控制信息）</mark>

FTP 包括两个组成部分 ：FTP 服务器（存储文件，用户通过 FTP 协议访问位于 FTP 服务器上的资源）、FTP 客户端

  1. FTP 的安全版本是 <mark style="background: #ABF7F7A6;">SFTP</mark>

  2. 在没有账号的情况下，通过 FTP 使用的用户名是 <mark style="background: #FF5582A6;">anonymous</mark>

使用 FTP 传输文件时，用户需要通过向 FTP 服务器提供凭据来获得文件传输许可，但是某些公共 FTP 服务器不需要凭据即可访问其文件，但是无法保证数据传输的安全性，任何未加密公共网络上的数据发送都是非常危险的，为了保护传输数据的安全，即衍生出 FTP 和 SFTP

FTP 有两种登录方式 ：<mark style="background: #ABF7F7A6;"> 匿名登录 </mark>（账号为 anonymous，密码为任何合法 email 地址）和 <mark style="background: #ABF7F7A6;"> 授权登录 </mark>（用户名为用户在远程系统中的用户账号，密码为用户在远程系统中的用户密码）

  1. FTP 消息 “登录成功” 的响应代码是 <mark style="background: #FF5582A6;">230</mark>

  2. 用于下载我们在 FTP 服务器上找到的文件的命令是 <mark style="background: #FFB86CA6;">get</mark>，用于上传本地文件到目标的命令是 <mark style="background: #FFB86CA6;">put</mark>

  3. 可以使用空白密码访问的共享名称是 <mark style="background: #FF5582A6;">WorkShares</mark>

  4. 可以使用 <mark style="background: #ABF7F7A6;">redis-cli</mark > 去连接 redis
