# 文件包含

  • 只要文件被包含了，无论什么都会被当成 PHP 代码执行
  • PHP 默认不会开启远程文件包含，想要开启需要在配置文件中添加 allow_url_include=on;
  • windows 也可以通过 SMB 服务开启某个设置来绕过 allow_url_include=on; 达到远程文件包含的效果

本地文件包含 ：简称 LFI ，指包含服务器上的资源

远程文件包含 ：简称 RFI ，指通过 http 协议包含其他地方的资源

# 函数介绍

include() : 如果包含的文件不存在，不会影响后面的代码执行

require() : 如果包含的文件不存在，会影响后面的代码执行

include_once()、require_once() : 如果文件被包含过一次，就不会再次包含

# php 伪协议

php 内置很多 url 风格的封装协议，类似于 fopen ()、copy ()、file_exists () 和 filesize () 的文件系统函数

# 包含日志文件

web 服务器，基本上把请求写入到他的日志文件中，有的是 access.log，有的是 error.log，所以可以尝试写入一些敏感的数据 <?php eval($_REQUEST[8]);?> ，然后让访问出错，这个就会被记录到日志文件中，利用本地包含这个日志文件，相当于得到了 shell，前提是得对日志文件有 读 的权限

# 包含 /proc/self/environ

这个文件是一个存放最新的环境变量信息的文件，是由 apache 服务器被访问时，会去更新的一个文件，更新其中的 User-Agent 信息，所以可以考虑吧在 User-Agent 中注入代码，然后包含这个文件获得 shell

# 绕过远程文件限制

在 PHP 的配置文件中，默认将 allow_url_include 设置为 Off，来限制 PHP 去加载远程的 HTTP/FTP 文件，这也有效的防御了远程文件包含，尽管 PHP 设置了 allow_url_include 和 allow_url_fopen 为 Off，但是没有阻止下载 SMB URL

当存在漏洞的 PHP 应用程序从攻击者控制的 SMB 共享机器上下载 PHP Webshell 时，SMB 共享机器应该允许其访问该文件，攻击者应该设置 SMB 机器可以匿名访问，因此，一旦存在漏洞的应用程序去 SMB 共享机器上下载 PHP Webshell 时，SMB 服务器不会判断任何身份，就可以成功包含这个 webshell

# 防御方法

  1. 设置白名单（进行包含的文件设置为一个数组，每次包含时进行匹配）
  2. 过滤危险字符（常见的伪协议 : php、data、file、http……）
  3. 设置文件目录（设置文件目录路由，把目录流量禁用就会显示 403）
  4. 关闭危险配置（两个配置文件进行 off allow_url_fopen、allow_url_include）
