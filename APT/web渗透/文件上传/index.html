文件上传

原理

在文件上传的功能处，若服务器端脚本语言未对上传的文件进行严格验证和过滤，导致恶意用户上传恶意的脚本文件时，就有可能获取执行服务端命令的能力，恶意用户可以通过上传一句话木马等 文件上传shell 从而达到控制 文件上传 网站的目的

method 表示上传的方法

在HTTP请求时，会添加一个 Content-Type 里面包含 multipart/form-data 还包括一个分隔符 boundary，而这条分割符是随机字符产生的，这个Content-Type表示文件在传输中内容程中定义的形式

每个分割符前面会自动添加两个-

第一个表示查看文件的基础信息

第二个表示文件内容的的详细信息，也会存在一个Content-Type表示文件类型

第三个表示文件上传的NAME

第四个表示文件闭合 自动添加两个-

分块传输

在头部加入 Transfer-Encoding: chunked 之后，就代表这个报文采用了分块编码 post 请求报文中的数据部分需要改为用一系列分块来传输。每个分块包含十六进制的长度值和数据，长度值独占一行，长度不包括它结尾的，也不包括分块数据结尾的，且最后需要用 0 独占一行表示结束 编码过程中长度需包括空格的长度。最后用 0 表示编码结束，并在 0 后空两行表示数据包结束，不然点击提交按钮后会看到一直处于 waiting 状态
