# XSS

  1. xss 原理及分类

用户输入的数据被当做前端代码执行

反射性、存储型、DOM 型

  1. xss 的特点
  1. 反射型 xss 提交的数据只对本次访问产生了影响（只会触发一次）会经过后端代码 echo 输出
  2. 存储型 xss 他会将 xss 语句存储到数据库或其他存储介质中，人访问这个页面的时候就会自动获取数据 会经过后端代码 echo 输出
  3. DOM 型 xss 是针对 dom 文档树进行的 xss 攻击 不会经过后端代码 dowcument.write ()、document.innerHTML ()、eval () 等
  1. xss 的利用方法

# CSRF

  1. csrf 原理

csrf 跨站请求伪造，原理就是网站的 cookie 在浏览器关闭时不会过期，cookie 的一般的过期时间都是一次会话，一次会话就是关闭浏览器，cookie 存在于浏览器中，但是不可能只访问一个网站，所以浏览器中的 cookie 不止一个，当访问网站的时候，浏览器会判断你访问的什么站，发送相应的数据包，其中就包含相应的 cookie

  1. csrf 和 xss 区别

csrf 是利用浏览器信任用户从而利用 cookie 的攻击手段

xss 是直接盗用对方的 cookie 在自己利用

csrf 不知道 cookie 是什么

xss 知道 cookie 是什么

  1. 防御方法
  1. 最好的方法是添加验证码（这个比 token 更有效）
  2. 添加 token 验证
  3. 对 referer 进行校验
  1. 绕过方法

基本上都是针对 referer 校验不完善进行的绕过

  1. 正则匹配域名，可以将文件名设置为该域名，正则就可以匹配到域名，认为是合法的就会放行
  2. 空 referer，在某些业务需求中可以绕过
