eval和assert的区别

相同点

二者都可以执行php语句，eval规范更加严格切必须符合php代码要求，而assert没有那么严格，也可以是用assert_option()来进行对assert的控制

不同点

eval中的引号必须是字符串，且为双引号，因为单引号不能解析字符串里的变量

  1. eval会报语法错误，assert会执行

<?php $_GET[a]($_GET[b]);?> => a=eval或assert b=$_REQUEST[‘a’]

preg_replace("/test/e", $_GET["a"], "just test"); php5.5后失效

?>

漏洞防御

  1. 使用硬编码 : 将参数固定，避免直接读取用户提交数据
  2. 将一些执行命令放入一个数组进行白名单限制
  3. 限制用户提交的参数为整形，根据对应关系执行命令
  4. 可变命令应当被禁止
