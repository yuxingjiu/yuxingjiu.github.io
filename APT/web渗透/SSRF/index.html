SSRF

漏洞简介

  1. SSRF(服务器端请求伪造)是一种攻击者构造形成由服务端发起请求的一个安全漏洞
  2. 一般情况下SSRF攻击的目标是从外网无法访问的内部系统（正因为它是由服务端发起的，所以它能够请求到与它相连而与外网隔离的内部系统）

漏洞原理

由于服务端提供了从其他服务器获取数据的功能且没有对目标地址做过过滤与限制

漏洞类型

  1. 显示响应

它显示攻击者的响应，因此在服务器获取攻击者要求的URL后，它将把响应发送回攻击者，返回结果到客户端，如传送一个网址，会返回这个网址的界面或对应的html代码

  1. 不显示响应

和上面正好相反，不会返回结果到客户端

可以使用抓包工具Burp、DNSbin等类似工具，这些工具可以通过强制服务器向攻击者控制的服务器发送DNS或HTTP请求来确认服务器（易验证，难利用）

  1. 响应部分数据

这可能是部分数据或错误信息，为攻击者提供了更多信息

漏洞利用

  1. 让目标站点发起网络请求，可以控制目标去访问百度

  2. DNS注入通过load_file()函数读取远程地址 ==> 让目标站点发起网络请求

  3. 要能访问公网或者内网

程序代码

<?php
    $URL = $_GET['url'];
    $CH = CURL_INIT();
    CURL_SETOPT($CH, CURLOPT_URL, $URL);
    CURL_SETOPT($CH, CURLOPT_HEADER, FALSE);
    CURL_SETOPT($CH, CURLOPT_RETURNTRANSFER, TRUE);
    CURL_SETOPT($CH, CURLOPT_SSL_VERIFYPEER, FALSE);
    CURL_SETOPT($CH, CURLOPT_FOLLOWLOCATION, TRUE);
    $RES = CURL_EXEC($CH);
    CURL_CLOSE($CH);
    echo $RES;
?>

file协议

file协议探测文件内容(?url=file:///c:/www/1.php)

dict协议

显示开放端口(?url=dict://127.0.0.1:81)，如果开放会显示报错信息，端口关闭无任何显示

ftp

?url=ftp://127.0.0.1:21可以测试访问，如果设置了匿名访问（anonymous）可以查看文件

防御

一般这种需要请求的接口，首先看这个请求是需要转发内网还是外网

  1. 如果是内网，就限制其不能访问其他地址，内网一般都是请求固定的几个接口，将这些接口固定，无需通过参数传递

  2. 如果是允许请求外网，那么应该是不能请求内网的，就可以把内网IP和域名全部进行过滤，这个域名需要注意的是有一些域名它的解析本来就是解析到内网的，就是可以先申请一个域名，然后把这个域名的解析地址让他强制解析到内网，接到一个内网地址，所以域名过滤的时候要过滤这个域名解析后的地址

  3. 如果是进允许请求外网网络的话，也可以限制一下请求频率防止被DDOS

  4. 如果是请求端口的话，可以做相对应的白名单限制

  5. 协议的话，只能是http或https

总的来说，ssrf其实尽量是不要通过服务器来去发送这些请求

利用302url跳转

=> 通过公网服务器创建php文件，存在ssrf漏洞的地方访问该公网服务器的php文件，php文件中是各种协议的请求，这样可以绕过对协议的控制

当url存在临时（302）或永久（301）跳转时，则继续请求跳转后的URL，那么可以通过http(s)的链接302跳转到gopher协议上

<?php
    $schema = $_GET['s'];
    $ip = $_GET['i'];
    $port = $_GET['p'];
    $query = $_GET['q'];
    if (empty($port)) {
        header("Location: $schema://$ip/$query");
    } else {
        header("Location: $schema://$ip:$port/$query");
    }
?>

利用xip.io或xip.name

xip.io是一个特殊的host，dns解析这个host会自动去掉后缀

http://127.0.0.1.ip.io => 127.0.0.1

www.127.0.0.1.xio.io => 127.0.0.1

foo.http://127.0.0.1.xip.io => 127.0.0.1

http://127.0.0.1.ip.name resolves to 127.0.0.1

www.127.0.0.1.xio.name resolves to 127.0.0.1

foo.http://127.0.0.1.xip.name resolves to 127.0.0.1
